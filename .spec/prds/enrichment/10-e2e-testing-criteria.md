# E2E / Human Testing Criteria — Route Enrichment

Version 1.0.0 · 2026-07-10 · **50 criteria across 12 UCs** — every AC referenced by ≥1
criterion. Types: [human-gate] · [e2e-automated] (Maestro, real device, live Convex) ·
[integration-test] (vitest vs real dev deployment; pipeline tier hits the REAL LLMs) ·
[api-contract] · [build-gate]. AC refs are positional within each UC (AC-1 = first ☐).

## GEN: Enrichment Generation

### UC-GEN-01: Generate a grounded "why" from structured route attributes

| # | Criterion | AC Ref | Type | Setup | Pass/Fail |
|---|---|---|---|---|---|
| T-GEN-001 | Real-LLM sample generation produces a grounded single paragraph | AC-1, AC-5 | [integration-test] | Real dev deployment; `backfill --sample=10` on eligible routes (`geometryStatus='generated'`); real z.ai GLM-5.2 | ≥1 `generated` row per eligible sampled route; `whyText` non-empty, one paragraph, ≤320 chars, lead sentence ≤100 |
| T-GEN-002 | Every claim in a generated paragraph traces to a supplied fact | AC-2 | [integration-test] | Run grounding verifier on the T-GEN-001 sample | Verifier maps 100% of extracted claims to a groundingFacts entry; any unmapped claim ⇒ fail |
| T-GEN-003 | Grounding snapshot persisted with vision seam absent | AC-3 | [api-contract] | Inspect a generated row | `groundingFacts.structured` + `geometryDerived` present; `visual` absent; `schemaVersion`, `promptVersion`, `model`, `inputsContentHash` set |
| T-GEN-004 | Batch is resumable over the full eligible catalog | AC-4 | [integration-test] | `backfill` with small `batchSize`; interrupt; re-run with `continueCursor` | Second run continues (no reprocessing of completed routes); report totals reconcile |
| T-GEN-005 | Provider/model live only in the `enrichment` tier | AC-6 | [build-gate] | Grep pipeline files; read `models.ts` | `enrichment` tier = `zai`/`glm-5.2`; zero provider/model literals in `curatedEnrichment*` files; tsc clean |

| # | Criterion | AC Ref | Type | Setup | Pass/Fail |
