**Type:** [FEATURE]

## CRITICAL CONSTRAINTS

1. **MUST** block downloads over cellular connection - WiFi-only requirement to prevent data charges
2. **STRICTLY** validate region bounds before download - prevent invalid or excessive region sizes
3. **MUST** provide progress updates every 5% or 5 seconds - keep users informed during long downloads
4. **NEVER** allow parallel downloads that exceed device storage - manage download queue carefully
5. **MUST** persist download state across app restarts - resume interrupted downloads automatically

---

## SPECIFICATION

**Objective:** Implement Mapbox offline pack download manager with queue management, progress tracking, storage limit detection, and WiFi-only enforcement.

**Success State:** OfflineManager module successfully downloads map regions, tracks progress accurately, pauses/resumes downloads appropriately, validates WiFi connection, persists state across app restarts, and manages storage limits effectively.

## ACCEPTANCE CRITERIA

### AC-001: Download Queue Management
**GIVEN** user requests multiple offline region downloads
**WHEN** downloads are queued
**THEN** system processes downloads one at a time or with configurable concurrency
**AND** system prioritizes downloads by request time or user preference
**AND** system prevents simultaneous downloads that would exceed storage limits
**AND** system provides queue status to user

**Verify:** Request multiple region downloads, verify they queue properly, check storage limits are respected.

### AC-002: Progress Tracking Callbacks
**GIVEN** offline region download is in progress
**WHEN** download progresses
**THEN** system invokes progress callback with bytes downloaded/total bytes
**AND** system updates progress percentage every 5% or 5 seconds
**AND** system provides estimated time remaining based on download speed
**AND** system reports resource download completion

**Verify:** Start download, observe progress callbacks, verify percentage updates incrementally, check ETA calculation.

### AC-003: WiFi Connection Validation
**GIVEN** user attempts to download offline region
**WHEN** device is not connected to WiFi
**THEN** system blocks download immediately
**AND** system displays "WiFi Required" error message
**AND** system does not queue download until WiFi is available
**AND** system automatically starts queued downloads when WiFi connects

**Verify:** Disable WiFi, attempt download, verify error message. Enable WiFi, verify queued download starts automatically.

### AC-004: Storage Limit Detection
**GIVEN** device has limited storage available
**WHEN** user requests region download that would exceed available storage
**THEN** system calculates required storage for region
**AND** system compares against available device storage
**AND** system blocks download if insufficient storage
**AND** system displays "Not Enough Storage" error with required amount

**Verify:** Fill device storage, attempt large region download, verify storage error appears with required amount.

### AC-005: Download Pause and Resume
**GIVEN** offline region download is in progress
**WHEN** user pauses download or loses WiFi connection
**THEN** system pauses download gracefully
**AND** system preserves downloaded data
**AND** system resumes download from last position when continued
**AND** system does not re-download completed resources

**Verify:** Start download, pause at 50%, resume, verify download continues from 50% not 0%.

### AC-006: Download State Persistence
**GIVEN** offline region download is in progress
**WHEN** app is backgrounded or killed
**THEN** system saves download state persistently
**AND** system resumes download automatically when app relaunches
**AND** system displays "Resuming download..." message if applicable
**AND** system recovers download progress from saved state

**Verify:** Start download at 40%, kill app, relaunch, verify download resumes from 40%.

### AC-007: Region Metadata Management
**GIVEN** offline region download completes successfully
**WHEN** system finalizes download
**THEN** system stores region metadata (name, bounds, size, date)
**AND** system associates metadata with Mapbox pack
**AND** system provides query interface for downloaded regions
**AND** system calculates total storage used by offline regions

**Verify:** Complete download, query for region metadata, verify all fields present and accurate.

## DESIGN

**References:**
- @rnmapbox/maps OfflineManager API: https://github.com/rnmapbox/maps/docs/offline-manager.md
- PRD Phase 2: Offline Maps & Local Routing - "Offline Region Download" section
- Mapbox Offline Documentation: https://docs.mapbox.com/android/maps/guides/offline/

**Patterns:**
- Singleton pattern for OfflineManager instance
- Queue-based download management
- Progress callback pattern for updates
- AsyncStorage for state persistence
- Event emitter for download status changes

**Anti-Patterns:**
- **DO NOT** allow downloads over cellular (WiFi-only)
- **DO NOT** skip storage validation (will cause crashes)
- **DO NOT** lose download progress on app restart
- **DO NOT** block UI thread during download
- **DO NOT** show technical error messages to users

**OfflineManager Interface:**
```typescript
interface OfflineManager {
  downloadRegion(params: RegionDownloadParams): Promise<void>
  pauseDownload(packName: string): Promise<void>
  resumeDownload(packName: string): Promise<void>
  deletePack(packName: string): Promise<void>
  getPacks(): Promise<OfflinePack[]>
  getPackSize(packName: string): Promise<number>
}

interface RegionDownloadParams {
  name: string
  bounds: [[number, number], [number, number]]  // [[sw_lng, sw_lat], [ne_lng, ne_lat]]
  styleURL: string
  minZoom: number
  maxZoom: number
  onProgress: (progress: DownloadProgress) => void
}
```

**Download State Machine:**
```
IDLE → DOWNLOADING → PAUSED → COMPLETE
  ↓         ↓           ↓
QUEUED   ← RESUME → FAILED
```

- Provide clear storage requirements to users
- Implement region management UI for deletion

**WiFi Enforcement:**
- Use React Native NetInfo for network detection
- Check connection type before allowing download
- Auto-start queued downloads when WiFi connects
- Show clear messaging about WiFi requirement

**Error Handling:**
- Download timeout: retry with exponential backoff
- Network changes: pause and show appropriate message
- Storage full: block and show storage management UI
- Corrupted pack: delete and offer re-download
