# CLR-001: Setup Wizard Flow Implementation

**Task ID:** CLR-001
**Epic:** Epic 1 - Shadow Setup
**Assigned To:** frontend-designer
**Estimate:** 480 min
**Type:** [FEATURE] [DESIGN]

---

## CRITICAL CONSTRAINTS

1. **MUST** block all app functionality until "Your Shadow" (local AI model) download completes - this is a hard gatekeeper requirement
2. **NEVER** allow model download over cellular connection - WiFi-only requirement to prevent data charges
3. **STRICTLY** follow LaneShadow UI/UX patterns from `styles/RULES.md` - copper-accented dark theme, industrial-warm aesthetic
4. **MUST** resume download automatically after app restart or interruption - download state persists across app lifecycle
5. **NEVER** show technical error messages to users - always provide user-friendly messaging with clear next steps

---

## SPECIFICATION

**Objective:** Implement onboarding setup wizard that guides users through mandatory local AI model download before accessing any app features.

**Success State:** User completes "Download Your Shadow" onboarding flow, model downloads successfully over WiFi, and app unlocks main functionality. Download progress is clearly visible with "Awakening Your Shadow" branding, and setup completion rate exceeds 90%.

---

## ACCEPTANCE CRITERIA

### AC-001: Welcome Screen Display
**GIVEN** user launches LaneShadow app for first time
**WHEN** app detects no local model is downloaded
**THEN** system displays "Welcome to LaneShadow" screen with copper-accented branding
**AND** user sees "Download Your Shadow" call-to-action button
**AND** user cannot bypass or dismiss the screen

**Verify:** Run app on fresh install, observe welcome screen appears with correct branding and no skip option.

### AC-002: WiFi Detection and Enforcement
**GIVEN** user is on welcome screen and taps "Download Your Shadow"
**WHEN** device is not connected to WiFi
**THEN** system displays "WiFi Required" alert with copper accent
**AND** system explains "Connect to WiFi to download your Shadow. Cellular downloads are not supported."
**AND** download button is disabled until WiFi is connected

**Verify:** Disconnect WiFi, tap download button, verify alert appears and button remains disabled. Connect WiFi, verify button becomes enabled.

### AC-003: Download Progress Tracking
**GIVEN** user has connected to WiFi and confirmed download
**WHEN** model download begins
**THEN** system displays "Awakening Your Shadow" progress screen
**AND** system shows progress bar with percentage (0-100%)
**AND** system displays estimated time remaining
**AND** system updates progress every 5% or 5 seconds (whichever is more frequent)

**Verify:** Start download, observe progress screen with "Awakening Your Shadow" branding, verify progress updates incrementally.

### AC-004: Background Download with App State Handling
**GIVEN** model download is in progress at 45%
**WHEN** user backgrounds app or kills app process
**THEN** download continues in background if supported by OS
**AND** system saves current download state persistently
**AND** system resumes download automatically when app relaunches
**AND** system displays "Resuming download..." message if applicable

**Verify:** Start download, background app at 45%, kill process, relaunch app, verify download resumes from saved state.

### AC-005: Download Completion and App Unlock
**GIVEN** model download completes successfully
**WHEN** download reaches 100%
**THEN** system displays "Your Shadow is Ready" confirmation screen
**AND** user sees "Start Riding" button to enter main app
**AND** system permanently dismisses setup wizard for this user
**AND** main app functionality becomes accessible

**Verify:** Complete download, observe confirmation screen, tap "Start Riding", verify main app loads and setup wizard never appears again.

### AC-006: Download Failure Handling
**GIVEN** model download fails due to network error or server issue
**WHEN** download failure occurs
**THEN** system displays user-friendly error message with copper accent
**AND** system provides "Retry Download" button
**AND** system offers "Contact Support" link if multiple retries fail
**AND** system logs technical error details for debugging

**Verify:** Simulate network failure during download, verify error message appears with retry option, verify retry resumes from last successful state.
