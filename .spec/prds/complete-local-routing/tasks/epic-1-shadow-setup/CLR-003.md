# CLR-003: Gatekeeper Implementation

**Task ID:** CLR-003
**Epic:** Epic 1 - Shadow Setup
**Assigned To:** react-native-ui-implementer
**Estimate:** 480 min
**Type:** [FEATURE]

---

## CRITICAL CONSTRAINTS

1. **MUST** block ALL app functionality until local AI model download is verified as complete - this is non-negotiable
2. **NEVER** allow navigation to any screen other than setup wizard when model is missing - no bypass, no exceptions
3. **STRICTLY** validate model integrity on every app launch - checksum verification required
4. **MUST** show "Setup Required" screen if model is corrupted or missing after initial setup
5. **NEVER** cache model download state permanently - always verify actual model file existence on app launch

---

## SPECIFICATION

**Objective:** Implement gatekeeper system that prevents app usage until local AI model is downloaded, validated, and verified as complete.

**Success State:** App launches into setup wizard if model is missing, shows "Setup Required" screen if model is corrupted, and unlocks main app only when model file exists and passes checksum validation. No user can bypass the gatekeeper to access app functionality without a valid model.

---

## ACCEPTANCE CRITERIA

### AC-001: App Launch Gatekeeper Check
**GIVEN** user launches LaneShadow app
**WHEN** app initializes
**THEN** system immediately checks for local AI model file existence
**AND** if model is missing, system routes to setup wizard
**AND** if model exists, system validates file integrity
**AND** only if model is valid, system loads main app navigation

**Verify:** Delete model file, launch app, verify setup wizard appears. Restore model file, launch app, verify main app loads.

### AC-002: Model Checksum Validation
**GIVEN** model file exists on device storage
**WHEN** app performs integrity check on launch
**THEN** system calculates SHA-256 checksum of model file
**AND** system compares checksum against expected value from Convex
**AND** if checksums match, model is marked as valid
**AND** if checksums differ, model is marked as corrupted

**Verify:** Corrupt model file by modifying bytes, launch app, verify "Setup Required" screen appears with re-download option.

### AC-003: Corrupted Model Recovery Flow
**GIVEN** model file fails checksum validation
**WHEN** app detects corrupted model
**THEN** system displays "Setup Required" screen with copper accent
**AND** system explains "Your Shadow needs to be restored. Please download again."
**AND** system provides "Restore Your Shadow" button
**AND** system deletes corrupted model file before re-downloading

**Verify:** Corrupt model file, launch app, observe "Setup Required" screen, tap restore, verify corrupted file deleted and download starts.

### AC-004: Navigation Route Guarding
**GIVEN** user has not completed model download
**WHEN** user attempts to navigate to any app screen (deep link, direct URL manipulation, etc.)
**THEN** system redirects to setup wizard
**AND** system shows toast notification "Complete setup to continue"
**AND** system logs navigation attempt for debugging

**Verify:** Attempt to deep-link into app without model, verify redirect to setup wizard with toast message.

### AC-005: Main App Unlock After Validation
**GIVEN** model download completes and passes checksum validation
**WHEN** user taps "Start Riding" on completion screen
**THEN** system sets modelSetupComplete flag in persistent storage
**AND** system routes to main app navigation
**AND** system enables all app features and screens
**AND** system never shows setup wizard again (unless model corrupted)

**Verify:** Complete setup, tap "Start Riding", verify main app loads with all features accessible. Relaunch app, verify setup wizard skipped.

### AC-006: Setup State Persistence
**GIVEN** user completes model download setup
**WHEN** app is killed and relaunched
**THEN** system checks model file existence (not just cached flag)
**AND** if model file exists and valid, system loads main app
**AND** if model file missing, system routes to setup wizard
**AND** cached flag alone is insufficient for app unlock

**Verify:** Complete setup, note cached flag is true. Delete model file but keep cached flag, relaunch app, verify setup wizard appears (flag alone insufficient).

## DESIGN

**References:**

- CLR-001: Setup Wizard Flow Implementation (provides UI screens)
- CLR-002: Local Model Integration (provides model file location)
- CLR-004: Model Download Persistence (provides state management)

**Patterns:**
- Root-level navigation guard in `_layout.tsx`
- AsyncStorage for setup completion flag (backup only)
- File system check for model existence (primary validation)
- SHA-256 checksum for integrity validation
- Redirect pattern with toast notifications

**Anti-Patterns:**
- **DO NOT** rely solely on cached flags - always verify file existence
- **DO NOT** allow any bypass of gatekeeper - no "skip for now" options
- **DO NOT** show technical checksum errors to users
- **DO NOT** block app launch for validation - validate asynchronously

**Gatekeeper Flow:**
```
App Launch → Check Model File Exists
  ├─ Missing → Setup Wizard
  ├─ Exists → Validate Checksum
      ├─ Valid → Main App
      └─ Invalid → "Setup Required" → Re-download
```
