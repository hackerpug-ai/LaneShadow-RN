## CRITICAL CONSTRAINTS

1. **MUST** Use MLX framework for local model runtime on iOS/macOS — no other model runways supported
2. **NEVER** download model on cellular connection — WiFi hard requirement enforced at system level
3. **STRICTLY** follow singleton pattern for in-memory model caching — only one Qwen3.5 instance per app lifecycle
4. **MUST** implement checksum validation (SHA-256) before model loading — corrupted files MUST trigger re-download
5. **NEVER** block main thread during model initialization — all loading operations MUST run on background queue

---

## SPECIFICATION

Integrate Qwen3.5 0.8B local model runtime into LaneShadow app for on-device leg label generation, enabling instant AI inference without network dependency while maintaining <1.5GB memory footprint and <0.5s inference time. Success state: Model downloads successfully on first launch, loads into memory on-demand, and generates "FROM → TO" labels locally.

---

## ACCEPTANCE CRITERIA

### AC-001: Happy Path — Successful Model Download and Loading
**GIVEN** user is on WiFi connection and app launches for first time
**WHEN** model download completes and file passes checksum validation
**THEN** model loads into memory successfully AND subsequent inference requests complete in <0.5s AND memory usage remains <1.5GB

### AC-002: Edge Case — Download Interruption and Resume
**GIVEN** model download is in progress (45% complete)
**WHEN** app is terminated by user or system
**THEN** download state is persisted to local storage AND download resumes from 45% on next app launch WITHOUT re-downloading existing bytes

### AC-003: Edge Case — Corrupted Model File Detection
**GIVEN** model file exists on device but has corrupted bytes
**WHEN** app attempts to load model for inference
**THEN** checksum validation fails AND corrupted file is deleted AND fresh download is initiated automatically

### AC-004: Error Case — Insufficient Storage Space
**GIVEN** device has <2GB free storage space
**WHEN** user initiates model download
**THEN** download is blocked BEFORE any bytes are written AND user sees error message: "Not enough storage. Need 2GB free space. Available: X GB" AND app remains functional (setup wizard doesn't crash)

## DESIGN

### Model Architecture
```
lib/ai/
├── local-model.ts          # MLX runtime wrapper (singleton)
├── model-download.ts       # Download manager with progress tracking
├── checksum.ts             # SHA-256 validation
└── types.ts                # Model metadata types

components/setup/
├── ModelDownloadScreen.tsx # Progress UI with WiFi gate
└── SetupWizard.tsx         # Onboarding flow orchestrator
```

### Data Flow
```
App Launch → Check Model Exists → [No: Download] → Validate Checksum → Load into Memory
                                                              ↓
                                                         [Yes: Load from Cache]
                                                              ↓
                                                    Ready for Inference (0.35s)
```

### Error Handling
- **Download Failure:** Retry with exponential backoff (max 3 attempts)
- **Checksum Mismatch:** Delete file, re-download from source
- **Out of Memory:** Show error, abort setup, prevent app access
- **Network Switch:** Pause download if WiFi drops, resume when restored

### Integration Points
- **Convex:** Store model metadata (version, download URL, checksum)
- **Setup Wizard:** Block app access until model ready
- **Leg Label Generation:** Use local model for instant "FROM → TO" labels
