**Anti-Patterns:**
- **DO NOT** skip weather data preservation (will break overlays)
- **DO NOT** ignore edge cases (will cause corruption)
- **DO NOT** modify encoding without tests (will introduce bugs)
- **DO NOT** break existing routes during migration
- **DO NOT** assume coordinate order is correct

**Conversion Utilities:**
```typescript
// Google: [lat, lng], Mapbox: [lng, lat]

export const convertGoogleToMapbox = (polyline: string): string => {
  const coords = decodeGoogle(polyline)  // [lat, lng]
  const mapboxCoords = coords.map(([lat, lng]) => [lng, lat])
  return encodeMapbox(mapboxCoords)
}

export const convertMapboxToGoogle = (polyline: string): string => {
  const coords = decodeMapbox(polyline)  // [lng, lat]
  const googleCoords = coords.map(([lng, lat]) => [lat, lng])
  return encodeGoogle(googleCoords)
}

export const convertWeatherSegments = (
  segments: WeatherSegment[],
  converter: (coords: [number, number]) => [number, number]
): WeatherSegment[] => {
  return segments.map(segment => ({
    ...segment,
    coordinates: segment.coordinates.map(converter)
  }))
}
```

**Edge Cases:**
- **Poles:** latitude = ±90°, longitude undefined
- **Date Line:** longitude crosses ±180°
- **Invalid ranges:** latitude > 90° or < -90°, longitude > 180° or < -180°
