## CRITICAL CONSTRAINTS

1. **MUST** Use schema.define() for Replicate collection schemas — no v.object() direct definitions
2. **NEVER** embed large objects (route geometry) directly in schema — use file: references for blobs
3. **STRICTLY** include version field in schema — enables schema migrations without data loss
4. **MUST** define all indexes used by queries — missing indexes MUST fail fast at build time
5. **NEVER** use optional fields for critical data (userId, geometry) — use required fields with defaults

---

## SPECIFICATION

Define provider-agnostic route schema using Replicate's schema.define() API, supporting versioned schema migrations, automatic CRDT merge behavior, and offline-first route storage with geometry, waypoints, and metadata. Success state: Schema compiles, validates, and supports route CRUD operations with automatic sync.

### AC-001: Happy Path — Schema Compilation and Validation

### AC-002: Edge Case — Schema Version Migration
**GIVEN** existing routes with schema version 1.0
**WHEN** developer updates schema to version 1.1 with new field (e.g., elevationGain)
**THEN** existing routes automatically get default value for new field AND new routes include elevationGain AND no data is lost

### AC-003: Edge Case — Large Geometry Storage
**GIVEN** route with 500KB polyline geometry
**WHEN** developer attempts to store geometry directly in route document
**THEN** schema validation rejects large object AND error message guides to use file: reference AND geometry stored separately

### AC-004: Error Case — Missing Index Definition
**GIVEN** query filters by userId field
**WHEN** schema lacks index on userId
**THEN** Convex dev server fails fast with error: "Missing index on userId for query: routesByUser" AND index must be added before query runs

## DESIGN

### Schema Definition (convex/schema.ts)
```typescript
import { defineSchema, v } from "convex/values";
import { schema } from "@trestleinc/replicate";

export default defineSchema({
  routes: schema.define({
    version: v.optional(v.number()), // Schema version
    userId: v.string(),
    name: v.string(),
    geometry: v.string(), // Encoded polyline (Mapbox format)
    bounds: v.object({
      northeast: v.object({ lat: v.number(), lng: v.number() }),
      southwest: v.object({ lat: v.number(), lng: v.number() }),
    }),
    waypoints: v.array(v.object({
      lat: v.number(),
      lng: v.number(),
      name: v.optional(v.string()),
    })),
    metadata: v.optional(v.any()), // Weather overlays, etc.
    createdAt: v.number(),
    updatedAt: v.number(),
  }).index("by_user", ["userId"]),
});
```

### Geometry Storage Pattern
```typescript
// For routes >100KB, use file: reference
interface RouteSchema {
  // ... other fields
  geometry: v.string(), // For <100KB polylines
  geometryFile: v.optional(v.id("_storage")), // For >100KB polylines
}
```

### Version Migration
```typescript
// v1.0 → v1.1 migration example
async function migrateRoute_v1_to_v1_1(route: RouteV1): RouteV1_1 {
  return {
    ...route,
    version: 1.1,
    elevationGain: 0, // Default for existing routes
  };
}
```
