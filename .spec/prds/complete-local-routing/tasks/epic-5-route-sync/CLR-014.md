## CRITICAL CONSTRAINTS

1. **MUST** Use useReplicate() hook for route collection — no manual sync logic
2. **NEVER** call Convex mutations directly for synced data — all writes go through Replicate collection
3. **STRICTLY** handle merge conflicts via Yjs CRDT rules — last-write-wins for primitive fields
4. **MUST** implement optimistic UI updates — local changes reflect immediately without network round-trip
5. **NEVER** block UI on sync failures — all sync errors MUST be non-blocking and recoverable

---

## SPECIFICATION

Implement local-first route sync collection using @trestleinc/replicate, enabling instant offline route editing (add waypoint, rename, reorder) with automatic bidirectional sync and conflict resolution when online. Success state: User edits route offline, changes persist locally, sync automatically when online, and concurrent edits merge without data loss.

---

## ACCEPTANCE CRITERIA

### AC-001: Happy Path — Offline Route Editing Syncs Automatically
**GIVEN** user is offline (airplane mode) and has existing route "Coastal Highway"
**WHEN** user adds waypoint "Pacifica" and removes waypoint "Half Moon Bay"
**THEN** changes save immediately to local storage AND UI updates optimistically AND when user goes online, changes sync to server automatically

### AC-002: Edge Case — Concurrent Edit Merge (No Conflict)
**GIVEN** user edits route name to "Coastal Run" on phone (offline)
**WHEN** user simultaneously edits route description on tablet (offline)
**THEN** both devices go online AND both changes sync AND route has new name AND route has new description (no conflict, different fields)

### AC-003: Edge Case — Concurrent Edit Merge (Same Field)
**GIVEN** user renames route to "Coastal Run" on phone (offline)
**WHEN** user simultaneously renames same route to "Highway 1" on tablet (offline)
**THEN** both devices go online AND Yjs CRDT merges with last-write-wins AND both devices show "Highway 1" (tablet's change won)

### AC-004: Error Case — Sync Failure Recovery
**GIVEN** user has 5 pending route edits offline
**WHEN** user goes online but server returns 500 error
**THEN** edits remain in local storage AND sync retries automatically with exponential backoff AND user can continue editing offline

## DESIGN

### Collection Hook (collections/use-routes.ts)
```typescript
import { useReplicate } from "@trestleinc/replicate/convex";
import { api } from "../convex/_generated";

export function useRoutes() {
  const { data: routes, insert, update, remove } = useReplicate(
    api.routes.default,
    "routes" // Collection name
  );

  return { routes, insert, update, remove };
}
```

### Optimistic Update Pattern
```typescript
// lib/sync/optimistic-update.ts
async function updateRouteOptimistically(
  routeId: string,
  updates: Partial<Route>
): Promise<void> {
  // 1. Update local state immediately
  queryClient.setQueryData(
    ["routes", routeId],
    (old) => ({ ...old, ...updates })
  );

  // 2. Sync to server in background
  await update(routeId, updates);
}
```

### Sync Status Indicator
```typescript
// components/sync/SyncStatusIndicator.tsx
export function SyncStatusIndicator() {
  const { syncStatus, pendingEdits } = useSyncStatus();

  return (
    <View>
      {syncStatus === "syncing" && <Text>Syncing...</Text>}
      {syncStatus === "offline" && <Text>Offline - {pendingEdits} edits pending</Text>}
      {syncStatus === "online" && <Text>Synced</Text>}
    </View>
  );
}
```
