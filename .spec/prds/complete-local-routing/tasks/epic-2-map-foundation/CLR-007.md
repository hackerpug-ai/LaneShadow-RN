## DESIGN

**References:**
- @rnmapbox/maps Documentation: https://github.com/rnmapbox/maps
- PRD Phase 1: Foundation - "Create MapboxMapView Wrapper" section
- LaneShadow Theme System: `styles/RULES.md`
- Coordinate Systems: Google [lat, lng] vs Mapbox [lng, lat]

**Patterns:**
- React component wrapper with theme prop
- Coordinate conversion utilities (separate module)
- Camera control utilities (separate module)
- TypeScript type definitions for all props
- Unit tests for conversion logic

**Anti-Patterns:**
- **DO NOT** hardcode style URLs in component
- **DO NOT** skip coordinate conversion (will cause bugs)
- **DO NOT** inline complex conversion logic
- **DO NOT** ignore TypeScript type safety
- **DO NOT** sacrifice performance for convenience

**Component Interface:**
```typescript
interface MapboxMapViewProps {
  theme: 'dark' | 'light'
  camera?: {
    center: [number, number]  // [lng, lat] for Mapbox
    zoom: number
    pitch?: number
    heading?: number
  }
  markers?: Marker[]
  polylines?: Polyline[]
  onCameraChange?: (camera: Camera) => void
  onPress?: (feature: GeoJSON.Feature) => void
}
```

**Coordinate Conversion:**
```typescript
// Google: [lat, lng]
// Mapbox: [lng, lat]

export const googleToMapbox = (coords: [number, number]): [number, number] => {
  const [lat, lng] = coords
  return [lng, lat]
}

export const mapboxToGoogle = (coords: [number, number]): [number, number] => {
  const [lng, lat] = coords
  return [lat, lng]
}
```
