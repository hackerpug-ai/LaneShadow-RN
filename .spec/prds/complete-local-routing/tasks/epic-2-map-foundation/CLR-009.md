BACKGROUND
--------------------------------------------------------------------------------

**Problem:** WaypointMarker and SearchResultMarker import `Marker` from `react-native-maps` directly. They cannot render as children of MapboxMapView.

**Why it matters:** Without Mapbox-native markers, waypoints and search results won't appear on the map after cutover.

**Current state:** Both components use `react-native-maps` `<Marker>` with custom SVG views as children. WaypointMarker renders pin-shaped markers with status color coding. SearchResultMarker renders numbered circle markers with selection state.

**Desired state:** Both components use `@rnmapbox/maps` `<MarkerView>` with the same SVG visual designs and coordinate conversion. Same tap behavior, same semantic color system.

NOTES
--------------------------------------------------------------------------------

- MarkerView supports custom children (View, SVG) unlike Mapbox's native PointAnnotation — use MarkerView.
- MarkerView.onPress is supported — can replace react-native-maps Marker.onPress directly.
- SVG components (react-native-svg) are framework-agnostic — no changes needed to SVG paths.
- The `tracksViewChanges={false}` optimization on SearchResultMarker is a react-native-maps concept — Mapbox handles this differently and it may not be needed.
