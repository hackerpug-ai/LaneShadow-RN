## DESIGN

**References:**
- Expo Plugin Docs: https://docs.expo.dev/guides/rnmapbox/
- Mapbox Style URLs: https://docs.mapbox.com/android/maps/guides/styles/
- PRD Phase 1: Foundation - "Configure Expo Plugin" section
- LaneShadow Theme System: `styles/RULES.md`

**Patterns:**
- Environment variable loading via process.env
- Expo plugin configuration array in app.config.js
- Theme-aware style URL selection
- Platform-specific config objects (ios, android)

**Anti-Patterns:**
- **DO NOT** hardcode access tokens in app.config.js
- **DO NOT** use public Mapbox tokens in production
- **DO NOT** skip iOS deployment target update
- **DO NOT** forget Android permissions (INTERNET, ACCESS_FINE_LOCATION)
- **DO NOT** use unstable map style URLs

**Configuration Example:**
```javascript
// app.config.js
export default {
  expo: {
    plugins: [
      [
        "@rnmapbox/maps",
        {
          RNMapboxMapsImpl: "mapbox",
          accessToken: process.env.MAPBOX_ACCESS_TOKEN
        }
      ]
    ]
  }
}
```

## NOTES

**Token Setup:**
1. Sign up for Mapbox account
2. Create access token at https://account.mapbox.com/
3. Required scopes: downloads:read, styles:read, fonts:read
4. Add token to Convex environment variables
5. Reference in app.config.js via process.env

**Custom Map Styles:**
- Default styles: mapbox://styles/mapbox/dark-v11, mapbox://styles/mapbox/light-v11
- Custom styles can be created in Mapbox Studio
- Custom style URL format: mapbox://styles/{username}/{style-id}
- Consider creating custom LaneShadow dark style with copper accents
