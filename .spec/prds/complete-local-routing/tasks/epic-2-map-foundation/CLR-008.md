BACKGROUND
--------------------------------------------------------------------------------

**Problem:** RoutePolyline and DeviationPolyline import `Polyline` from `react-native-maps` directly. They cannot render as children of the new MapboxMapView.

**Why it matters:** Without Mapbox-native polylines, the home screen cutover is blocked — the map would render but no route geometry would appear.

**Current state:** Both components use `react-native-maps` `<Polyline>` with `[lat, lng]` coordinate arrays. RoutePolyline supports tap-to-select segments with haptic feedback and semantic colors. DeviationPolyline renders colored segments for detour visualization.

**Desired state:** Both components use `@rnmapbox/maps` `<ShapeSource>` + `<LineLayer>` primitives with coordinate conversion via `googleToMapbox`/`convertCoordinateArray`. Same visual output, same interaction behavior.

CODE PATTERN (Reference)
--------------------------------------------------------------------------------

ShapeSource + LineLayer pattern from mapbox-map-view.tsx:
```typescript
import { ShapeSource, LineLayer } from '@rnmapbox/maps'
import type { FeatureCollection, LineString, Position } from 'geojson'
import { convertCoordinateArray } from '../../lib/mapbox/coordinate-converter'

// Build GeoJSON FeatureCollection
const feature: FeatureCollection<LineString> = {
  type: 'FeatureCollection',
  features: [{
    type: 'Feature',
    properties: {},
    geometry: {
      type: 'LineString',
      coordinates: convertCoordinateArray(googleCoords),
    },
  }],
}

<ShapeSource id={polylineId} shape={feature}>
  <LineLayer
    id={`${polylineId}-layer`}
    style={{
      lineColor: strokeColor,
      lineWidth: strokeWidth,
      lineOpacity: 1.0,
      lineCap: 'round',
      lineJoin: 'round',
    }}
  />
</ShapeSource>
```
