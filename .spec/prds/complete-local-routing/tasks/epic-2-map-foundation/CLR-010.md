BACKGROUND
--------------------------------------------------------------------------------

**Problem:** RouteMiniMap uses `react-native-maps` MapView with PROVIDER_GOOGLE for non-interactive route preview thumbnails in chat cards. It won't render correctly alongside MapboxMapView in the same app after cutover.

**Why it matters:** Mixed map providers in one app can cause native conflicts (both initialize their own map engine). Also, the Google Maps style won't match the Mapbox theme on the main screen.

**Current state:** RouteMiniMap creates a standalone Google Maps MapView with disabled interactions, decodes polyline geometry, calculates region from bounds, renders a single Polyline with primary color. Used in `components/chat/cards/route-mini-map.tsx`.

**Desired state:** RouteMiniMap uses `@rnmapbox/maps` MapView with the same theme-aware style URLs, renders polyline via ShapeSource + LineLayer, and fits camera to bounds using Mapbox Camera.

```typescript
import Mapbox from '@rnmapbox/maps'
import { MAP_STYLES } from '../../../lib/mapbox/styles'
import { convertCoordinateArray } from '../../../lib/mapbox/coordinate-converter'

const { MapView: MapboxMap, Camera, ShapeSource, LineLayer } = Mapbox

// Calculate bounds center for Camera
const center: [number, number] = [
  (bounds.southwest.lng + bounds.northeast.lng) / 2,
  (bounds.southwest.lat + bounds.northeast.lat) / 2,
]
const zoom = Math.log2(360 / Math.abs(bounds.northeast.lat - bounds.southwest.lat)) - 0.5

<MapboxMap
  style={styles.map}
  styleURL={MAP_STYLES[dark ? 'dark' : 'light']}
  scrollEnabled={false}
  zoomEnabled={false}
  rotateEnabled={false}
  pitchEnabled={false}
>
  <Camera centerCoordinate={center} zoomLevel={zoom} />
  <ShapeSource shape={feature}>
    <LineLayer style={{ lineColor: primaryColor, lineWidth: 3 }} />
  </ShapeSource>
</MapboxMap>
```
