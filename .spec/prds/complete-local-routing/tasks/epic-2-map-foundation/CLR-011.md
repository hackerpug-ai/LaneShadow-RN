BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The home screen (`app/(app)/(tabs)/index.tsx`) and saved route detail screen (`app/(app)/saved-route/[id].tsx`) still import and render `MapViewWrapper` (Google Maps). The `MapboxMapView` created in CLR-007 exists but is not consumed by any screen.

**Why it matters:** This is the actual cutover — the point where the user sees Mapbox instead of Google Maps. Without it, all prior Epic 2 work is unused infrastructure. P0 because it unblocks all downstream epics (offline regions, local routing).

**Current state:** Home screen uses `MapViewWrapper` ref (`MapViewHandle`) for camera controls (setCameraPosition, zoomBy, recenterToUser, animateToRegion, fitToCoordinates). Child components (RoutePolyline, SearchResultMarker) render as direct children of MapView. Saved route screen uses `MapViewWrapper` with `buildRoutePolylines`.

**Desired state:** Both screens use `MapboxMapView` with `MapboxMapViewHandle`. Child components (migrated in CLR-008/009) render as children of Mapbox MapView. Camera controls map to Mapbox equivalents. User location tracking works. Theme switching applies Mapbox style URLs.

CODE PATTERN (Reference)
--------------------------------------------------------------------------------

Import swap pattern:
```typescript
// BEFORE (Google Maps)
import { MapViewWrapper, type MapViewHandle } from '../../../components/map/map-view'

// AFTER (Mapbox)
import {
  MapboxMapView,
  type MapboxMapViewHandle,
} from '../../../components/map'
```

Handle adaptation — MapboxMapViewHandle must support the same call signatures:
```typescript
// Google Maps handle
setCameraPosition({ coordinates, zoom, duration })
zoomBy(delta)
recenterToUser()
animateToRegion(region, duration)
fitToCoordinates(coordinates, options)

// Mapbox handle (may need extension)
setCamera(camera, duration)  // rename to setCameraPosition
zoomIn(delta) / zoomOut(delta)  // map zoomBy to these
// recenterToUser needs user location tracking
fitToCoordinates(coordinates, padding)  // similar but different padding format
```
