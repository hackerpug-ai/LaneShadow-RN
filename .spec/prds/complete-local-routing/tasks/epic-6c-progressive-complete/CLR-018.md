# CLR-018: Enrichment Status Hooks

**Task ID:** CLR-018
**Epic:** Epic 6c - Progressive Complete
**Assigned To:** react-native-ui-implementer
**Estimate:** 480 min
**Type:** [FEATURE]

---

## CRITICAL CONSTRAINTS

1. **MUST** track enrichment status reactively - UI updates automatically as enrichment progresses
2. **STRICTLY** handle rapid status changes without performance degradation - enrichment can complete quickly
3. **MUST** clean up subscriptions properly - prevent memory leaks in React components
4. **NEVER** block UI on enrichment updates - all status tracking must be asynchronous
5. **MUST** work offline for local enrichment status - cloud enrichment may be unavailable

---

## SPECIFICATION

**Objective:** Implement React hooks for enrichment status tracking that provide reactive updates, progress reporting, toast notifications, and error handling.

**Success State:** useEnrichmentStatus hook tracks enrichment state reactively, updates UI automatically as enrichment progresses, triggers toast notifications on completion, handles errors gracefully, and works offline for local enrichment.

## ACCEPTANCE CRITERIA

### AC-001: Enrichment Status Tracking
**GIVEN** route enrichment is in progress
**WHEN** component uses useEnrichmentStatus hook
**THEN** hook returns current enrichment status (draft/partial/complete)
**AND** hook updates status reactively as enrichment progresses
**AND** hook provides progress percentage (0-100)
**AND** hook does not cause unnecessary re-renders

**Verify:** Use hook in component, observe status changes from draft → partial → complete, verify reactive updates.

### AC-002: Progress Percentage Reporting
**GIVEN** enrichment is in progress
**WHEN** enrichment status updates
**THEN** hook reports progress percentage accurately
**AND** hook updates progress incrementally during enrichment
**AND** hook provides estimated time remaining
**AND** hook handles progress jumps gracefully

**Verify:** Monitor progress during enrichment, verify percentage increases smoothly, check ETA calculation.

### AC-003: Toast Notification Triggering
**GIVEN** enrichment completes successfully
**WHEN** status changes to "complete"
**THEN** hook triggers toast notification automatically
**AND** toast displays "Route enhancement complete" message
**AND** toast dismisses automatically after 3 seconds
**AND** toast does not appear for partial status

**Verify:** Complete enrichment, observe toast notification, verify auto-dismiss after 3 seconds.

### AC-004: Error State Handling
**GIVEN** enrichment fails during processing
**WHEN** error occurs
**THEN** hook updates status to "error"
**AND** hook provides error message for display
**AND** hook offers retry option to user
**AND** hook logs error for debugging

**Verify:** Simulate enrichment failure, verify error status appears, check retry functionality.

### AC-005: Subscription Cleanup
**GIVEN** component using hook is unmounted
**WHEN** component unmounts
**THEN** hook cleans up enrichment subscriptions
**AND** hook prevents memory leaks
**AND** hook stops status updates after unmount
**AND** hook does not trigger toasts after unmount

**Verify:** Mount component with hook, unmount quickly, verify no memory leaks or post-unmount updates.

### AC-006: Offline Support
**GIVEN** device is offline
**WHEN** local enrichment completes
**THEN** hook tracks local enrichment status
**AND** hook shows "partial" status (local complete, cloud pending)
**AND** hook does not show errors for missing cloud enrichment
**AND** hook syncs to cloud when online

**Verify:** Enable airplane mode, create route, observe "partial" status, disable airplane mode, observe sync to "complete".

## DESIGN

**References:**
- React Hooks Documentation: https://react.dev/reference/react
- PRD Phase 3: Hybrid Enrichment - "Progressive Enhancement UI" section
- CLR-016 (Dual-Model Orchestration) - Provides enrichment state machine

**Patterns:**
- Custom React hook pattern
- Subscription/unsubscription pattern
- Event-driven status updates
- Toast notification pattern
- Cleanup on unmount

**Anti-Patterns:**
- **DO NOT** block UI on enrichment updates
- **DO NOT** cause unnecessary re-renders
- **DO NOT** leak subscriptions on unmount
- **DO NOT** show toasts for partial status
- **DO NOT** ignore offline status

**Hook Interface:**
```typescript
interface UseEnrichmentStatusResult {
  status: 'draft' | 'partial' | 'complete' | 'error'
  progress: number  // 0-100
  estimatedTimeRemaining: number  // seconds
  error?: Error
  retry: () => void
}

const useEnrichmentStatus = (routeId: string): UseEnrichmentStatusResult
```

**Status Flow:**
```
draft (route created)
  ↓
partial (local enrichment complete)
  ↓
complete (cloud enrichment complete)
  ↓
error (if enrichment fails)
```

- **draft**: Route created, no enrichment
- **partial**: Local enrichment complete (leg labels)
- **complete**: Cloud enrichment complete (creative label, rationale, highlights)

- Local enrichment: 0-50% (target: 0.35s)

- Auto-dismiss after 3 seconds
- Show user-friendly message
- Support action buttons (e.g., "View Route")

- Show error state in UI
- Provide retry option
