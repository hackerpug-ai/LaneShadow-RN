## Task Description

Design and implement progressive enhancement UI that shows route enrichment completing in real-time. Creative labels, rationales, and highlights fade in incrementally as they arrive from the cloud enrichment service. Status badges update from "partial" to "complete" and toast notifications provide feedback.

---

## Human Test Deliverable

**Test Scenario:** User creates route and watches enhancement complete progressively

**Steps:**
1. Create route "SF to Santa Cruz"
2. See skeleton states (Epic 6a complete)
3. See leg labels appear immediately (0.35s)
4. See "partial" badge with copper color
5. See "Enhancing..." toast appear
6. Wait 3.9s for cloud enrichment
7. See creative label "Coastal Curve Chronicles" fade in
8. See rationale text appear below label
9. See highlight tags fade in staggered (100ms delay)
10. See badge change to "complete" with green color
11. See toast update to "Enhancement complete"
12. See toast auto-dismiss after 3s

**Gate:** Full enrichment < 5s, Animations smooth (60fps)

## Design Requirements

### Theme Compliance
- **Dark-first design:** Design in dark mode first, adapt to light
- **Copper accents:** Use for interactive elements and progress
- **Glassmorphic overlays:** For toast and floating elements
- **Enrichment colors:** Teal (partial), Purple (complete)

See: `styles/RULES.md:line:17`

### Animation Principles
- **Smooth performance:** Target 60fps, use native driver
- **Perceived speed:** Staggered reveals make enrichment feel faster
- **Non-blocking:** Toast doesn't block map interaction
- **Auto-dismiss:** Don't clutter UI with stale toasts

### Fade-In Timing
- **Duration:** 300ms (React Native Reanimated default)
- **Easing:** ease-in-out
- **Stagger:** 100ms delay between elements

### Progress Feedback
- **Status badge:** Always visible, updates immediately
- **Toast:** Shows current stage, progress bar animates
- **Completion:** Badge changes + toast updates + auto-dismiss

### ProgressiveEnhancementToast

```typescript
// components/enrichment/progressive-enhancement-toast.tsx
import { useSemanticTheme } from '@/hooks/use-semantic-theme'
import Animated, { FadeInDown, FadeOutUp } from 'react-native-reanimated'

export const ProgressiveEnhancementToast: React.FC<{
  visible: boolean
  stage: string
  progress: number
  onDismiss: () => void
}> = ({ visible, stage, progress, onDismiss }) => {
  const { semantic } = useSemanticTheme()

  if (!visible) return null

  return (
    <Animated.View
      entering={FadeInDown.duration(300)}
      exiting={FadeOutUp.duration(300)}
      style={[
        styles.toast,
        {
          backgroundColor: `${semantic.color.card.default}CC`, // 80% opacity
          borderColor: `${semantic.color.primary.default}4D`, // 30% opacity
        },
      ]}
    >
      <View style={styles.header}>
        <Text style={[styles.stage, { color: semantic.color.onSurface.default }]}>
          {stage}
        </Text>
        <TouchableOpacity onPress={onDismiss}>
          <Icon name="close" size={20} color={semantic.color.onSurface.muted} />
        </TouchableOpacity>
      </View>

      <View style={[styles.progressBar, { backgroundColor: semantic.color.surfaceVariant.default }]}>
        <Animated.View
          style={[
            styles.progressFill,
            {
              width: `${progress}%`,
              backgroundColor: semantic.color.primary.default,
            },
          ]}
        />
      </View>
    </Animated.View>
  )
}

const styles = StyleSheet.create({
  toast: {
    position: 'absolute',
    top: 80, // Below MapHeaderOverlay
    alignSelf: 'center',
    maxWidth: 400,
    padding: 16,
    borderRadius: 16,
    borderWidth: 1,
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 4 },
    shadowOpacity: 0.15,
    shadowRadius: 8,
    elevation: 3,
  },
  header: {
    flexDirection: 'row',
    justifyContent: 'space-between',
    alignItems: 'center',
    marginBottom: 12,
  },
  stage: {
    fontSize: 14,
    fontWeight: '500',
  },
  progressBar: {
    height: 4,
    borderRadius: 2,
    overflow: 'hidden',
  },
  progressFill: {
    height: '100%',
  },
})
```

### HighlightTagsStagger

```typescript
// components/enrichment/highlight-tags-stagger.tsx
import Animated, { FadeIn } from 'react-native-reanimated'
import { useSemanticTheme } from '@/hooks/use-semantic-theme'

export const HighlightTagsStagger: React.FC<{
  highlights: string[]
}> = ({ highlights }) => {
  const { semantic } = useSemanticTheme()

  return (
    <View style={styles.container}>
      {highlights.map((highlight, index) => (
        <Animated.View
          key={index}
          entering={FadeIn.delay(index * 100).duration(300)}
          style={[
            styles.tag,
            {
              backgroundColor: `${semantic.color.primary.default}1A`, // 10% opacity
              borderColor: `${semantic.color.primary.default}4D`, // 30% opacity
            },
          ]}
        >
          <Text style={[styles.tagText, { color: semantic.color.primary.default }]}>
            {highlight}
          </Text>
        </Animated.View>
      ))}
    </View>
  )
}

const styles = StyleSheet.create({
  container: {
    flexDirection: 'row',
    flexWrap: 'wrap',
    gap: 8,
  },
  tag: {
    paddingHorizontal: 12,
    paddingVertical: 6,
    borderRadius: 9999,
    borderWidth: 1,
  },
  tagText: {
    fontSize: 14,
    fontWeight: '500',
  },
})
```

## Constraints

### Must
- Follow all LaneShadow UI/UX patterns (styles/RULES.md)
- Work in dark and light themes
- Animate smoothly at 60fps (use native driver)
- Be non-blocking (toast doesn't block map)
- Auto-dismiss toasts after completion
- Support accessibility (reduce motion, screen readers)
- Use semantic tokens (no hardcoded colors)

### Must Not
- Use hardcoded colors or timing
- Block map interaction with toast
- Show stale toasts (> 3s after complete)
- Animate when user prefers reduced motion
- Cause layout shift during animations
