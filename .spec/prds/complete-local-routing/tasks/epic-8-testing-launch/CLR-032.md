## Task Description

Test UX risk areas for Progressive Enhancement (Epic 6) flow to ensure users perceive enrichment as smooth and valuable. Focus on skeleton state smoothness, fade-in animation timing, status badge clarity, toast notification timing, and partial vs complete distinction.

---

## Human Test Deliverable

**Test Scenario:** User creates route and watches enhancement complete progressively

**Steps:**
1. Open route planning screen
2. Create route "SF to Santa Cruz"
3. See route geometry appear immediately
4. See skeleton placeholder for creative label
5. See pulse animation for weather badges
6. Wait 0.35s for leg labels
7. See leg labels fade in smoothly
8. See "partial" badge (teal color)
9. See "Enhancing..." toast appear
10. Wait for creative label (3.9s avg)
11. See creative label "Coastal Curve Chronicles" fade in
12. See rationale text appear below
13. See highlight tags fade in staggered
14. See badge change to "complete" (purple)
15. See toast update to "Enhancement complete"
16. See toast auto-dismiss after 3s

**Gate:** Skeleton smooth (60fps), Fade-in timing feels right, Badge changes noticeable

## Specification

### Test Scenarios

#### Scenario 1: Standard Progressive Enhancement
**Flow:** User creates route with stable internet

**Steps:**
1. Create route "SF to Santa Cruz"
2. See skeleton states (label, weather, details)
3. See leg labels fade in (0.35s)
4. See badge = "partial" (teal)
5. See toast = "Enhancing..."
6. Wait 3.9s for cloud enrichment
7. See creative label fade in
8. See rationale appear
9. See highlights stagger in
10. See badge = "complete" (purple)
11. See toast = "Complete" (dismisses in 3s)

**Success Criteria:**
- Skeleton states smooth (60fps)
- Fade-in timing feels natural
- Badge change noticeable
- Toast timing appropriate
- User understands partial vs complete

#### Scenario 2: Rapid Route Creation
**Flow:** User creates multiple routes quickly

**Steps:**
1. Create route A
2. Immediately create route B
3. Immediately create route C
4. See all three routes with skeletons
5. See leg labels appear for all (staggered)
6. See enrichments complete independently
7. Verify no animation jank
8. Verify badge states correct per route

**Success Criteria:**
- No performance degradation
- All animations smooth (60fps)
- Badge states independent per route
- Toasts don't overlap

#### Scenario 3: Offline During Enrichment
**Flow:** User goes offline during cloud enrichment

**Steps:**
1. Create route with WiFi
2. See leg labels appear (0.35s)
3. See badge = "partial"
4. Turn off WiFi
5. See toast = "Offline - will enrich when connected"
6. See badge stays "partial"
7. Turn on WiFi after 10s
8. See enrichment resume
9. See badge change to "complete"

**Success Criteria:**
- Clear offline messaging
- Badge stays "partial" correctly
- Enrichment resumes smoothly
- User understands offline state

#### Scenario 4: Slow Network
**Flow:** User on slow 3G connection

**Steps:**
1. Simulate 3G network (500ms latency, 1Mbps)
2. Create route
3. See leg labels appear quickly (0.35s)
4. See badge = "partial"
5. Wait longer for cloud enrichment (10-15s)
6. See creative label fade in eventually
7. Verify no timeout errors
8. Verify user didn't abort

**Success Criteria:**
- Leg labels still fast (< 0.5s)
- Cloud enrichment completes (slower but works)
- User doesn't perceive as "broken"
- No timeout errors

#### Scenario 5: Enrichment Failure
**Flow:** Cloud enrichment fails

**Steps:**
1. Create route
2. See leg labels appear (0.35s)
3. Simulate cloud enrichment failure
4. See badge = "partial" (stays)
5. See toast = "Enrichment failed - showing basic info"
6. See partial enrichment (leg labels only)
7. Verify route still usable
8. Verify user can still navigate

**Success Criteria:**
- Graceful degradation to partial
- Clear failure messaging
- Route remains functional
- User doesn't perceive as "broken"

#### Scenario 6: Reduce Motion Preference
**Flow:** User has reduce motion enabled

**Steps:**
1. Enable "Reduce Motion" in system settings
2. Create route
3. See leg labels appear instantly (no fade-in)
4. See creative label appear instantly
5. See highlights appear instantly (no stagger)
6. Verify no animations
7. Verify all content still appears

**Success Criteria:**
- All animations disabled
- Content still appears
- No timing issues
- Accessibility respected

### UX Metrics to Measure

#### Skeleton Smoothness
**Metric:** Frame rate during skeleton rendering
**Target:** ≥ 60fps
**Measurement:** Performance profiling

**Test:**
- Record skeleton rendering
- Measure frame rate
- Check for dropped frames
- Target: 0 dropped frames

#### Fade-in Timing Perception
**Metric:** User perception of fade-in duration
**Target:** "Feels right" (≥ 80%)
**Measurement:** Post-test survey

**Survey Question:**
"How did the timing of the label appearing feel?"
- Too slow (1)
- Slow (2)
- Just right (3) ✓
- Fast (4)
- Too fast (5)

**Target:** ≥ 80% select "Just right"

#### Badge Change Noticeability
**Metric:** User notices badge change from partial to complete
**Target:** ≥ 90% notice the change
**Measurement:** Post-test survey

**Survey Question:**
"Did you notice the badge change from 'Partial' to 'Complete'?"
- Yes, clearly (1) ✓
- Yes, barely (2)
- No (3)

**Target:** ≥ 90% select "Yes, clearly" or "Yes, barely"

#### Toast Timing Appropriateness
**Metric:** User perception of toast auto-dismiss timing
**Target:** "About right" (≥ 80%)
**Measurement:** Post-test survey

**Survey Question:**
"How long should the completion toast stay visible?"
- Too short (1)
- Short (2)
- About right (3) ✓
- Long (4)
- Too long (5)

**Target:** ≥ 80% select "About right" for 3s duration

#### Partial vs Complete Distinction
**Metric:** User understands difference between partial and complete
**Target:** ≥ 90% understand the distinction
**Measurement:** Post-test survey

**Survey Question:**
"What's the difference between 'Partial' and 'Complete' enrichment?"
- Multiple choice:
  - a) Partial = leg labels, Complete = full details ✓
  - b) Partial = offline, Complete = online
  - c) Partial = failed, Complete = success
  - d) Don't know

**Target:** ≥ 90% select correct answer (a)

## Risk Mitigation

### Risk: Skeleton Jank
**Mitigation:**
- Use native driver for animations
- Optimize shimmer rendering
- Test on low-end devices
- Profile performance regularly

### Risk: Fade-in Too Fast/Slow
**Mitigation:**
- User testing for timing perception
- Configurable duration (300ms default)
- A/B test if needed
- Respect reduce motion preference

### Risk: Badge Change Unnoticeable
**Mitigation:**
- High contrast colors (teal vs purple)
- Icon change (spinner vs star)
- Subtle scale animation
- Position in prominent location

### Risk: Toast Annoying
**Mitigation:**
- Non-blocking (doesn't block map)
- Dismissible (X button)
- Auto-dismiss (3s after complete)
- Smart positioning (below header)

### Risk: Partial vs Complete Confusing
**Mitigation:**
- Clear color coding
- Distinct icons
- Tooltip or help text
- Progressive enhancement education
