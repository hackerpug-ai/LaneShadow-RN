## Task Description

Test UX risk areas for Shadow Setup (Epic 1) flow to ensure users successfully complete the mandatory local model download. Focus on abandonment rate, progress clarity, WiFi requirement messaging, background download behavior, and app state handling.

---

## Human Test Deliverable

**Test Scenario:** First-time user downloads "Your Shadow" (local AI model)

**Steps:**
1. Fresh install of LaneShadow app
2. Launch app for first time
3. See "Welcome to LaneShadow" screen
4. Tap "Get Started" button
5. See "Download Your Shadow" screen
6. See WiFi requirement message
7. See progress bar at 0%
8. See estimated time remaining
9. Switch from WiFi to cellular
10. See download pause with "WiFi required" message
11. Switch back to WiFi
12. See download resume
13. Background app (home button)
14. Return to app after 30 seconds
15. See download continued in background
16. Wait for download to complete
17. See "Your Shadow is Ready" celebration screen
18. Tap "Start Riding" button
19. See main app screen

**Gate:** Abandonment rate < 10%, Progress messaging clear

## Specification

### Test Scenarios

#### Scenario 1: Standard WiFi Download
**Flow:** User on WiFi throughout download

**Steps:**
1. Launch app → See welcome screen
2. Tap "Get Started" → See download screen
3. See progress 0% → "Starting download..."
4. See progress 25% → "About 2 min left"
5. See progress 50% → "About 1 min left"
6. See progress 75% → "About 30 sec left"
7. See progress 100% → "Your Shadow is Ready!"

**Success Criteria:**
- User completes download without aborting
- Progress updates feel responsive
- Time estimates are accurate (±20%)

#### Scenario 2: Cellular Block
**Flow:** User tries to download on cellular

**Steps:**
1. Launch app on cellular only
2. See welcome screen
3. Tap "Get Started" → See download screen
4. See "WiFi required" warning message
5. Download button disabled
6. See "Connect to WiFi to continue" message
7. User enables WiFi
8. Download button becomes enabled
9. User taps download → Download starts

**Success Criteria:**
- Clear WiFi requirement messaging
- User understands why download blocked
- Download starts immediately when WiFi enabled

#### Scenario 3: Background Download
**Flow:** User backgrounds app during download

**Steps:**
1. Start download on WiFi
2. See progress at 20%
3. Press home button (background app)
4. Wait 30 seconds
5. Return to app (tap icon)
6. See progress at 40% (continued in background)
7. See "Download continued while you were away" message

**Success Criteria:**
- Download continues in background
- Progress updates when returning
- User sees clear status message

#### Scenario 4: App Killed
**Flow:** User kills app during download

**Steps:**
1. Start download on WiFi
2. See progress at 30%
3. Force quit app (swipe away in app switcher)
4. Wait 10 seconds
5. Relaunch app
6. See "Resuming download..." message
7. See progress resume at 30%

**Success Criteria:**
- Download resumes from last point
- No data loss or corruption
- Clear "resuming" messaging

#### Scenario 5: Download Failure
**Flow:** Download fails due to network error

**Steps:**
1. Start download on WiFi
2. See progress at 50%
3. Simulate network failure (turn off WiFi)
4. See "Download failed" message
5. See "Retry" button
6. See error explanation: "Network connection lost"
7. User taps "Retry"
8. Download resumes from 50%

**Success Criteria:**
- Clear error messaging
- One-tap retry works
- No data corruption

#### Scenario 6: Insufficient Storage
**Flow:** User has insufficient storage

**Steps:**
1. Start download on WiFi
2. See progress at 0%
3. See "Not enough storage" message
4. See "Need 1.5 GB, only 500 MB available"
5. See "Free up space" button
6. See "Skip for now" button (disabled, hard gate)
7. User frees up space
8. Sees "Storage now available" message
9. Download starts automatically

**Success Criteria:**
- Clear storage requirement messaging
- Helpful action to free space
- Cannot skip (hard gate enforced)

### UX Metrics to Measure

#### Abandonment Rate
**Metric:** Percentage of users who start but don't complete download
**Target:** < 10%
**Measurement:** Analytics event tracking at each step

**Events:**
- `shadow_setup_started` - User taps "Get Started"
- `shadow_setup_completed` - User reaches "Your Shadow is Ready"
- `shadow_setup_abandoned` - User closes app during download

**Funnel:**
```
100% → shadow_setup_started
 95% → download_started
 90% → download_completed
 90% → shadow_setup_completed ✓
```

#### Progress Clarity
**Metric:** User understanding of download progress
**Target:** 90% of users understand current state
**Measurement:** Post-download survey (1 question)

**Survey Question:**
"How clear was the download progress?"
- Very clear (5)
- Clear (4)
- Neutral (3)
- Confusing (2)
- Very confusing (1)

**Target:** Average ≥ 4.0

#### WiFi Requirement Understanding
**Metric:** User comprehension of WiFi requirement
**Target:** 95% understand why cellular blocked
**Measurement:** Post-download survey (1 question)

**Survey Question:**
"Why did the app require WiFi for download?"
- Multiple choice with correct answer
- Target: 95% select correct reason

#### Background Behavior Perception
**Metric:** User awareness of background download
**Target:** 80% know download continues in background
**Measurement:** Post-download survey (1 question)

**Survey Question:**
"Did you know the download continued while you were away?"
- Yes / No

## Risk Mitigation

### Risk: High Abandonment Rate
**Mitigation:**
- Clear value proposition ("Why download?")
- Accurate time estimates
- Background download support
- Resume after interruption

### Risk: Unclear Progress
**Mitigation:**
- Percentage + time remaining
- Stage indicators ("Downloading...", "Verifying...")
- Smooth progress bar (not jumpy)
- Clear completion celebration

### Risk: WiFi Requirement Confusion
**Mitigation:**
- Explicit messaging ("WiFi required for 1.5 GB download")
- Visual indicator (WiFi icon)
- Explain why ("Large file size would exceed data limits")
- Auto-start when WiFi detected

### Risk: Background Download Unclear
**Mitigation:**
- Notification while backgrounded
- "Download continued while away" message on return
- Persistent notification with progress
- Clear "download complete" notification
