# CAPS-S07-T07 — iOS DesignReviewCaptureTests refresh against retrofitted IdleScreen

> **Task ID:** CAPS-S07-T07 · **Sprint:** [Sprint 07](./SPRINT.md) · **Agent:** swift-implementer · **Estimate:** 120 min · **Type:** INFRA · **Status:** Backlog · **Priority:** P0 · **Effort:** S
> **PRD Refs:** UC-FID-01

## Background

After CAPS-S07-T05 retrofits IdleScreen to use the new capsule + map controls, the existing `DesignReviewCaptureTests.swift` (from IDLE-S06-IOS-T04) will FAIL because it asserts the legacy testTags `idlescreen-greeting`, `idlescreen-greeting-headline`, `idlescreen-greeting-meta`, `idlescreen-advisory-card`, `idlescreen-current-user-greeting`. This task refreshes the suite to target `idle-context-capsule` and `idle-map-controls` while preserving the canonical `idle-screen.<state>.<theme>` attachment naming.

## Critical Constraints

**MUST:**
- Modify `ios/LaneShadowUITests/DesignReview/DesignReviewCaptureTests.swift` so all `test_idleScreen_*` methods (the 7 idle variants × applicable themes from Sprint 06 IDLE-S06-IOS-T04 — 14 methods total) wait for new accessibility identifiers `idle-context-capsule` and `idle-map-controls` instead of legacy ones
- Update `captureIdleScreen` helper (or per-test capture invocation) to keep emitting canonical attachment names `idle-screen.<state>.<theme>` matching `scripts/design-review/manifest.ts` discovery rules
- Remove explicit assertions/waits referencing legacy a11y identifiers; replace with assertions on `idle-context-capsule` (must exist on every variant) and `idle-map-controls` (must exist on every variant)
- Run `xcodebuild test -only-testing:LaneShadowUITests/DesignReviewCaptureTests` against retrofitted iOS Simulator build from CAPS-S07-T05 and verify all 7 idle variants × themes (≥14 methods) produce non-empty XCTAttachments named `idle-screen.<state>.<theme>`
- Preserve real Clerk auth pattern from Sprint 03 — NO auth stubs; every method continues to use `setupDeterminismEnvironment(app:colorScheme:)` for dark variants
- Pipe refreshed captures through `pnpm design:review --screens idle-screen` and verify report consumes them (no manifest-discovery drops)

**NEVER:**
- Modify production iOS code from this task (CAPS-S07-T05 owns IdleScreen.swift and IdleViewModel.swift)
- Bypass real Clerk auth — capture tests must use real auth per RULES.md §Real Device E2E Testing
- Re-introduce assertions on legacy `idlescreen-greeting-*` / `idlescreen-advisory-card` / `idlescreen-current-user-greeting` identifiers
- Hand-edit `.spec/design/system/refs/idle-screen/*.png` references; those are regenerated by `pnpm design:references` in CAPS-S07-T09
- Soften severity gates or rename attachments to dodge a manifest mismatch — fix the upstream cause

**STRICTLY:**
- Follow `IDLE-S06-IOS-T04` precedent — same number of methods (14), same attachment naming schema, same auth+determinism setup
- Use `XCUIElement.waitForExistence(timeout:)` for new identifiers; no `Thread.sleep`
- Keep test method bodies short and grep-friendly; helpers belong in `DesignReviewHelpers.swift`

## Specification

**Objective:** Refresh the iOS XCUITest design-review capture suite so all idle-screen capture methods target the post-retrofit accessibility surface (`idle-context-capsule` + `idle-map-controls`), drop legacy greeting/advisory selectors, and keep producing canonical `idle-screen.<state>.<theme>` attachments consumable by `pnpm design:review`.

**Success State:** `xcodebuild test -only-testing:LaneShadowUITests/DesignReviewCaptureTests` exits 0; the .xcresult bundle contains 14 attachments named `idle-screen.<state>.<theme>`; subsequent `pnpm design:review --screens idle-screen` discovers all 14 and produces a report with entries for each.

## Design

**References:** `.spec/design/system/views/idle-screen/idle-screen.html` + `IDLE-S06-IOS-T04`

**Interaction Notes:** Tests are non-interactive captures driven by sandbox story routing through `AppLauncher.launchApp(app, sandbox: true, sandboxStoryId:)`. The new `idle-context-capsule` identifier sits in the topOverlays slot; `idle-map-controls` sits trailing in the same slot — both must be `waitForExistence(timeout: 10)` before capture. Capture targets the full idle screen so the report still receives a screen-level PNG.

**Pattern:** `ios/LaneShadowUITests/DesignReview/DesignReviewCaptureTests.swift:33-122` — the existing 7-variant idle method shape; refresh means swapping wait selectors and removing legacy greeting selector assertions

**Pattern Source:** Sprint 06 IDLE-S06-IOS-T04 — same author owns the precedent; this task is the post-retrofit successor

**Anti-Pattern:** Stubbing or mocking the auth flow; renaming attachments to dodge manifest mismatches; deleting legacy variant tests instead of refreshing them; introducing `Thread.sleep` workarounds
