## BACKGROUND

This task builds the React hook layer that bridges the local discovery database (CUR-008) and SQL queries (CUR-009) to the UI components (DESIGN-001, DESIGN-002, DESIGN-005). Three hooks are required:

1. **useRouteDiscovery** — Returns nearby filtered routes for map display, with sort options (Best/Nearest).
2. **useRouteEnrichment** — Loads rich route data cache-first, falling back to server fetch when online.
3. **useIntentSearch** — Orchestrates the full intent search flow: normalize → cache lookup → Haiku (if online + miss) → cache write → params_to_sql → results. Returns OFFLINE_UNSUPPORTED for cache-miss + offline.

The intent search flow is the most complex. It implements AD-11: normalize the user's intent string, look up in op-sqlite cache, call server-side Haiku on miss + online, or return offline state on miss + offline. Enum validation drops hallucinated values (P5). The LLM never sees route candidates (P1).

**PRD references:** TRD-6.2 (query patterns), AD-11 (Haiku + cache single path), P0 (no on-device LLM), P1 (text → structure only), P5 (deterministic parser).

## CODE PATTERN

```typescript
// hooks/use-route-discovery.ts
export function useRouteDiscovery({
  lat, lng, radiusDeg = 0.5, archetype, sortBy = 'best'
}: RouteDiscoveryParams) {
  const [routes, setRoutes] = useState<CuratedRouteLean[]>([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const db = getDiscoveryDb();
    const nearby = queryNearby(db, lat, lng, radiusDeg, { archetype, limit: 100 });
    if (sortBy === 'nearest') {
      nearby.sort((a, b) =>
        haversine(lat, lng, a.centroid_lat, a.centroid_lng) -
        haversine(lat, lng, b.centroid_lat, b.centroid_lng)
      );
    }
    setRoutes(nearby);
    setLoading(false);
  }, [lat, lng, radiusDeg, archetype, sortBy]);

  return { routes, loading };
}
```

```typescript
// hooks/use-intent-search.ts
export function useIntentSearch(center: { lat: number; lng: number }) {
  const [state, setState] = useState<IntentSearchState>({ status: 'idle' });

  const search = useCallback(async (rawIntent: string) => {
    const db = getDiscoveryDb();
    const normalized = normalizeIntent(rawIntent);
    const cached = lookupIntentCache(db, normalized, CURRENT_SCHEMA_VERSION);

    if (cached) {
      bumpHitCount(db, normalized);
      const routes = runParamsQuery(db, cached, center);
      setState({ status: 'ok', routes, params: cached, source: 'cache' });
      return;
    }

    if (await isOnline()) {
      setState(prev => ({ ...prev, status: 'searching' }));
      const params = await fetchHaikuParams(rawIntent, center);
      const validated = validateEnums(params);  // drops hallucinated values
      writeIntentCache(db, normalized, validated, CURRENT_SCHEMA_VERSION);
      const routes = runParamsQuery(db, validated, center);
      setState({ status: 'ok', routes, params: validated, source: 'haiku' });
    } else {
      const recentIntents = topHitIntents(db, 6);
      setState({ status: 'offline_unsupported', recentIntents });
    }
  }, [center]);

  return { ...state, search, clear: () => setState({ status: 'idle' }) };
}
```

```typescript
// lib/discovery/intent/normalize.ts
const STOP_WORDS = ['i want to', 'show me', 'find me', 'some', 'a', 'the', 'please', 'rides', 'roads'];

export function normalizeIntent(raw: string): string {
  let s = raw.toLowerCase().trim();
  for (const word of STOP_WORDS) {
    s = s.replace(new RegExp(`\\b${word}\\b`, 'g'), '');
  }
  return s.replace(/\s+/g, ' ').trim();
}
```

## NOTES

- The intent search hook is the most safety-critical piece — it must never invoke on-device LLM or pass route data to an LLM. The LLM receives only the user's raw intent string and returns structured params.
- The `validateEnums` function must handle the case where Haiku returns an archetype not in the enum — it should silently drop the invalid value rather than crash.
- `params_to_sql` generates parameterized SQL — the SQL is constructed from validated params, never from raw LLM output.
- The `CURRENT_SCHEMA_VERSION` constant must be shared between the intent cache module and the Haiku extraction prompt schema version on the server.
