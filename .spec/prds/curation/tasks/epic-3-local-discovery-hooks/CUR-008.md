## BACKGROUND

The Curation PRD (TRD-6) specifies a local op-sqlite database called `discovery.db` that holds three tables: the lean tier of curated routes (bulk-synced from Convex), an enrichment cache for lazy-loaded rich data, and an intent-param cache for offline replay of intent searches. This is the foundation for all local discovery queries (CUR-009) and React hooks (CUR-010).

The app never writes to `curated_routes` — it is a read-only mirror of Convex data. Sync is pull-only: bulk on first launch, delta by `contentVersion` on subsequent launches. The enrichment cache uses LRU eviction at ~500 rows. The intent cache has no size cap but invalidates by `schema_version`.

**PRD references:** TRD-6.1 (schema), TRD-6.2 (query patterns), AD-2 (Convex canonical, SQLite cache), AD-8 (lean/rich tier split), AD-9 (shared stable ID).

### AC-002: Bulk lean sync populates routes
**GIVEN** a fresh database with zero rows in `curated_routes`
**WHEN** the app performs its first sync
**THEN** all lean-tier routes from Convex are inserted into `curated_routes`
**AND** each row has all required fields populated (route_id, name, state, source, primary_archetype, centroid_lat, centroid_lng, composite_score, content_version, synced_at)
**AND** row count matches the Convex `curated_routes` table count

**Verify:** Query `SELECT COUNT(*) FROM curated_routes` and compare to Convex record count.

### AC-003: Delta sync only fetches new versions
**GIVEN** the local database already has synced routes
**WHEN** a subsequent sync runs
**THEN** only routes with `content_version > local_max_content_version` are fetched
**AND** existing rows are updated in-place (UPSERT by route_id)
**AND** no rows are re-downloaded unnecessarily

**Verify:** Run sync twice — second sync should issue zero inserts/updates if no versions changed on the server.

### AC-004: Enrichment cache evicts at ~500 rows
**GIVEN** the enrichment cache has 500 rows
**WHEN** a new enrichment is fetched and cached
**THEN** the oldest row (by `fetched_at`) is evicted to maintain the ~500 cap
**AND** the new row is inserted
**AND** total row count remains approximately 500

**Verify:** Insert 501 enrichments, verify row count stays at 500 and the oldest was evicted.

## NOTES

- The lean tier is never evicted — it is the offline contract. Only the enrichment cache has LRU eviction.
- `contentVersion` is a monotonically increasing integer per route. Delta sync compares local max against server versions.
- The intent_param_cache table is created here but populated by CUR-010 (intent search hooks).
- Use op-sqlite (already a project dependency) for SQLite access — do not add new SQLite libraries.
