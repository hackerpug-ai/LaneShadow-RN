BACKGROUND
--------------------------------------------------------------------------------

**Problem:** `convex/lib/env.ts` exports `AI_PROVIDER = 'anthropic'` and `AI_MODEL = 'claude-sonnet-4-6'`. These are consumed by `sendMessage.ts` (metadata) and `tools/enrichRoute.ts` (`getModel('openai', AI_MODEL)` — a bug: `AI_MODEL` is a Claude model ID but it's handed to the openai provider). Both constants become dead weight once the registry (AI-001) owns model selection.

**Why it matters:** Keeping the dead constants around invites drift. Downstream AI-003 replaces all usages, so env must be ready first.

**Current state:** `convex/lib/env.ts` exports `AI_PROVIDER` and `AI_MODEL`. `CEREBRAS_API_KEY` is NOT read in the Convex env module; it's only in the user's shell.

**Desired state:** `convex/lib/env.ts` no longer exports `AI_PROVIDER`/`AI_MODEL`. `CEREBRAS_API_KEY` is set in the Convex deployment env and documented in `.env.example`. Convex functions that reference it do so via `process.env.CEREBRAS_API_KEY`.

**Reference:** `.spec/proposals/cerebras-provider-migration.md` — Steps 2 and 6.
