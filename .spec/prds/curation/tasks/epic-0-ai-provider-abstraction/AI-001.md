BACKGROUND
--------------------------------------------------------------------------------

**Problem:** Six Convex agent files call `getModel('anthropic', 'claude-sonnet-4-6')` / `getModel('anthropic', 'claude-haiku-4-5')` / `getModel('openai', AI_MODEL)` directly. Swapping providers or models requires editing every call site, and agents have no shared vocabulary for "high-reasoning" vs "low-reasoning" work.

**Why it matters:** We want to migrate to Cerebras (`@mariozechner/pi-ai` already supports it) and to make future swaps a one-file change. Without a registry, Epic 2 curation agents will inherit the same hardcoded-provider problem.

**Current state:** No `convex/actions/agent/lib/models.ts` exists. Agents import `getModel` from `@mariozechner/pi-ai` directly.

**Desired state:** A single registry file owns provider/model mapping. Agents import `getAgentModel('high' | 'low')` and never reference provider or model strings. `getAgentModelInfo(level)` returns the `{ provider, model }` tuple for metadata/logging (used by `sendMessage.ts`).

**Reference:** `.spec/proposals/cerebras-provider-migration.md` — Step 1.

```typescript
// convex/actions/agent/lib/models.ts
import { getModel } from '@mariozechner/pi-ai'

export type IntelligenceLevel = 'low' | 'high'

const MODEL_MAP: Record<IntelligenceLevel, { provider: string; model: string }> = {
  // qwen-3-235b-a22b-instruct-2507: 131K ctx, $0.60/$1.20 per 1M
  // 100% tool-match in benchmark vs llama3.1-8b (38% — disqualified for format errors)
  // See .spec/research/cerebras-vs-haiku/VERDICT.md
  low:  { provider: 'cerebras', model: 'qwen-3-235b-a22b-instruct-2507' },
  // zai-glm-4.7: 131K ctx, $2.25/$2.75 per 1M — reasoning-heavy orchestration + generation
  high: { provider: 'cerebras', model: 'zai-glm-4.7' },
}

export function getAgentModel(level: IntelligenceLevel) {
  const { provider, model } = MODEL_MAP[level]
  return getModel(provider, model as any)
}

export function getAgentModelInfo(level: IntelligenceLevel) {
  return MODEL_MAP[level]
}
```

import { describe, test, expect, vi } from 'vitest'

  getModel: vi.fn((provider: string, model: string) => ({ __provider: provider, __model: model })),

import { getModel } from '@mariozechner/pi-ai'
import { getAgentModel, getAgentModelInfo } from '../models'

describe('agent model registry', () => {

    const m = getAgentModel('high')

    expect(m).toMatchObject({ __provider: 'cerebras', __model: 'qwen-3-235b-a22b-instruct-2507' })
  })
})
