## BACKGROUND

This task wires the IntentSearchSheet design component (DESIGN-005) to the `useIntentSearch` hook (CUR-010) and integrates it into the discovery screen (CUR-012). The intent search bar allows natural language route discovery (UC-DISC-07): users type a free-text description of their ideal ride, and the system either returns cached results instantly or calls server-side Haiku to extract query params.

Key behaviors: cache hits show no spinner (instant), online cache misses show a spinner (~1-2s), offline cache misses show "Connect to search" with recent-intent chips. KeyboardAvoidingInput is required per the keyboard avoidance feedback pattern.

**PRD references:** UC-DISC-07 (Intent-Based Search), AD-11 (Haiku + cache single path), TRD-6.2.

## CODE PATTERN

```typescript
// components/discovery/IntentSearchBar.tsx
import { useIntentSearch } from '@hooks/use-intent-search';
import { IntentSearchSheet } from './intent-search-sheet';

interface IntentSearchBarProps {
  center: { lat: number; lng: number };
  visible: boolean;
  onDismiss: () => void;
  onResultsChange: (routes: CuratedRouteLean[]) => void;
}

export function IntentSearchBar({ center, visible, onDismiss, onResultsChange }: IntentSearchBarProps) {
  const { search, clear, ...searchState } = useIntentSearch(center);
  const [recentIntents, setRecentIntents] = useState<string[]>([]);

  const handleSearch = async (rawIntent: string) => {
    const result = await search(rawIntent);
    if (result.status === 'ok') {
      onResultsChange(result.routes);
    } else if (result.status === 'offline_unsupported') {
      setRecentIntents(result.recentIntents);
    }
  };

  const handleClear = () => {
    clear();
    onResultsChange([]);
  };

  const handleRecentIntentTap = (intent: string) => {
    handleSearch(intent);  // re-run cached search
  };

  return (
    <IntentSearchSheet
      visible={visible}
      searchState={mapToDesignState(searchState)}
      onSearch={handleSearch}
      onClear={handleClear}
      onRecentIntentTap={handleRecentIntentTap}
      recentIntents={recentIntents}
    />
  );
}
```

## NOTES

- The recent-intent chips in the offline state come from `topHitIntents(db, 6)` in the useIntentSearch hook. They represent previously cached searches ordered by hit count.
- When a chip is tapped in offline mode, it re-runs the search — which should find the intent in cache and succeed without network.
- The intent summary pill text should come from the normalized intent or a display-friendly version of it.
