## BACKGROUND

This task composes the design components from DESIGN-001 (RouteDiscoveryScreen layout) and DESIGN-002 (RoutePin) with the data hooks from CUR-009 (queries) and CUR-010 (useRouteDiscovery) into the working discovery screen. The screen shows a full-bleed map with copper pins, archetype filter chips with counts, state filter, sort toggle (Best/Nearest), route details bottom sheet, and full offline support after initial sync.

This is the main integration task for the discovery experience. It wires mock design components to real data via hooks.

**PRD references:** UC-DISC-01 through UC-DISC-06, TRD-6.2, AD-2, S8 (UI Infrastructure).

## CODE PATTERN

```typescript
// app/(tabs)/discovery.tsx
import { useRouteDiscovery } from '@hooks/use-route-discovery';
import { RouteDiscoveryScreen } from '@components/discovery/route-discovery-screen';

export default function DiscoveryTab() {
  const { location } = useLocation();  // existing location hook
  const [selectedArchetypes, setSelectedArchetypes] = useState<string[]>([]);
  const [sortMode, setSortMode] = useState<'best' | 'nearest'>('best');

  const { routes, loading } = useRouteDiscovery({
    lat: location.lat,
    lng: location.lng,
    radiusDeg: 0.5,
    archetype: selectedArchetypes.length === 1 ? selectedArchetypes[0] : undefined,
    sortBy: sortMode,
  });

  return (
    <RouteDiscoveryScreen
      routes={routes}
      loading={loading}
      selectedArchetypes={selectedArchetypes}
      onArchetypeChange={setSelectedArchetypes}
      sortMode={sortMode}
      onSortModeChange={setSortMode}
      onRoutePress={(routeId) => {/* open details sheet */}}
    />
  );
}
```
