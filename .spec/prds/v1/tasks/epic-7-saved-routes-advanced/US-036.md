## CRITICAL CONSTRAINTS

### MUST
- Re-plan button on route detail view
- On tap: navigate to HomeMapScreen, create new session pre-loaded with saved route context
- Display saved route as first attachment in the new session
- Show refinement suggestion chips (e.g. "Make it shorter", "Avoid highways", "More scenic")
- Original saved route remains unchanged

### NEVER
- Modify the original saved route during re-plan
- Reuse the original session — always create a new one
- Delete or archive the source saved route

### STRICTLY
- New session must reference the saved route ID for traceability

## SPECIFICATION

**Objective:** Allow users to start a new ride planning session using an existing saved route as the starting context, enabling iterative refinement without losing the original.

**Success looks like:** User taps Re-plan on a saved route, lands on the map screen with a new session that has the saved route pre-loaded as context, can refine via chat, and the original route is untouched.

### Interaction Notes
- Re-plan button: secondary style, placed below Save/Share actions in detail view
- On tap: create new session via existing session creation flow
- Pass savedRouteId and route snapshot as initial context
- Suggestion chips: ["Make it shorter", "Avoid highways", "More scenic", "Change destination"]
- Navigation: router.replace to HomeMapScreen (not push, to avoid back-stack issues)

### Code Pattern

```typescript
// useReplanSession hook
export function useReplanSession() {
  const createSession = useCreateSession()
  
  return async (savedRoute: SavedRoute) => {
    const sessionId = await createSession({
      replanFromRouteId: savedRoute._id,
      initialContext: {
        origin: savedRoute.routeSnapshot.origin,
        destination: savedRoute.routeSnapshot.destination,
        routeSnapshot: savedRoute.routeSnapshot,
      },
    })
    router.replace({ pathname: '/(app)/(tabs)/', params: { sessionId } })
  }
}
```

### Anti-pattern (DO NOT)
Do NOT clone the saved route into a new saved route — re-plan creates a new session, not a new saved route. Do NOT modify any field on the original saved route document.
