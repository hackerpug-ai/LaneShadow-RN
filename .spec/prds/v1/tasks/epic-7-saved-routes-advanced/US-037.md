## CRITICAL CONSTRAINTS

### MUST
- Mark as Ridden toggle on route detail view
- Ridden badge with formatted date on list card
- Ridden/Planned filter in saved routes list
- Untoggle support (mark as not ridden)

### NEVER
- Delete ridden history — untoggle simply clears the flag and timestamp
- Auto-mark routes as ridden (always user-initiated)

### STRICTLY
- Date format: relative ("2 days ago") for < 7 days, absolute ("Mar 15") for older

## SPECIFICATION

**Objective:** Allow users to mark saved routes as ridden with a timestamp, display ridden status on list cards, and filter the list by ridden/planned status.

**Success looks like:** User can toggle ridden status on a saved route, see the ridden badge with date on list cards, and filter the saved routes list to show only ridden or only planned routes.

### Interaction Notes
- Toggle: Switch component with label "Mark as Ridden"
- Badge on list card: small pill with checkmark icon + formatted date
- Filter: segmented control or chip group at top of list ["All", "Ridden", "Planned"]
- Date formatting: use relative time for < 7 days, short date for older
- Toggle off: confirmation not needed, just clear the fields

### Code Pattern

```tsx
// Ridden badge on card
{route.markedAsRidden && (
  <View style={styles.riddenBadge}>
    <CheckIcon size={12} />
    <Text variant="caption">{formatRiddenDate(route.riddenAt)}</Text>
  </View>
)}

// Filter in list
const [filter, setFilter] = useState<'all' | 'ridden' | 'planned'>('all')
const filtered = routes.filter(r => {
  if (filter === 'ridden') return r.markedAsRidden
  if (filter === 'planned') return !r.markedAsRidden
  return true
})
```

### Anti-pattern (DO NOT)
Do NOT add a confirmation dialog for toggling ridden status — it should be instant and reversible. Do NOT filter on the server for ridden status — client-side filter is sufficient for V1.
