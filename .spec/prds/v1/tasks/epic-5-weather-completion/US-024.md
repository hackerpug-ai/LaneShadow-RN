# US-024: Implement conditions scoring and 'Best for today' ranking

> Task ID: US-024
> Type: FEATURE
> Priority: P0
> Estimate: 45 minutes
> Assignee: backend-engineer

## CRITICAL CONSTRAINTS

### MUST
- Add computeConditionsScore: starts at 100, deducted by rain heavy=-40/moderate=-25/light=-10, wind high=-20/moderate=-10, temp hot/cold=-10
- Add conditionsScore to PlannedRouteOptionView
- Sort options descending by conditionsScore
- Populate overlaysPreview from real data (not hardcoded 'unavailable')
- Add weatherFetchedAt timestamp for staleness detection

### NEVER
- Use agentic reasoning for scoring — strictly deterministic
- Return negative scores (floor at 0)
- Use floating point for scores — integer only

### STRICTLY
- Scoring formula is additive deductions from 100, floored at 0
- No rain + moderate wind = 90
- Heavy rain + high wind + cold = 30

## SPECIFICATION

**Objective:** Implement a deterministic conditions scoring system that ranks route options by weather favorability and marks the top-ranked route as 'Best for today'.

**Success looks like:** Each route option has a conditionsScore. Options are sorted descending by score. overlaysPreview is populated from real weather data. The highest-scored route is identifiable for 'Best for today' badging.
