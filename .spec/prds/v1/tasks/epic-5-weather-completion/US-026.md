# US-026: Implement rain and temperature polyline overlay rendering

> Task ID: US-026
> Type: FEATURE
> Priority: P0
> Estimate: 150 minutes
> Assignee: ui-developer

## CRITICAL CONSTRAINTS

### MUST
- Extend buildRoutePolylines() for rain and temperature overlay types
- Rain bands: <30% = green, 30-60% = yellow, >60% = danger
- Temperature bands: <45F = cold (info color), 45-65F = cool, 65-85F = comfortable, >85F = hot (warning color)
- Use semantic color tokens for all overlay colors
- Tap on a polyline segment shows the weather value (rain% or temp)

### NEVER
- Use hardcoded hex colors — must use semantic tokens
- Break existing wind overlay rendering
- Render overlays when no data is available (disable toggle instead)

### STRICTLY
- Rain probability drives color, not precipitation amount
- Temperature bands use Fahrenheit thresholds

## SPECIFICATION

**Objective:** Extend the map's polyline rendering to support rain and temperature overlays alongside the existing wind overlay, with segment-level tap interaction.

**Success looks like:** Users can toggle rain and temperature overlays on the map. Polyline segments are colored by probability/comfort bands using semantic tokens. Tapping a segment reveals the value. Toggles are disabled when data is unavailable.
