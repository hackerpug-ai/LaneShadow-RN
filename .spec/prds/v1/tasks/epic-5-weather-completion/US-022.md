# US-022: Extend weatherProvider to fetch rain + temperature alongside wind

> Task ID: US-022
> Type: FEATURE
> Priority: P0
> Estimate: 60 minutes
> Assignee: backend-engineer

## CRITICAL CONSTRAINTS

### MUST
- Extend Open-Meteo URL to include precipitation_probability, precipitation, temperature_2m alongside existing wind params
- Single API call per point (not separate calls for different weather variables)
- Return combined ProbedConditionsPoint with wind + rain + temp data
- Keep concurrency limiter (MAX_CONCURRENT=8) and per-point timeout (8s)
- Update mapConditions to produce { wind: WindOverlay, rain: RainOverlay, temperature: TemperatureOverlay }

### NEVER
- Make separate API calls for different weather variables
- Break existing wind overlay functionality
- Remove or bypass the concurrency limiter

### STRICTLY
- Rain levels: none (prob<20% AND precip<0.1mm), light, moderate, heavy (prob>80% OR precip>4mm)
- Temperature levels: use existing getWorstTemperatureLevel thresholds

## SPECIFICATION

**Objective:** Extend the weather provider to fetch rain and temperature data alongside wind in a single Open-Meteo API call, and update mapConditions to produce all three overlay types.

**Success looks like:** 25 probe points return wind+rain+temp per point. mapConditions returns all 3 overlay types. URL includes all 6 variables. Existing wind behavior is unchanged.
