## CRITICAL CONSTRAINTS

### MUST
- Audit all new/modified components for testID and accessibilityLabel
- Add missing labels to all interactive elements
- Test with screen reader (VoiceOver on iOS)
- All interactive elements must be accessible

### NEVER
- Remove existing testID or accessibilityLabel values
- Add accessibility props that duplicate visible text without context

### STRICTLY
- Follow 08-technical-ui.md testID naming conventions

## SPECIFICATION

**Objective:** Ensure all V1 components meet accessibility standards with proper testID and accessibilityLabel props, and validate with screen reader testing.

**Success looks like:** Screen reader can navigate all interactive elements in the app. Every button, input, card, and toggle has a meaningful accessibilityLabel. All components have testIDs following project conventions.

### Naming Convention

```typescript
// testID convention: component-name.element-name
testID="saved-route-card.rating"
testID="chat-input.send-button"
testID="route-detail.navigate-button"

// accessibilityLabel: descriptive, action-oriented
accessibilityLabel="Rate route 4 out of 5 stars"
accessibilityLabel="Mark route as ridden"
accessibilityLabel="Open route in Google Maps"
accessibilityLabel={`Saved route: ${route.name}, ${route.rating} stars`}
```
