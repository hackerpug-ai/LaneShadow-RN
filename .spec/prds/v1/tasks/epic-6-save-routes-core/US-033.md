# US-033: Wire rename and delete in route detail view

> Task ID: US-033
> Type: FEATURE
> Priority: P0
> Estimate: 60 minutes
> Assignee: ui-developer

## CRITICAL CONSTRAINTS

### MUST
- Rename via inline edit (tap name to edit, confirm on blur or enter)
- Delete with confirmation dialog before executing
- Rename updates reflected across list and detail views (Convex reactivity)
- Delete navigates back to list after confirmation

### NEVER
- Hard delete — always call the soft-delete mutation
- Allow empty route names (validate before save)
- Skip the delete confirmation dialog

### STRICTLY
- Confirmation dialog text: "Delete '{name}'? This cannot be undone."
- Rename input auto-focuses when activated
- Delete button uses danger color styling

## SPECIFICATION

**Objective:** Wire rename and delete actions into the route detail view, connected to the Convex backend mutations.

**Success looks like:** Users can tap a route name to rename it inline. Rename propagates everywhere via Convex reactivity. Users can delete with a confirmation dialog, which soft-deletes and navigates back to the list.

### Interaction Notes
- Inline rename: TextInput replaces Text component on tap, auto-focus, submit on Enter/blur
- Delete button in action area with danger styling (semantic.color.danger)
- Confirmation uses Alert.alert() or custom dialog component
- After delete: router.back() to return to list
- Rename optimistic update via Convex mutation

### Rename Flow
```
[Route Name Text] → tap → [TextInput with current name, focused]
  → edit → Enter/blur → mutation → [Updated Text]
  → empty → blur → revert to original name
```

### Delete Flow
```
[Delete Button (danger)] → tap → Alert("Delete 'name'?", Cancel/Delete)
  → Delete → mutation → router.back()
  → Cancel → dismiss
```
