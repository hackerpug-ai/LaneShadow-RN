# US-031: Add Save Route action to RouteAttachmentCard

> Task ID: US-031
> Type: FEATURE
> Priority: P0
> Estimate: 60 minutes
> Assignee: ui-developer

## CRITICAL CONSTRAINTS

### MUST
- Save Route button on RouteAttachmentCard and route detail view
- Wire to savedRoutes.save mutation with optimistic update
- Confirmation toast with route name on success
- Disable button after save to prevent duplicates
- Haptic feedback (medium) on save

### NEVER
- Allow multiple saves of the same route (button must disable)
- Show a modal for save — use inline button + toast
- Skip haptic feedback

### STRICTLY
- Button states: enabled (unsaved) → loading (saving) → disabled (saved)
- Toast format: "Route '{name}' saved"

## SPECIFICATION

**Objective:** Add a Save Route action to the route card so riders can persist routes from the chat planning flow.

**Success looks like:** Tapping Save on a route card calls the Convex save mutation, shows a confirmation toast, and disables the button. The saved route appears immediately in the saved routes list via optimistic update.

The save action captures a snapshot of the route at the moment of save, including weather data. This means the saved route preserves the conditions that were relevant at planning time.
