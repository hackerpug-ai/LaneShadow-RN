## SPECIFICATION

**Objective:** Add four internal mutations to manage the `thinking_card` lifecycle: create, append thinking text, append structured steps, and finalize.

**Success looks like:** The thinking card can be created lazily, streamed to with thinking deltas and tool steps, then finalized as complete. All mutations follow existing patterns in sessionMessages.ts.

## DESIGN

### createThinkingCard

```typescript
export const createThinkingCard = internalMutation({
  args: { sessionId: v.id('planning_sessions') },
  returns: v.object({ messageId: v.id('session_messages') }),
  handler: async (ctx, args) => {
    const messageId = await ctx.db.insert('session_messages', {
      sessionId: args.sessionId,
      role: 'system',
      content: '',
      kind: 'thinking_card',
      status: 'streaming',
      thinkingSteps: [],
      createdAt: Date.now(),
    })
    return { messageId }
  },
})
```

### appendThinkingText

Same pattern as `appendReasoningChunk` — reads current content, appends delta, patches.

### appendThinkingStep

```typescript
export const appendThinkingStep = internalMutation({
  args: {
    messageId: v.id('session_messages'),
    step: thinkingStepValidator,
  },
  returns: v.null(),
  handler: async (ctx, args) => {
    const doc = await ctx.db.get(args.messageId)
    if (!doc) throw new Error('Message not found')
    const steps = doc.thinkingSteps ?? []
    steps.push(args.step)
    await ctx.db.patch(args.messageId, { thinkingSteps: steps })
  },
})
```

### finalizeThinkingCard

Same pattern as `finalizeAssistantMessage` — patches `status: 'complete'`.
