## Dependencies

```
AG ────────► WX
  │             │
  │             ▼
  └──────────► SR
```

- **AG** has no dependencies — it is the primary planning entry point using agentic AI
- **WX** depends on AG — weather badges and overlays are displayed within route results generated by the agent
- **SR** depends on both AG (routes must be generated before they can be saved) and WX (saved route detail views include weather overlays)

## Group Rationale

### AG — Agentic Conversational Planning

This is LaneShadow's primary differentiator. No competitor offers an agentic conversational planning experience on a map. The AG group covers the full planning flow: chat-based input, agent-powered intent understanding, route generation as chat attachments, iterative refinement via follow-up messages, session management, temporary message overlays on the map, and expanded chat history.

**Agent Architecture**: The system uses pi core to create a ride planning agent that:
- Interprets natural language ride descriptions through agentic reasoning
- Maintains conversation context across sessions for iterative refinement
- Generates conversational responses and route descriptions
- Orchestrates deterministic workflows for route computation, weather fetching, and persistence
- Handles errors and edge cases with helpful, conversational fallbacks

All other groups depend on having routes to work with.

### WX — Weather & Conditions

Weather data is partially built (wind overlay exists). This group closes the full story: rain and temperature overlays on the map polyline, weather badges on route attachment cards, conditions-based route ranking, the expandable hourly weather timeline, departure time adjustment via chat, and weather error recovery. Weather is always-on in the planning experience — not a toggle to find.

### SR — Saved Routes & Favorites

Retention depends on route persistence. This group covers the full lifecycle of a saved route: creation from chat attachments, browsing, search, filtering, renaming, deletion, rating, and marking as ridden. It also covers the favorite roads personalization loop, re-planning from saved routes, and export to external navigation apps.
