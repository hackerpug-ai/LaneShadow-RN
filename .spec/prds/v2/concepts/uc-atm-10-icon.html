§01

Purpose & Scope

LSIcon is the sole icon primitive for LaneShadow V2. All 31 glyphs are design-owned SVGs — no platform system icons (SF Symbols, Material Icons) are permitted anywhere in the codebase. Each icon resolves its color through the ContentColor enum (never a raw hex), its size through IconSize tokens, and its stroke through the shared icon.stroke.width = 1.5px token.

Icons are bundled at build time: iOS emits Asset.xcassets or SwiftUI Shape conversions; Android emits Compose ImageVector resources. The pnpm icons:check script enforces that every basename matches an IconName enum value exactly.

Forbidden patterns — audit grep rejects these
  • UIImage(systemName:) — SF Symbols API on iOS
  • Image(systemName:) — SwiftUI SF Symbols shorthand
  • Icons.Filled.* — Material Icons filled family (Android/Compose)
  • Icons.Outlined.* — Material Icons outlined family (Android/Compose)
  • Icons.Rounded.* — Material Icons rounded family (Android/Compose)
  • Icons.Sharp.* — Material Icons sharp family (Android/Compose)
  • Icons.TwoTone.* — Material Icons two-tone family (Android/Compose)
  • MaterialIcon — Any MaterialIcon import alias
  • systemImage: — SwiftUI Label(systemImage:) pattern

The pnpm icons:check script greps for all patterns above and fails CI if any match is found. Post-Sprint-2 the codebase must show zero hits on these patterns across both iOS and Android targets.

§05

Motion & Hover

Icons themselves carry no animation. Interactive contexts that wrap an LSIcon may apply the following token-referenced transitions.

Press state — parent button scales the icon to 0.92× over 100ms ease-out. Resolves from motion.duration.fast (100ms) + motion.easing.decelerate.

Hover state (web / Mac Catalyst) — icon color transitions from content.secondary to content.primary over 150ms. Resolves from motion.duration.quick (150ms) + motion.easing.standard.

Bookmark fill toggle — cross-fade between bookmark and bookmarkFill over 180ms with a spring scale bounce (scale 0.85 → 1.08 → 1.0). Resolves from motion.recipe.favoriteToggle.

§06

Rationale & AC Trace

Design-owned icons ensure visual consistency across platforms without platform symbol drift. 1.5px stroke was chosen as the minimum weight that reads clearly at 12px (xs) while remaining crisp at 32px (xl) — confirmed via optical testing at 1x and 2x pixel density.

AC-01tokens/icons/ contains exactly the named SVG files; basenames match IconName enum (enforced by pnpm icons:check).
AC-02LSIcon(name: .compass, size: .md) renders at sizing.icon.md with 1.5px rounded stroke; color: .signal resolves to color.signal.default.
AC-03Android renders identical glyph per parity manifest — visual diff tolerance ≤ 2px at 2x density.
AC-04Post-Sprint-2 grep: zero references to UIImage(systemName:) / Image(systemName:) / Icons.Filled.* / Icons.Outlined.* anywhere in the codebase.
AC-05"Atoms / Icon" sandbox story renders every name × size (5 sizes × 31 glyphs = 155 cells) in a swatch grid on both platforms.
AC-06iOS XCTest and Android JUnit verify LSIcon rejects raw color inputs at type-check and that stroke width resolves from icon.stroke.width token.
§07

Dark Mode

All 31 glyphs rendered on surface.primary (dark) = ink-800. Stroke and fill icons both resolve via --content-primary: var(--ink-050) — no glyph-level color override needed. Signal-colored icons (compass, pin, sparkle) remain copper-500 unchanged since signal tokens are not reassigned in .mode-dark.
