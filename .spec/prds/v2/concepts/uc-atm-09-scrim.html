§01 · Purpose

Depth without darkness.

LSScrim dims the map canvas when a drawer or modal needs visual priority. The warm-black tint (rgba(34,24,16,…) — the same ink-800 base used across the dark palette) preserves the paper feeling rather than flattening to a cold grey overlay. At 0.35 the map remains recognizable. At 0.60 the drawer or modal holds full attention. At 0.95 the scrim is functionally opaque — reserved for critical modal interruptions.

Touches pass through by default. The rider can still interact with map controls visible beneath. The blocking property is opt-in and enables tap-to-dismiss on the scrim layer itself — needed for BottomSheet and Modal, not for SessionsDrawer which uses an explicit close button.

§03 · States / Variants

Blocking vs non-blocking — and three use-sites.

Touch behavior
Touch-through (default) pointer-events: none. Taps pass through to map controls beneath. Used by LSSessionsDrawer — rider can still interact with the visible map strip.
Blocking (explicit opt-in) pointer-events: auto. Captures taps. Fires onTap callback. Used by BottomSheet (tap-to-dismiss) and Modal (tap-to-cancel). Must be explicitly enabled — never default.

§05 · Motion

Fade-in on mount — consumed by organisms.

Recipe
Trigger
Effect
none (static atom)
Mount / opacity prop change
LSScrim itself has no intrinsic animation. Fade-in and fade-out transitions are owned by the organism that renders the scrim (LSBottomSheet, LSModal, LSSessionsDrawer). The organism animates opacity from 0 to target — this keeps motion logic at the correct layer and lets each organism tune its own enter/exit timing.
§06 · Rationale

Warm-black, not cold.

Design Intent

Using rgba(34,24,16,…) — the ink-800 warm base — instead of pure rgba(0,0,0,…) preserves the Copper paper feeling even when the map is dimmed. A cold scrim conflicts with the warm paper aesthetic and makes the overlaid surface look fluorescent by contrast.

Six opacity levels are defined because different overlay contexts have different information requirements. The SessionsDrawer at 0.35 still lets the rider see the map edge; a save-route modal at 0.60 demands full visual attention. Exposing the full scale as tokens means product engineers never need to invent opacity values.

Touch-through by default reflects the map-first philosophy. The Navigator's map is always the primary surface. Blocking must be opted into intentionally, not accidentally inherited.
