§01

Purpose & Scope

UC-ATM-11 establishes the typed cross-platform contract for LSMap before either iOS (UC-ATM-12) or Android (UC-ATM-13) implementation begins. Its sole output is a typed API surface, a token reference map, sandbox fixture data, access-token loading conventions, and a stub renderer. No Mapbox SDK symbols appear in this contract — they are internal implementation details hidden behind the platform wrappers.

This contract lands first so iOS and Android engineers can work in parallel without contract drift. The stub renders an LSGlassPanel error fallback for every story until the platform implementations replace it.

§02

Typed API

All types below are cross-platform. iOS idiom uses Swift structs + enums. Android idiom uses Kotlin data classes + sealed classes. Field names are identical across platforms — only syntax differs.

CameraPosition — map viewport anchor

// Cross-platform: Swift struct / Kotlin data class
struct CameraPosition {
  center:   LatLng         // { lat: Double, lon: Double }
  zoom:     Double         // Mapbox zoom level 0–22; typical ride view: 10–14
  pitch?:   Double         // 0–60 degrees; nil = 0 (flat)
  bearing?: Double         // 0–360 degrees; nil = 0 (north-up)
}

Annotation — typed map point

enum AnnotationKind {
  // Color resolves from status tokens — not Mapbox SDK colors
  start     // color.status.success   (#3D8A5E) · 14px circle · 2.5px card border
  end       // color.status.recording (#C43C3C) · 18px ring circle · inner dot 6px
  waypoint  // color.status.info      (#3A6FBE) · 12px circle
}

struct Annotation {
  kind:       AnnotationKind
  coordinate: LatLng
  label?:     String          // Optional floating label text
}

RouteVariant + PolylineData — multi-polyline rendering

enum RouteVariant {
  best              // color.route.best  #EE7C2B · stroke-width sizing.stroke.md (2px)
  alt1              // color.route.alt1  #4D8470 · stroke-width sizing.stroke.md
  alt2              // color.route.alt2  #6B7B8F · stroke-width sizing.stroke.md
  custom(ColorToken) // escape hatch — still token-gated, no raw color
}

struct PolylineData {
  coordinates:  [LatLng]        // ordered path points
  variant:      RouteVariant   // determines stroke color
  strokeWidth?: StrokeSize     // .sm(1) / .md(2) / .lg(3); default .md
}

MapMode + CameraFit + MapError

enum MapMode {
  preview     // gestures disabled; static viewport
  interactive // pan / zoom / rotate enabled; onTap fires
}

enum CameraFit {
  static                                   // use CameraPosition as-is
  polyline(padding: SpacingToken)         // fit bounds of first polyline
  polylines(padding: SpacingToken)        // fit union bounds of all polylines; spacing.4 = 16dp/pt
}

enum MapError {
  missingToken       // MAPBOX_ACCESS_TOKEN not set at build time
  networkUnavailable // device offline or DNS failure
  styleLoadFailed    // Studio URL unreachable or malformed
}

§03

Fallback & Token Setup

The stub LSMap renders an LSGlassPanel error fallback for every story until UC-ATM-12 and UC-ATM-13 replace it. The fallback must not crash — it only renders a translucent glass callout over the dormant paper map background.

LSGlassPanel error fallback — two error variants

MapError.missingToken

Map unavailable

Missing access token — set MAPBOX_ACCESS_TOKEN in build environment.

MapError.networkUnavailable

Map unavailable

No network — check connection and try again.

iOS — access token convention

// ios/LaneShadow/Info.plist
// Key: MBXAccessToken
// Value: $(MAPBOX_ACCESS_TOKEN)
//
// Build-time substitution via Xcode build setting:
// MAPBOX_ACCESS_TOKEN = pk.eyJ1Ijoi...
// Set in local .xcconfig (gitignored)
// or via CI/CD environment variable.
//
// Never commit a literal token.
// LSMap reads MBXAccessToken from
// Bundle.main.infoDictionary at runtime.
// If nil → MapError.missingToken → fallback.

Android — access token convention

// android/app/src/main/res/values/secrets.xml
// Generated pre-build by Gradle task from
// MAPBOX_ACCESS_TOKEN environment variable.
// File is gitignored.
//
// build.gradle.kts:
// tasks.register("generateSecrets") {
//   doFirst {
//     val token = System.getenv(
//       "MAPBOX_ACCESS_TOKEN") ?: ""
//     // writes secrets.xml with token value
//   }
// }
// preBuild.dependsOn("generateSecrets")
//
// LSMap reads from R.string.mapbox_access_token.
// If blank → MapError.missingToken → fallback.

§05

Motion — routeDrawOn

When polylines are injected into LSMap, a stroke-dashoffset animation draws the path from start to end. This recipe is motion.recipe.routeDrawOn and applies to the best polyline only; alt1/alt2 fade in at opacity 0 to 1 simultaneously.

Duration: 600ms per 100km of route distance, capped at 1400ms. Resolves from motion.duration.routeDrawOn.

Easing: cubic-bezier(0.25, 0.1, 0.25, 1.0) — resolves from motion.easing.standard.

Alt fade duration: 300ms ease-out, starts when the best polyline draw reaches 40% progress.

Camera fit animation: CameraFit.polylines triggers a Mapbox camera ease over 400ms once polylines are drawn. Resolves from motion.duration.cameraEase (400ms).

Rationale & AC Trace

The contract-first approach allows iOS and Android engineers to work concurrently without waiting for each other. The stub renders proof-of-life via the same LSGlassPanel used by UC-ATM-05, confirming that token + fixture wiring is correct before SDK code lands.

AC-01tokens/api/LSMap.contract.md contains all typed definitions — CameraPosition, Annotation, RouteVariant, PolylineData, MapMode, CameraFit, MapError — with field-level docs. LSMap signature takes polylines: [PolylineData] (plural).
AC-02No Mapbox SDK symbols (MapView, MapboxMap, CircleAnnotation, LineLayer) appear in the contract surface — only in the platform wrappers.
AC-03map.style.light, map.style.dark, sizing.stroke.md, color.route.{best,alt1,alt2} token references documented in contract; annotation-kind mapping to color.status.* documented.
AC-04tokens/sandbox/fixtures/routes.fixtures.json contains 3+ routes; route_results_three_alts has PolylineData entries with variants .best, .alt1, .alt2.
AC-05iOS stub LSMap type-checks, compiles, and renders LSGlassPanel fallback until UC-ATM-12 replaces it.
AC-06Android stub analogous to iOS — type-checks, compiles, renders fallback until UC-ATM-13 replaces it.
AC-07"Atoms / Map (stub)" story renders fallback on both platforms — proving token + fixture pipeline is wired end-to-end before SDK code lands.
