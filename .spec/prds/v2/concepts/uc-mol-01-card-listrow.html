§01

Purpose & Atomic Composition

LSContentCard wraps LSCard (UC-ATM-05) with a structured slot schema — header, title, subtitle, metadata, action footer — giving organisms a single, consistent API for card-shaped content. LSListRow composes LSAvatar (UC-ATM-04), LSIcon (UC-ATM-10), and optionally LSButton (UC-ATM-02) into a horizontal interactive row that respects minimum touch-target sizing.

Neither molecule hard-codes primitive values. Every color references a semantic token, every size references a dimension token. Organisms can render a ride card, a settings list, or a community feed row by composing these molecules — no raw View or Box with literal colors needed.

§05

Motion Recipes

listRowPress — background transitions to surface.inset on touchDown, returns on touchUp. Duration: 120ms ease-out. No scale transform on list rows — scale is reserved for card promotion flows.

cardEnter — cards entering a feed list animate with opacity 0→1 + translateY 8px→0. Duration: 240ms cubic-bezier(0.22, 1, 0.36, 1). Stagger delay 40ms per item in a list. No spring physics at this tier — springs are reserved for organism-level navigation transitions.

Toggle state change: background color transition 200ms ease-in-out. Thumb translate transition 200ms ease-in-out.

§06

Rationale + AC Traceability

LSContentCard was extracted because every Navigator organism — route results, saved rides, community cards — needed the same surface.card + radius.lg + elevation.2 container but with different slot content. Coalescing into one molecule prevents organisms from each choosing their own radii or shadows and guarantees visual consistency across 8+ screens.

LSListRow enforces the 44pt touch-target floor mandated by both Apple HIG and Material You. The leading/title gap of spacing.3 (8px) was chosen empirically: it gives icon-to-text breathing room without the leading content becoming detached from its label at small viewport widths.

Developer can render LSContentCard(title: "Route X", subtitle: "42 mi · 1h 12m") and see color.surface.card + radius.lg + elevation.2 + layout spacing from spacing.* tokens; title in typography.ui.title.md, subtitle in typography.ui.body.md. [06-uc-mol.md §UC-MOL-01]
Developer can render LSListRow(leading: .avatar(...), title: "Name", subtitle: "Detail", trailing: .chevron) and see minimum height sizing.touchTarget, leading/title gap spacing.3, vertical padding spacing.2. [06-uc-mol.md §UC-MOL-01]
Developer can tap an LSListRow with onTap and see the handler invoked exactly once; rows without onTap do not surface a pressed highlight. [06-uc-mol.md §UC-MOL-01]
Developer can find stories: "Molecules / ContentCard" — With Image Header, Title Only, Title+Subtitle+Chips, With Actions. [06-uc-mol.md §UC-MOL-01]
