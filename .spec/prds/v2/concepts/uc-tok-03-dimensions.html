§01

Purpose

All spatial relationships — the gap between a label and its icon, the height of a button, the corner radius of a card — derive from this token set. The 4px base grid keeps proportions consistent across iOS (pt) and Android (dp). Pixel values annotated here are the canonical source; platform generators convert to the correct unit at build time.

The touch target minimum exists to ensure accessibility. The elevation tiers exist to communicate surface hierarchy. The opacity scale exists so that disabled, overlay, and dim states are consistent everywhere.

§03

States & variants

The spacing scale is unit-agnostic at the token level. Platform generators emit the correct unit: px for web, pt for iOS, dp for Android. The numeric value is always the 1x pixel equivalent. Touch target sizing is enforced at generation time — if a component height token falls below 44px (iOS) or 48dp (Android), the validator emits an error.

Icon sizes are frame sizes, not SVG viewport sizes. The SVG may have a built-in viewBox of 24×24; the rendered frame is scaled to the token value. stroke-width is always 1.5px at the SVG level — the visual weight scales proportionally with frame size.

§05

Motion

Dimension tokens carry no animation. Component layout adapts to Dynamic Type scale changes via platform-managed re-layout, not motion tokens. See UC-TOK-04 for all named motion recipes.

§06

Rationale & references

Design decisions

The 4px base grid was chosen because it divides cleanly into common screen densities (1x, 1.5x, 2x, 3x) without sub-pixel rendering artifacts. The 13 spacing rungs cover all layout needs without introducing arbitrary intermediate values.

The elevation.overlay tier is deliberately distinct from tier 3 because glass panels, bottom sheets, and drawers all sit at a different perceptual height than regular cards — they float above the map, not just above adjacent content.

PRD acceptance criteria

  • AC-01 spacing section with 13 rungs 0–64px in semantic.tokens.json
  • AC-02 sizing section: touchTarget, icon.{xs,sm,md,lg,xl}, component heights, stroke widths, icon.stroke.width
  • AC-03 radius section: none, xs, sm, md, lg, xl, pill
  • AC-04 opacity section: disabled(0.38), overlay(0.60), focus(0.12), dim(0.76), veil(0.95)
  • AC-05 elevation section: 0, 1, 2, 3, overlay — overlay distinct from 2/3
  • AC-06 All tokens have $type: "dimension" with correct $value pattern
  • AC-07 Touch-target meets iOS 44px / Android 48dp at generation time
  • AC-08 pnpm tokens:validate exits zero errors
§07

Dark surfaces

Dimension tokens are unit values — they carry no color. The dark showcase confirms that spacing rulers, sizing circles, radius boxes, and elevation cards all remain visible and readable on warm-ink surfaces.
