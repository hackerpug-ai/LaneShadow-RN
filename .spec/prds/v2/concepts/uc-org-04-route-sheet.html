§01 · Purpose

Save or ride this.

LSRouteSheet is the bottom sheet presented on RouteDetailsScreen. It composes from LSBestBadge, LSInstrumentReadout (4 metrics), LSWeatherTimeline (6 cells), and a sticky action row. It is the rider's final decision moment — the sheet surfaces all the data needed to commit or save.

Presented via LSBottomSheet (UC-MOL-03) with .large detent. The drag handle allows partial reveal — riders can peek metrics and drag to full height for weather detail.

§06 · Rationale

Decision surface.

Design Decision

The sheet uses a two-action model (Save + Ride this) with a 1:2 flex ratio. Ride this is physically larger because it is the committed action — the one that starts navigation. Save is secondary, available for riders who want to bookmark before deciding.

The weather timeline uses tinted cells rather than icons alone. A rain-blue cell communicates weather condition faster than reading a label because color perception is pre-attentive.

The drag handle matches the border-strong token (a warm gray) rather than signal copper — it is a gesture affordance, not a call to action.
