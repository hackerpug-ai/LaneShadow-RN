§01 · Purpose

One bar, five states.

LSChatInput is the Navigator's primary interaction surface — pinned 12pt from the device bottom, 12pt from each side. It layers three optional stacks from bottom to top: the input bar (always present), an optional suggestion chip row, and an optional LocationContextBar.

The input bar is backed by LSGlassPanel.chrome at 54pt height with radius.xl (18pt). Leading: a 36pt ghost collapse button. Trailing: a 42pt circular copper send button when value is non-empty, else a 42pt ghost filter button. Icons are 18–20pt.

§06 · Rationale

Design decisions.

Glass surface on map

The input floats above the map layer. LSGlassPanel.chrome with blur(14px) prevents the input from visually competing with map features underneath. The border at rgba(255,255,255,0.55) gives lift without a hard edge. Dark mode swaps to rgba ink-based glass.

  • 54pt height matches sizing.component.inputHeight token
  • 42pt send exceeds accessibility minimum (44pt target via visual frame)
  • 36pt collapse — less prominent, same tap target via padding

AC Traceability — 06-uc-mol.md §UC-MOL-06

  • Input bar renders at height sizing.component.inputHeight (54pt), radius.xl, LSGlassPanel.chrome backing
  • Trailing slot shows LSButton(.ghost, .sliders) when value empty; swaps to LSButton(.primary, .send) when non-empty
  • Tapping send fires onSend(value); input clears to placeholder
  • Tapping collapse fires onCollapse exactly once

  • locationBadge renders LSLocationContextBar above suggestion row
  • isThinking=true replaces trailing with LSSpinner; input submission disabled
