§01

Purpose

The pipeline enforces a single source of truth. Every color, spacing value, typography spec, motion recipe, icon name, and Mapbox style URL that exists in the app must trace back to semantic.tokens.json — never to a literal in component code. This document describes the generation pipeline, the 25-icon SVG catalog, font registration, Mapbox style token format, and the lefthook drift-check that prevents stale generated files from reaching main.

lefthook pre-commit drift-check sequence
1
Stage validation git diff --cached --name-only Detect if any tokens/semantic/**, tokens/icons/**, tokens/fonts/**, or tokens/platforms/** files are staged.
always runs
2
Schema validation pnpm tokens:validate Validates semantic.tokens.json against semantic.tokens.schema.json. Fails on any type mismatch, missing $type, or invalid $value pattern.
blocks commit on fail
3
Sync check pnpm tokens:sync-check Re-runs the generator in dry-run mode, diffs output against committed generated files. Exits non-zero if any platform output is stale relative to current source + icons + fonts.
blocks commit on fail
4
Icon set check pnpm icons:check Compares SVG filenames in tokens/icons/*.svg against the IconName enum in both Swift and Kotlin generated outputs. Fails if any icon is missing from an enum or an enum has an icon with no SVG file.
blocks commit on fail
5
Commit allowed exit 0 All three checks pass. lefthook allows the commit to proceed. Generated files and source are in sync.
commit proceeds
!
Any check fails exit 1 + error message lefthook blocks the commit and prints which check failed with the diff or schema error. Developer must run pnpm tokens:generate (or fix the schema) and re-stage before committing.
commit blocked
§03

States & variants

The generator runs deterministically — same input hash always produces the same output. The generated file banner includes the SHA-256 of the concatenated source inputs (semantic.tokens.json + sorted icon filenames + font manifest). If the banner hash mismatches the current input hash, pnpm tokens:sync-check exits non-zero.

Platform outputs are checked in to the repository so that native builds are not blocked by a Node dependency at build time. The drift-check enforces that checked-in outputs are never stale.

§04

Consumer atoms

LSIcon (iOS)
Reads LSIcon.Name enum. Loads SVG from xcassets or renders as native Shape. stroke-width: icon.stroke.width
LSIcon (Android)
Reads com.laneshadow.theme.generated.IconName. Loads via Compose Image painter from bundled SVG asset.
LSMap
Style URL loaded from LaneShadowTheme.map.style.light / .dark token. Never a hardcoded URL.
All typography atoms
Font registration at app startup (UIFont.registerFonts / Android ResourcesCompat). All three families available before first render.
§05

Motion

The pipeline itself has no motion behavior. The icon SVG assets use static geometry only — no SMIL animations. Motion recipes referencing icon-containing components are declared in UC-TOK-04.

§06

Rationale & references

Design decisions

The drift-check runs in lefthook rather than CI so that the feedback loop is immediate — the developer finds out about stale outputs before a push, not after a CI run. This prevents a class of "works on my machine" issues where a designer updates tokens but forgets to regenerate.

Icons are stored as SVG files (not icon fonts or image assets) to preserve crisp rendering at all display scales and to enable easy per-icon color overrides via the stroke attribute. The 1.5px stroke-width was chosen to read at every icon size from xs (12px frame) to xl (32px frame) without looking too heavy or too thin.

PRD acceptance criteria

  • AC-01 pnpm tokens:generate writes Swift/Kotlin/TS outputs from source
  • AC-02 LaneShadowTheme importable in Swift with no raw literals
  • AC-03 LaneShadowTheme importable in Kotlin with no raw literals
  • AC-04 pnpm tokens:sync-check exits 0 when in sync, non-zero on drift
  • AC-05 pnpm icons:check compares SVG set vs both platform enums
  • AC-06 lefthook pre-commit runs validate + sync-check + icons:check
  • AC-07 Generated files include banner with input hash
  • AC-08 map.style.light/dark tokens have $type: "string", match pattern
