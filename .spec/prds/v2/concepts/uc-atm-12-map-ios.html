§01

Purpose & Scope

UC-ATM-12 replaces the iOS stub from UC-ATM-11 with a production UIViewRepresentable wrapper that loads the Copper Mapbox Studio style, renders multiple polylines with per-variant colors, places typed annotations, honors preview vs interactive mode, camera-fits to polyline union bounds, and renders error fallbacks without crashing. Out of scope: offline packs, turn-by-turn nav, geocoding, 3D terrain, traffic layers.

The nine stories below are the canonical sandbox story set. Story names must match iOS and Android exactly — per the UC-SBX-01 parity manifest — so QA can run visual diff across platforms.

§03

UIViewRepresentable Wrapper

The production SwiftUI wrapper hides all Mapbox SDK symbols behind the UC-ATM-11 contract. Reference pseudocode — not compilable but captures the structural contract.

LSMapViewRepresentable — structural reference (Swift pseudocode)

// Production file: ios/LaneShadow/Components/LSMapViewRepresentable.swift
// SDK import is contained here — never leaks into contract surface.
import MapboxMaps

struct LSMap: View {
  let mode:          MapMode
  let camera:        CameraPosition
  var cameraFit:     CameraFit        = .static
  var polylines:     [PolylineData]   = []
  var annotations:   [Annotation]     = []
  var showFavorites: Bool             = false
  var onTap:         ((LatLng) -> Void)? = nil
  @Environment(\.colorScheme) private var colorScheme

  var body: some View {
    let token = Bundle.main.object(
      forInfoDictionaryKey: "MBXAccessToken") as? String
    if token == nil || token!.isEmpty {
      LSGlassPanel(error: .missingToken)
    } else {
      _LSMapViewRepresentable(
        mode: mode, camera: camera,
        cameraFit: cameraFit, polylines: polylines,
        annotations: annotations, onTap: onTap,
        styleURL: colorScheme == .dark
          ? Tokens.map.style.dark
          : Tokens.map.style.light,
        token: token!
      )
      .onReceive(networkMonitor.$isOnline) { online in
        // → MapError.networkUnavailable if offline
      }
    }
  }
}

struct _LSMapViewRepresentable: UIViewRepresentable {
  // makeUIView: init MapView, set ResourceOptions(accessToken)
  // updateUIView: diff polylines, add/update LineAnnotations per variant
  // color: Tokens.color.route.{best,alt1,alt2} — never raw hex
  // strokeWidth: Tokens.sizing.stroke.md — never raw number
  // cameraFit: camera.ease(to: unionBounds, padding: Tokens.spacing.s4)
  // scrollIsolation: gestureManager.options.panDecelerationFactor handled
  //   so outer ScrollView scroll is not hijacked
}

Scroll-isolation requirement: When an interactive LSMap is placed inside a SwiftUI ScrollView, the map's gesture recognizer must NOT hijack the outer vertical scroll when the user's drag originates outside the map bounds. Implementation: UIScrollView.gestureRecognizers delegation via shouldRecognizeSimultaneouslyWith — tested by XCTest verifying the outer scroll fires when drag starts above/below the map frame.

§05

Motion — routeDrawOn

Story 3 and 4 animate the best polyline using motion.recipe.routeDrawOn. On iOS this is implemented via a MapboxMaps layer property animation on line-dasharray or stroke-dashoffset — the approach is platform-specific but the token-referenced duration and easing are cross-platform.

Duration: motion.duration.routeDrawOn = max(600ms, min(1400ms, routeKm * 6ms)).

Camera fit ease: motion.duration.cameraEase = 400ms, motion.easing.standard.

Dark style transition: Mapbox loadStyle() is called with the new URL when colorScheme changes. The underlying MapView is NOT unmounted — the SDK reloads the style in-place (Story 7 verifies this).

§06

Rationale & AC + Tests

Nine stories give a complete coverage surface: two modes, three polyline configurations, camera fit, both themes, and both error paths. The test suite mirrors the story list exactly — each story has a corresponding XCTest.

XCTest: multi-polyline rendering — three LineAnnotations added; each has stroke color = Tokens.color.route.{best,alt1,alt2}; width = Tokens.sizing.stroke.md.
XCTest: style URL resolution — colorScheme light → Tokens.map.style.light URL loaded; dark → Tokens.map.style.dark URL loaded.
XCTest: token-gated fallback — nil MBXAccessToken → LSGlassPanel rendered, no MapView created, no crash.
XCTest: annotation rendering — Annotation.start → CircleAnnotation fill = color.status.success; Annotation.end → ring fill = color.status.recording.
XCTest: camera-fit-to-union-bounds — cameraFit: .polylines → camera eases to union of all polyline coordinate bounds with spacing.4 padding.
XCTest: onTap callback — interactive mode + simulated tap → onTap fires exactly once with the tapped LatLng.
XCTest: scroll-isolation — interactive LSMap inside ScrollView → vertical drag originating above map frame scrolls the ScrollView, not the map.
XCTest: no-symbol-leak — post-build grep of app binary confirms zero references to UIImage(systemName:) / Image(systemName:) in LSMap implementation files.
XCTest: dark style no-unmount — toggle colorScheme dark/light; verify underlying MapView instance identity is preserved (same pointer).
