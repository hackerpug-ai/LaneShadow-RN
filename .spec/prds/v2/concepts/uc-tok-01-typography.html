§01

Purpose

Typography in Navigator is not a size scale — it is a role system. Three semantic families exist because three fundamentally different communication registers are in use simultaneously: the Navigator voice (opinionated, warm, serif), the application chrome (clean, functional, Geist), and instrument readouts (tabular numerics, monospaced precision).

Every component downstream consumes a token name from one of these three namespaces. No raw font-size values appear in component code. When a designer changes typography.opinion.xl, every heading in every surface updates atomically across iOS, Android, and TypeScript.

§05

Motion

Typography tokens carry no motion behavior. Type rendering on iOS and Android is subject to Dynamic Type animation when the user changes their preferred text size in system settings — this is handled by the platform, not the design token layer. See UC-TOK-04 for all motion primitives and named recipes.

§06

Rationale & references

Design decisions

Newsreader was chosen over a standard editorial serif for its optical-size axis, allowing the same font to read as intimate caption text and commanding display headline without switching files. The italic variant carries the Navigator voice in UI — it signals opinion, not data.

JetBrains Mono is set with font-feature-settings: "tnum" 1 so all numeric readouts are tabular-lining — columns of numbers align precisely regardless of digit width. This is non-negotiable for distance, duration, and temperature readouts.

PRD acceptance criteria

  • AC-01 Three top-level namespaces in semantic.tokens.json
  • AC-02 Each variant has family, weight, size, lineHeight fields
  • AC-03 Three font families declared as bundled assets (see UC-TOK-05)
  • AC-04 Typography swatch story renders on iOS (UC-ATM-01)
  • AC-05 Android byte-identical values within density rounding
  • AC-06 pnpm tokens:validate exits zero errors
§07

Dark surfaces

All three type families remain legible on warm-ink dark surfaces. Content tokens swap automatically inside .mode-dark — no parallel style definitions needed in component code. The copper signal is unchanged.
