§01

Purpose & Scope

UC-ATM-13 replaces the Android stub from UC-ATM-11 with a production AndroidView wrapper inside a Composable that loads the Copper Mapbox Studio style, renders multiple polylines with per-variant colors, places typed annotations, honors preview vs interactive mode, camera-fits to union bounds, and renders error fallbacks without crashing. Out of scope: offline packs, turn-by-turn nav, geocoding, 3D terrain, traffic.

Story names and visual output must be identical to UC-ATM-12 per the UC-SBX-01 parity manifest. QA runs visual diffs against iOS frames — only the phone chrome (Android punch-hole vs iOS Dynamic Island) is permitted to differ.

§03

AndroidView Composable

The production Composable wraps MapView via AndroidView. All Mapbox SDK imports are contained here. Reference pseudocode — Kotlin structural contract, not compilable as-is.

LSMap Composable — structural reference (Kotlin pseudocode)

// Production file: android/app/src/main/java/.../components/LSMap.kt
// SDK import contained here — never leaks into contract surface.
import com.mapbox.maps.MapView
import com.mapbox.maps.plugin.annotation.generated.*

@Composable
fun LSMap(
  mode:          MapMode,
  camera:        CameraPosition,
  cameraFit:     CameraFit        = CameraFit.Static,
  polylines:     List<PolylineData> = emptyList(),
  annotations:   List<Annotation>   = emptyList(),
  showFavorites: Boolean          = false,
  onTap:         ((LatLng) -> Unit)? = null
) {
  val token = stringResource(R.string.mapbox_access_token)
  val isDark = isSystemInDarkTheme()

  if (token.isBlank()) {
    // MapError.MissingToken → LSGlassPanel fallback
    LSGlassPanelFallback(error = MapError.MissingToken)
    return
  }

  val mapView = remember { MapView(context) }

  AndroidView(
    factory = { mapView },
    update = { mv ->
      // Style URL resolves from Tokens — no raw URL strings
      val styleUrl = if (isDark)
        Tokens.map.style.dark else Tokens.map.style.light
      mv.getMapboxMap().loadStyleUri(styleUrl)

      // Polylines: one LineLayer per PolylineData entry
      // color: Tokens.color.route.{best,alt1,alt2} — no raw hex
      // lineWidth: Tokens.sizing.stroke.md — no raw dp literal

      // Annotations: CircleAnnotationManager per AnnotationKind
      // start fill: Tokens.color.status.success
      // end fill: Tokens.color.status.recording

      // CameraFit.Polylines: easeCamera to union LatLngBounds
      // padding: Tokens.spacing.s4.toInt() → 16dp EdgeInsets

      // Gesture mode
      mv.getMapboxMap().setGesturesEnabled(mode == MapMode.Interactive)

      // onTap via OnMapClickListener
      if (mode == MapMode.Interactive) {
        mv.getMapboxMap().addOnMapClickListener { latLng ->
          onTap?.invoke(latLng)
          true
        }
      }
    }
  )
}

// Scroll-isolation: NestedScrollInteropConnection
// Applied to the AndroidView Modifier so LazyColumn / Column(verticalScroll)
// parent scroll is not hijacked by map gestures when drag originates
// outside the map composable bounds.
// Modifier.nestedScroll(rememberNestedScrollInteropConnection())

Scroll-isolation requirement: When an interactive LSMap Composable is placed inside a LazyColumn or Column(Modifier.verticalScroll(...)) parent, the map must NOT hijack the outer vertical scroll when the user's drag originates outside the map bounds. Android implementation uses Modifier.nestedScroll(rememberNestedScrollInteropConnection()) on the AndroidView composable. Tested by Compose UI test verifying the parent scroll fires when drag starts above/below the map composable bounds.

§05

Motion — routeDrawOn

Android implements motion.recipe.routeDrawOn via a Mapbox LineLayer property animator — animating the line-gradient or line-trim-offset property over the token-specified duration. The result must be visually equivalent to the iOS stroke-dashoffset approach.

Duration: motion.duration.routeDrawOn = max(600ms, min(1400ms, routeKm * 6ms)). Computed in the Composable using ValueAnimator.

Camera fit animation: mapView.camera.easeBy(CameraOptions, animationOptions: { duration = 400 }). Resolves from motion.duration.cameraEase token.

Dark style transition: loadStyleUri() is called with the new URL when isSystemInDarkTheme() changes. The underlying MapView is NOT recreated — side-effect in the AndroidView update block handles it (Story 7 verifies this).

Android story names exactly match iOS (UC-ATM-12). The parity manifest in UC-SBX-01 can diff them mechanically. All token values resolve to the same numbers in dp as in pt — the only platform difference is the rendering engine (Mapbox Android SDK vs iOS SDK) and the host idiom (AndroidView vs UIViewRepresentable).

JUnit + Compose UI: token-gated fallback — blank R.string.mapbox_access_token → LSGlassPanelFallback rendered; no MapView created; no crash.

§07

Dark Android

Dark map on Android (Story 7 — Dark Style) uses map.style.dark loaded via mapView.getMapboxMap().loadStyleUri(). The MapView instance is preserved across theme changes — the AndroidView composable's update lambda calls loadStyleUri() in-place when isSystemInDarkTheme() changes, mirroring the iOS no-unmount guarantee. The design-system mock renders with background #1B140E and contour rgba(242,238,232,0.16). Glass chrome chips use rgba(242,238,232,0.12) border. Android punch-hole camera remains dark.

Route stroke tokens are visually identical to iOS dark (Story 7 parity). Annotation dot borders flip to ink-700 identical to iOS. QA visual diff confirms the only permitted difference is the Android punch-hole vs iOS Dynamic Island chrome.
