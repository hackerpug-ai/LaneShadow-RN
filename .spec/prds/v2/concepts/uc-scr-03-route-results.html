§01 · Purpose

A recommendation with receipts.

RouteResultsScreen is where the Navigator argues for its answer. The best route is drawn in copper — bold, 3.5px stroke, slight drop shadow. Alternatives render dashed in sage (alt1) and slate (alt2) so the eye can triangulate without the map feeling like a ribbon chart. The camera auto-fits all three polylines with spacing.4 padding so the union is always visible.

The NavigatorMessage carries three responsibilities in one overlay: (1) state the recommendation in Newsreader prose, (2) attach three compact route cards with the first pre-selected, and (3) expose pin + dismiss actions so the rider can park the message or close it entirely. When pinned, the auto-dismiss recipe does not fire.

The chat input stays available but switches its placeholder to a refine-prompt — "Refine — 'make it shorter' / 'avoid Hwy 1'". This is the loop's closing move: the rider can accept an attachment (→ UC-SCR-04 RouteDetails), refine via chat (→ UC-SCR-02 PlanningScreen), or discard everything and ask again.

§06 · Rationale

Why the best is selected, not chosen.

Design principles honored

The best route is pre-selected, not pre-committed. The rider still taps to advance to details — this is an opinion the Navigator offers, not a decision it makes. The copper stroke is bold to read at distance; the sage and slate alts are dashed so they don't visually compete for primacy.

Three cards, not five, not ten. A motorcycle rider planning a ride doesn't want a list — they want a recommendation and two honest alternatives. If the pipeline produces only two viable candidates, the message says so directly instead of padding.

  • AC-02 iOS + Android identical.
