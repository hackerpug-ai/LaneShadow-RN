§01 · Purpose

Three atoms. Zero interaction semantics.

These three atoms are display-only primitives. LSAvatar shows a rider identity — either a photo or initials fallback. LSDivider separates content sections with a 1px color.border.subtle hairline. LSSpinner communicates loading state in the system's copper accent color.

None of these atoms owns interaction. Tap callbacks, selection state, and press effects are handled by organisms that compose them. The divider does not receive focus. The spinner does not animate to completion — it animates indefinitely until the consuming organism removes it.

§03 · States / Interactions

Display-only. No press states.

LSAvatar has one state: rendered. Tap interaction is layered by consuming organisms (profile card tap → navigation). The avatar itself does not change appearance on press. The initials fallback activates when the image URL fails to load or no URL is provided.

LSDivider is purely decorative structure. It has no interactive states, no focus, no press handling. Accessibility: it is marked as role="separator" on both platforms so screen readers identify it as a structural separator, not focusable content.

LSSpinner animates indefinitely while visible. It is removed from the tree (not hidden) when the loading state resolves — this prevents GPU resources from being wasted on an invisible animation. Accessibility: set accessibilityLabel = "Loading" on the containing element, not the spinner itself.

§05 · Motion

Spinner only. 700ms linear loop.

Only LSSpinner animates. It uses a CSS border rotation at 700ms linear infinite. The animation property is transform: rotate(360deg) — always hardware-accelerated. No easing curves.

The track ring is signal.tint (#F9D5B5, copper-200) at full opacity. The active arc is signal.default (#EE7C2B, copper-500) as the border-top-color. This matches the platform convention for activity indicators — iOS UIActivityIndicatorView uses this pattern natively.

LSAvatar and LSDivider have no motion. Fade-in of avatar images is handled by the image loader, not the atom.

§06 · Rationale

Three atoms, shared token patterns.

Acceptance Criteria Coverage

  • AC-1 — LSAvatar(.image, .md) shows circular image at sizing.icon.md equivalent (40px). Initials fall back to ui.label.md on surface.card.
  • AC-2 — LSDivider() renders 1px horizontal in color.border.subtle (#EDE7E1).
  • AC-3 — LSSpinner() renders with live animation tinted color.signal.default (#EE7C2B).
  • AC-4 — Sandbox "Atoms / Display" story family contains stories for each atom on both platforms.

Design Rationale

These three atoms are grouped together because they share a behavioral property: zero interaction semantics. Icon, Badge, and Chip are split into separate UCs (ATM-10, ATM-07, ATM-06) because those atoms do carry variant/state logic that warrants their own spec surface.

The spinner uses copper signal color deliberately — it is the system's primary affordance color. Loading state is not a neutral event in the Navigator context; it means the system is working for the rider. Copper communicates agency, not just busyness.

Avatar initials use signal.whisper (#FCE8D4) background with signal.tint (#F9D5B5) border — a warm paper palette consistent with the system rather than a random color-per-user approach.

§07 · Dark Mode

Avatar, Divider, Spinner on ink-800.

LSAvatar resolves surface.card → ink-700 for the circle background and border.subtle for the ring. The initials copper tint (signal.whisper → copper-100) is unchanged — it reads warmly on the dark base. LSDivider switches from paper-300 to rgba(242,238,232,0.07). LSSpinner keeps its copper arc — signal colors are mode-invariant.
