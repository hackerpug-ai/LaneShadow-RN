§01 · Purpose

One atom. Thirteen variants across three families.

LSText is the single typed text primitive. Downstream code never reaches for font-size directly — it selects a variant token: .opinion.xl, .ui.body.md, .instrument.lg. The atom resolves the font family, size, weight, line-height, letter-spacing, and tabular-num feature setting.

Color is a separate ContentColor enum resolving from color.content.*. Raw color literals are rejected at type-check. Dynamic Type on iOS and font-scale on Android scale all instances proportionally when the user adjusts system font size.

§03 · States / Interactions

Accessibility, truncation, italic.

LSText is a display atom with no interactive states. Its state surface is limited to: normal rendering, truncated (single-line with ellipsis, or multi-line clamped to maxLines), and highlighted (when a consumer wraps it inside a selection context).

Italic is available on the opinion family only (Newsreader is italic-capable at opsz 6–72). Geist and JetBrains Mono do not use italic in the Copper system — weight contrast carries hierarchy instead.

Dynamic Type scaling on iOS and font-scale on Android multiply every font-size token by the system scale factor. Components using LSText must not clip at larger scales — use flexible heights and scroll containers.

§05 · Motion

Text is static.

LSText has no intrinsic animation. Motion is applied by composing organisms — e.g., the Navigator message fade-in or the route title slide-up on sheet expansion. The text atom itself does not own motion tokens. Downstream molecules own transition timing.

§06 · Rationale

Why three families, not one?

Acceptance Criteria Coverage

  • AC-1 — opinion.xl rendered in Newsreader at 30px / 1.12lh on iOS and Android identically.
  • AC-2 — instrument.lg rendered in JetBrains Mono at 18px / tnum on both platforms.
  • AC-3 — Full cross-family matrix visible in this mock; story parity required per AC-3/4.
  • AC-5 — Dynamic Type / font-scale preview validates proportional scaling.
  • AC-6 — ContentColor enum demonstrated across all seven color roles.
  • AC-7 — No raw color literals appear in any LSText usage.

Design Rationale

Three families serve three cognitive modes in the riding experience: Opinion (Newsreader) voices the Navigator — editorial, persuasive, human. UI (Geist) labels controls — neutral, legible, efficient. Instrument (JetBrains Mono) carries numbers — tabular-aligned, scannable mid-glance. Mixing them in a single surface creates hierarchy without requiring size jumps.

The italic capability of Newsreader is intentional. Navigator voice phrases use italic for emphasis ("Where are we riding today?"). This is a semantic choice, not a stylistic decoration — italic Newsreader reads as the Navigator's tone of voice.
