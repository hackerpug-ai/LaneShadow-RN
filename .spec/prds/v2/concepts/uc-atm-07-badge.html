§01 · Purpose

Status at a glance, weather in the same breath.

LSBadge renders context-sensitive metadata in a pill shape. The status group (info, success, warning, error, recording) communicates system state using color pairs from color.status.*. The weather group (clear, rain, wind, storm, hot, cold) communicates conditions using color.weather.*.tint backgrounds with color.weather.*.default foregrounds.

LSBestBadge is a distinct atom — not a variant of LSBadge. It carries a unique semantic role: "the Navigator's recommendation." Its copper fill and white text are reserved exclusively for this meaning. The star icon reinforces the premium signal without requiring text to explain it.

§06 · Rationale

Two type systems under one interface.

Design Intent

Status and weather are unified under BadgeVariant because both communicate contextual metadata in the same shape and at the same size. The type system prevents mixing — a developer cannot accidentally pass a raw color or invent a new variant. The full set is exhaustive: every badge in the Navigator UI resolves to exactly one of the eleven variants defined here.

LSBestBadge is kept separate because its semantic weight is categorically different. It is the Navigator's opinion — the copper fill is reserved for this meaning. If it were just another variant of LSBadge, future developers might use the copper fill for other purposes, diluting the signal.

AC Traceability

  • AC-1 LSBadge(count:3, variant:.status.recording) — correct bg + fg + rec dot
  • AC-2 LSBadge(label:"Rain 3pm", variant:.weather.rain) — tint bg + rain.default fg + 0.5px border
  • AC-3 LSBadge(label:"18mph NW", variant:.weather.wind) — wind colors + .wind icon
  • AC-4 LSBestBadge() — signal.default bg + signal.on text + filled star
  • AC-5 Story: one story per status variant + weather variant + LSBestBadge
  • AC-6 All 5 status + 6 weather + 1 best = 12 badge variants documented
  • AC-7 Weather border = 0.5px at 55% alpha of weather.*.default
