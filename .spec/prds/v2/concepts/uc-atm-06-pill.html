§01 · Purpose

The shape that everything inherits.

LSPill is a pure container — it provides the pill shape (radius.pill = 999px), the height constraint (sizing.pill.sm/md/lg), and the horizontal padding defaults. It carries no color, no border, no shadow. Those are the responsibility of the molecules that consume it: LSBadge, FilterChip, WeatherPill, SuggestionChip.

Not interactive by itself. Semantic interaction — filter toggling, suggestion tap, weather readout — is layered by molecules in UC-MOL-05. The atom only guarantees the bounding shape.

§06 · Rationale

Shape separation from meaning.

Design Intent

Separating the pill shape from its semantic coloring prevents the proliferation of nearly-identical components. Badge, FilterChip, SuggestionChip, and WeatherPill all have different colors, borders, and behaviors — but they all have the same geometry. LSPill is that geometry, extracted once.

Three sizes are defined because the three usage contexts have different information densities: sm for compact metadata badges (recording count, status dot), md for filter chips in horizontal scrollers, lg for primary action pills in route detail drawers.

AC Traceability

  • AC-1 LSPill(size: .md) renders at token-specified height: 32px
  • AC-2 Custom padding .spacing3 resolves from token, not raw dp/pt
  • AC-3 Story family: Small / Medium / Large / Icon+Label / Icon Only
  • AC-4 iOS + Android: pill height matches sizing.pill.{size} exactly
  • AC-5 No default background — inherits from consumer context
  • AC-6 radius.pill = 999px applied; not a computed value
  • AC-7 Icon gap = 5px (spacing.icon-gap token)
