§01 · Purpose

One layout, zero repetition.

LSMapLayer is the foundational canvas organism used on every Navigator screen. It takes an LSMap atom and a set of named slots, then handles all z-order stacking, safe-area padding, and overlay positioning so screens never re-implement layout geometry. It is the single source of truth for: map → scrim → overlays → sheet → drawer → top bar.

Each slot is optional. Screens that only need a map and a top bar pass just those two. Screens that need a full drawer-over-scrim-over-map composition pass all slots. The organism sorts them correctly regardless.

§06 · Rationale

One organism, six screens.

Design Decision

Rather than each screen managing its own overlay stacking, LSMapLayer centralizes the z-order contract. This prevents drift — if one screen miscalculates a z-index, all screens suffer. With one organism, a single fix propagates everywhere.

Safe-area padding is similarly owned here. Screens pass content; LSMapLayer applies correct insets per platform safe area.
