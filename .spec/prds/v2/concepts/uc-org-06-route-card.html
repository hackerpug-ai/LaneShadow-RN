§01 · Purpose

Domain composition.

LSRouteCard is the full route-card organism used in catalog views, search results, and discovery surfaces. Unlike LSRouteAttachmentCard (compact, UC-MOL-08), this card features an LSMap preview that renders the route polyline auto-framed with cameraFit: .polyline(padding: .spacing3), start/end annotations, and a full info row below.

The card's data model mirrors the routes read type in server/convex/schema.ts — it never fetches independently.

§06 · Rationale

Map first, data second.

Design Decision

The map preview occupies the top 160pt of the card — more than half — because the route's shape and terrain are more legible as a visual preview than as stats. A rider scanning a catalog can immediately see whether a route winds through hills or follows a flat coastal highway.

The saved state uses LSIcon(.heartFill) rather than a bookmark because hearts communicate emotional connection — a saved ride is one the rider loved or wants to revisit, not just a to-do item.

Difficulty tags use tinted fills (green/amber/red) so cognitive scanning is pre-attentive. A rider can filter by difficulty without reading labels.
