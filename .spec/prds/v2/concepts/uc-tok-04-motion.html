§01

Purpose

Motion in Navigator communicates hierarchy and causality. An overlay that enters too slowly feels sluggish; one that exits too quickly feels abrupt. The recipe system prevents engineers from choosing ad-hoc durations and ensures that every animation in every screen matches the design-approved choreography. Primitives (durations, easings) exist to compose recipes — consumers never reference primitives directly.

Gradient tokens are retained in this group because they are dynamic color sequences — closer to motion than static color. They are consumed by backgrounds and overlays that animate with the copper accent.

§03

States & variants

Reduced motion: when the platform reports prefers-reduced-motion, all recipe durations collapse to motion.duration.instant (0ms) and opacity transitions are preserved but transform translations are zeroed. Recipes define their own reduced-motion fallback in the recipe schema.

Loop recipes (sketchPolylineLoop, phaseDotPulse) are explicitly marked loop: true and must be cancelled when the component unmounts. Non-loop recipes run once and resolve.

§05

Motion reference

This document is the canonical motion reference for V2 Navigator. All primitives (duration + easing) and all named recipes are enumerated in §02. Downstream component files must import recipes by token name — no inline animation values.

§06

Rationale & references

Design decisions

The two-level architecture (primitives + recipes) ensures that changing a single duration primitive automatically updates every recipe that references it. This is critical for accessibility — when the designer bumps motion.duration.standard from 240ms to 300ms, all screen transitions update without hunting through component files.

The sketchPolylineLoop recipe uses a leading-dot pulse rather than a shimmer because it reads better on the paper-map aesthetic — it evokes a pencil drawing a route, not a loading skeleton.

PRD acceptance criteria

  • AC-01 motion.duration section: instant/fast/standard/slow/deliberate
  • AC-02 motion.easing section: 4 cubic-bezier values
  • AC-03 motion.recipe section: 8 recipes with duration, easing, transform payload
  • AC-04 motion.recipe items validate against shared recipe schema
  • AC-05 Sprint 2+ organisms reference recipes by name, not primitives
  • AC-06 gradient section with 3 named gradients as color-token arrays
  • AC-07 pnpm tokens:validate exits zero errors
§07

Dark surfaces

Motion tokens carry no color — durations, easing curves, and recipe payloads are identical in both modes. The dark showcase confirms that animation demo cells, the easing curve SVGs, and the recipe table remain legible on warm-ink backgrounds. Copper polylines and phase dots pop distinctly against the dark surface.
