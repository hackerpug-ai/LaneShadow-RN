§01 · Purpose

Three domain patterns, defined once.

These three molecules encode Navigator-specific domain structure so that organisms (PlanningScreen, RouteDetailsScreen) do not re-implement it. The phase indicator translates planning state to an animated vertical list. The weather timeline encodes the color-coded time-of-day forecast in a compact grid. The instrument readout aligns numeric metrics in the tabular mono family.

All three compose exclusively from UC-ATM-07, UC-ATM-08, UC-ATM-10 atoms — no raw typography or layout primitives with literal colors are permitted.

§06 · Rationale

Design decisions.

Mono type for numbers

The instrument readout uses JetBrains Mono at t-instr-lg (18pt) for values — tabular-numeral alignment ensures that scanning a column of numbers at a glance is accurate. Labels in t-label-sm (8.5pt) uppercase Geist give structure without competing with the values. Opinion italic for route names in the phase header maintains the three-family typographic identity in a single component.

  • phaseDotPulse 900ms — slower than UI micro-animations; signals "the system is working"
  • wx-cell tint+border — each condition is perceptually distinct even in partial color blindness
  • instr-cell border-right — visual separator uses border.subtle, not a gap, so values align in a continuous band

AC Traceability — 06-uc-mol.md §UC-MOL-07

  • LSPhaseIndicator renders compass chip + opinion-italic header + 5-step list with LSPhaseDot per step
  • Active step shows phaseDotPulse ring animation; labels in mono type
  • LSWeatherTimeline renders 6 cells per-condition tinted bg, weather icons, temperatures in mono
  • LSInstrumentReadout renders 4-column grid (N-column when count!=4) with top/bottom LSDividers
  • Stories: PhaseIndicator: Default, All Done, All Pending; WeatherTimeline: 6 Hours, Mixed, All Clear; InstrumentReadout: 4 Metrics, 3 Metrics, Long Values
  • Tests verify composition from atoms only; raw typography with literal colors fails inspection
