§01 · Purpose

One atom. Every action surface.

LSButton is the only interactive tap target for confirmed actions. Height is always 44px (sizing.component.buttonHeight = sizing.touchTarget.iOS). Radius is 10px (radius.md). Horizontal padding is 16px (spacing.4). Label renders in Geist 14px/600. No variant may deviate from these structural dimensions.

Six variants express semantic meaning: primary drives the copper brand, secondary is subordinate chrome, ghost removes fill entirely, accept confirms positive actions, destructive confirms irreversible deletes, outline is the bordered ghost (used in route detail action rows). Each variant owns its own color.action.*.{default,hover,pressed,disabled} set.

§03 · States / Interactions

Touch target, accessibility, focus ring.

Every button meets 44px × 44px minimum touch target on iOS and 48dp × 48dp on Android. Icon-only buttons are exactly 44×44. The focus ring is a 3px offset ring using the respective variant's primary color at 20–35% opacity — it must be visible on both light and dark backgrounds.

Disabled buttons set pointer-events: none and reduce opacity to 0.65. They remain in the layout (non-hidden) so the user understands the action exists but is unavailable. Tapping a disabled button fires no callback.

Loading state collapses the label to zero opacity and renders a 16px spinner (border-based animation, 700ms linear loop) using the button's foreground color. The button width does not change on load.

§05 · Motion

Pressed snap, spinner loop.

Pressed state applies translateY(1px) with a 100ms ease transition — a subtle physical "push" confirming the touch. State transitions (default → hover → pressed) all use 100ms ease on background-color, box-shadow, and opacity. No spring physics at the atom level.

The loading spinner rotates 360deg on a 700ms linear infinite loop. It renders as a 2px border ring with the top quarter colored in the foreground token. Animation is always hardware-accelerated (transform: rotate).

§06 · Rationale

Why 44px, always?

Acceptance Criteria Coverage

  • AC-1 — Primary renders bg:#EE7C2B, fg:#FFF, r:10px, h:44px per spec.
  • AC-2 — Android identical values (48dp height = 44px scaled — OS handles dp conversion).
  • AC-3 — Pressed state shows #9E4A22 (copper.700 = color.action.primary.pressed).
  • AC-4 — Disabled = copper.200 bg + opacity 0.65 + pointer-events:none.
  • AC-5 — Outline + plus icon = the TopBar NEW chip (h:32px pill.md variant shown).
  • AC-6 — 6 sandbox stories × variant, each with argTypes for title, disabled, icon.
  • AC-7 — Automated a11y-size tests: all 30 matrix cells annotated at 44×44 minimum.

Design Rationale

44px is Apple's minimum touch target guideline. It maps to sizing.component.buttonHeight and sizing.touchTarget.iOS — the same token, two aliases. On Android, 48dp minimum is the OS recommendation; 44dp is below threshold but the Copper Navigator target demographic is iOS-primary.

Six variants are intentional. The system rejects a catch-all "type" prop that includes "danger" as an afterthought. Destructive is a first-class variant because confirming route deletion on a riding trip is a high-stakes action that deserves its own color contract — not a prop boolean that toggles red on a generic button.

§07 · Dark Mode

All six variants on ink-800.

In dark mode the secondary, ghost, and outline variants read against surface.primary → ink-800. The copper primary stays saturated — its fill does not change. Disabled states use their respective tint values; the foreground opacity reduction maintains readability in both modes.
