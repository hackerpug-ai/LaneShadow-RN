BACKGROUND
--------------------------------------------------------------------------------

**Problem:** UC-QUAL-01 (three-stage deduplication) depends on a Convex Native Geospatial index for efficient proximity queries (nearest-neighbor with `maxDistance`, rectangular range). The PRD specifies `@convex-dev/geospatial` (Beta component, tested to 1M points), rejecting PostGIS and ML-based approaches. We don't know yet whether the Beta package integrates cleanly, whether nearest-neighbor latency is acceptable, or whether our existing `curated_routes` schema accommodates the required `location` field shape.

**Why it matters:** If the Beta package has integration issues, poor latency, or requires schema refactoring that breaks existing mobile queries, the dedup architecture (AD-001, AD-005, AD-010) needs to pivot. This validation is the Week 0 gate for the dedup implementation.

**Current state:** `convex/schema.ts` has a `curated_routes` table with `centroidLat`/`centroidLng` floats but no GeoJSON Point field. `@convex-dev/geospatial` is not installed. No geospatial queries exist.

**Desired state:** Package installed, `convex/geospatialIndex.ts` defines the index, `convex/schema.ts` has an optional `location` field added non-breaking, a seed mutation populates 100 synthetic test routes across 5+ US states, and a validation mutation confirms both query types work under 500ms.

Example implementation pattern:
```typescript
// convex/geospatialIndex.ts

import { GeospatialIndex } from "@convex-dev/geospatial";

export const geospatial = new GeospatialIndex(components.curated_routes, {
```

```typescript
// convex/schema.ts (modified)
curated_routes: defineTable({
  // ... existing fields unchanged
  location: v.optional(v.object({
    type: v.literal("Point"),
    coordinates: v.array(v.number()),  // [lng, lat] per GeoJSON
  })),
})
```

```typescript
// convex/seedGeospatialTest.ts
// VALIDATION ONLY — remove before production migration
import { internalMutation } from "./_generated/server";
import { v } from "convex/values";

const TEST_ROUTES = [
  { name: "Test Route TN-01", state: "TN", lng: -86.78, lat: 36.17 },
  // ... 99 more spanning 5+ states
];

export const seedRoutes = internalMutation({
  args: {},
  handler: async (ctx) => {
    for (const r of TEST_ROUTES) {
      await ctx.db.insert("curated_routes", {
        name: r.name,
        state: r.state,
        source: "validation_only",
        centroidLat: r.lat,
        centroidLng: r.lng,
        location: { type: "Point", coordinates: [r.lng, r.lat] },
      });
    }
    return { inserted: TEST_ROUTES.length };
  },
});
```

```typescript
// convex/geospatialValidation.ts
import { internalQuery } from "./_generated/server";
import { geospatial } from "./geospatialIndex";

export const validateNearestNeighbor = internalQuery({
  args: {},
  handler: async (ctx) => {
    const start = Date.now();
    const results = await geospatial.nearest({
      point: { lat: 36.17, lng: -86.78 },
      limit: 10,
    });
    const latency_ms = Date.now() - start;
    const status = results.length >= 1 && latency_ms < 500 ? "PASS" : "FAIL";
    return { status, latency_ms, count: results.length };
  },
});
```
