SPECIFICATION
--------------------------------------------------------------------------------

**Objective:** Add a production-grade Scenic Byways GIS ingest path that crawls and validates the GIS source, reconciles it with FHWA, writes staging output with upgraded geometry and metadata, and preserves that provenance through Convex and the existing mobile route detail flow.

**Success looks like:** Scenic Byways data lands in staging and Convex with `location`, designation, description, and source provenance intact; FHWA overlap is merged into a single discoverable route entry; and the existing mobile detail experience can render Scenic Byways provenance without inventing a new UX surface.

--------------------------------------------------------------------------------
BACKGROUND
--------------------------------------------------------------------------------

**Problem:** Epic 2 established the FHWA baseline, but the current repo has no Scenic Byways module, no Scenic Byways crawl-plan artifacts, and no guaranteed route from new source provenance into the existing mobile detail path.

**Why it matters:** Scenic Byways is the government-grade geometry and metadata upgrade in Epic 4. If it duplicates FHWA or drops provenance before mobile, the source adds technical debt instead of improving catalog quality.

**Current state:** `scripts/curation/pipeline/sources/` has `fhwa.py`, `motorcycleroads.py`, and `bestbikingroads.py` but no Scenic Byways module. Convex/mobile contracts already support many optional provenance fields, but `convex/db/curation.ts`, `lib/discovery/sync-lean.ts`, and `lib/discovery/db.ts` still sync only card-level fields.

**Desired state:** Scenic Byways runs end-to-end through crawl-plan validation, ingestion, reconciliation, staging, Convex serialization, and existing mobile detail rendering.
