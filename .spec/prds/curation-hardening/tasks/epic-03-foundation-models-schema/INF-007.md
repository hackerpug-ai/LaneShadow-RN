BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The `convex_push.py` serialization function (`_route_to_dict`) must serialize all new Route / EnrichedRoute fields — especially `searchEmbedding` (1536 floats), `searchText`, `candidateIdentifiers`, `matchConfidence`, `llmReconciliationLog`, and the enrichment output fields. The snake_case → camelCase mapping must be correct. The batch size must be reduced from 50 to 10 because 1536-float embeddings make the payload ~30x larger per route.

**Why it matters:** Without serialization updates, new Route fields are silently dropped during push to Convex. INF-004 (embedding backfill) depends on this task — if `searchEmbedding` doesn't serialize, the backfill runs, reports success, and nothing lands. The payload-size problem is critical: 50 routes × 1536 floats × 8 bytes ≈ 600 KB per batch, which is fine for HTTP but hurts Convex ingestion latency.

**Current state:** `_route_to_dict()` in `scripts/curation/pipeline/sync/convex_push.py` converts Route/EnrichedRoute to camelCase JSON and pushes via POST /api/ingest-routes in batches of 50. Uses `hasattr()` for enriched fields. Serializes the original ~12 Route fields + 6 EnrichedRoute scores. No awareness of embedding, identifiers, LLM artifacts, or enrichment outputs.

**Desired state:** `_route_to_dict()` serializes all new fields with proper camelCase mapping. Batch size drops from 50 to 10 (via a `DEFAULT_BATCH_SIZE` constant so it's easy to tune). Existing Route objects without new fields still serialize correctly (backward-compat via `hasattr` + `is not None` guards).

**Architectural note:** The previous INF-006 plan (in the old Epic 3) was a similar "serialization update" task — same file, different field list. This replaces that entire plan with the semantic-matching field set.

### Batch size change

```python
DEFAULT_BATCH_SIZE: int = 10    # reduced from 50 for embedding payload size
```

All references to `50` in the file should be replaced with `DEFAULT_BATCH_SIZE`.

### Payload size sanity

- Old: 50 routes × ~2 KB = ~100 KB per POST
- New: 10 routes × ~15 KB (of which 12 KB is the embedding) = ~150 KB per POST
- Still well under typical HTTP limits; the lower batch size reduces per-POST latency and makes retries cheaper

def _route_to_dict(route: Route) -> dict:
    d: dict = {
        "routeId": route.route_id,
        "name": route.name,
        "state": route.state,
        "source": route.source,
        "centroidLat": route.centroid_lat,

    if hasattr(route, "embedding") and route.embedding is not None:

    _maybe_set(d, route, "source_count", "sourceCount")
    _maybe_set(d, route, "quality_tier", "qualityTier")

    return d


def _maybe_set(d: dict, obj, attr: str, camel_key: str) -> None:
    """Helper: set d[camel_key] = obj.attr if the attribute exists and is not None/empty."""
    if not hasattr(obj, attr):
        return
    val = getattr(obj, attr)
    if val is None:
        return
    # For list/dict, omit if empty to reduce payload
    if isinstance(val, (list, dict)) and not val:
        return
    d[camel_key] = val
