BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The Convex schema needs 18+ new optional fields on curated_routes and a brand-new route_mentions table for social source attribution. All new fields must be v.optional() for non-breaking migration.

**Why it matters:** The mobile app has existing users. Any required field added to curated_routes will cause runtime crashes on older client versions. The route_mentions table is needed by Epic 9/10 community NLP pipeline.

**Current state:** convex/schema.ts has curated_routes with curatedRouteValidator from models/curated-routes.ts. Existing fields include name, state, source, centroid coords, 6 scores, archetype, content fields. Location (GeoJSON Point) already exists. No route_mentions table.

**Desired state:** curatedRouteValidator has 18+ new v.optional() fields. route_mentions table exists with routeMentionValidator and 3 indexes. by_quality_tier index added to curated_routes.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------

MUST: Add ALL new optional fields to curated_routes validator
MUST: Create route_mentions table with routeMentionValidator
MUST: Add by_quality_tier index to curated_routes

NEVER: Make any new field required (non-optional) — mobile clients will break
NEVER: Remove or rename existing curated_routes fields or indexes

New route_mentions table pattern:
```typescript
export const routeMentionValidator = v.object({
  roadName: v.string(),
  highwayNumber: v.optional(v.string()),
  state: v.optional(v.string()),
  sentimentScore: v.number(),
  sentimentLabel: v.string(),
  aspectScores: v.record(v.number()),
  attributes: v.record(v.boolean()),
  confidence: v.number(),
  source: v.string(),
  sourceAuthority: v.number(),
  postUrl: v.string(),
  postScore: v.number(),
  mentionDate: v.number(),
  processedAt: v.number(),
  routeId: v.optional(v.id("curated_routes")),
  extractionModel: v.string(),
  extractionCost: v.number(),
});
```

Anti-pattern: Do NOT use `v.nullish()` — it allows both null and undefined, which can cause unexpected behavior in Convex. Use `v.optional()` instead.
