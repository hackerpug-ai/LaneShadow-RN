BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The extraction schema needs a full rewrite for semantic matching. The previous plan just added an `extraction_confidence` field to an existing `RouteAttributes` Pydantic model. The new plan replaces that with a comprehensive `PostExtraction` schema that captures everything a single LLM call extracts from a community post: road/highway/state/landmark mentions, sentiment, aspect scores, attributes, warnings, confidence, model, and cost. This becomes the contract between Epic 9 (ingestion) and the rest of the pipeline.

**Why it matters:** This is the data contract for every LLM call the downstream pipeline makes. It must be stable, structured, and cheap to validate. It replaces three things from the old plan: (1) per-attribute rapidfuzz extraction, (2) a separate sentiment classifier, (3) a separate aspect scorer. All three become one LLM call returning a `PostExtraction`.

**Current state:** `scripts/curation/pipeline/extraction/schema.py` has `EXTRACTION_SCHEMA_VERSION = 1` and a `RouteAttributes` Pydantic BaseModel with 8 score fields, Season enum, RoadSurface enum, `primary_archetype_hint`. No `PostExtraction`, no road_name_mentions, no highway_refs, no landmark_refs.

**Desired state:** `EXTRACTION_SCHEMA_VERSION = 2`. New `PostExtraction` Pydantic BaseModel with fields for mentions, sentiment, aspects, attributes, warnings, and extraction metadata. `RouteAttributes` remains for backward compatibility (read path) but new extraction calls use `PostExtraction`. A caching policy constant is defined. Extraction cost budget documented in module docstring.

**Architectural note:** `PostExtraction` is shaped so it can be passed directly to `openai.chat.completions.create(response_format=PostExtraction)` (OpenAI Structured Outputs) or equivalently to Anthropic's tool-use / structured output. Downstream code in Epic 9/10 will invoke this.

### PostExtraction field contract

```python
class PostExtraction(BaseModel):
    """
    Structured output from a single LLM call over a community post.

    Produced once per raw post (Epic 9 ingestion) and stored in Convex
    `route_posts_raw.payload`. Downstream matching (Epic 6), enrichment (Epic 10),
    and reconciliation read from this shape.
    """

    # ------------------------------------------------------------------------
    # Identifier mentions — what roads/places did the post talk about?
    # ------------------------------------------------------------------------
    road_name_mentions: list[str] = Field(
        default_factory=list,
        description="Road/route names extracted from the post, including nicknames. "
                    "Examples: ['Tail of the Dragon', 'The Dragon', 'Deals Gap']",
    )
    highway_refs: list[str] = Field(
        default_factory=list,
        description="Highway number references normalized to a canonical form. "
                    "Examples: ['US-129', 'I-40', 'SR-28']",
    )
    state_refs: list[str] = Field(
        default_factory=list,
        description="State references (two-letter codes or full names). Examples: ['TN', 'Tennessee']",
    )
    landmark_refs: list[str] = Field(
        default_factory=list,
        description="Nearby landmarks, towns, rivers, or geographic anchors. "
                    "Examples: ['Chattanooga', 'Great Smoky Mountains', 'Fontana Dam']",
    )

    # ------------------------------------------------------------------------
    # Sentiment & aspect scoring
    # ------------------------------------------------------------------------
    sentiment: Literal["positive", "neutral", "negative"] = Field(
        ...,
        description="Overall sentiment of the post toward the road(s) mentioned",
    )
    aspect_scores: dict[str, float] = Field(
        default_factory=dict,
        description="Per-aspect scores in the range [0.0, 1.0]. "
                    "Known aspects: curvature, scenery, traffic, surface_quality, elevation_drama. "
                    "Unknown aspects are allowed but will be ignored by scoring.",
    )
    attributes: dict[str, bool] = Field(
        default_factory=dict,
        description="Boolean attribute flags. "
                    "Known attributes: has_gas, has_food, wet_weather_ok, beginner_friendly, "
                    "requires_adv_bike, closed_in_winter.",
    )
    warnings: list[str] = Field(
        default_factory=list,
        description="Caveats or warnings the post raised (construction, closures, hazards, etc.)",
    )

    # ------------------------------------------------------------------------
    # Extraction metadata (populated by the extraction client, not the LLM)
    # ------------------------------------------------------------------------
    extraction_confidence: float = Field(
        ..., ge=0.0, le=1.0,
        description="Model-reported confidence in the extraction (0.0 = guess, 1.0 = high confidence)",
    )
    extraction_model: str = Field(
        ...,
        description="Model identifier, e.g. 'claude-haiku-4-5-20251001'",
    )
    extraction_cost: float = Field(
        ..., ge=0.0,
        description="USD cost of this extraction call (input + output tokens)",
    )
    extracted_at: datetime = Field(
        default_factory=lambda: datetime.now(timezone.utc),
        description="Timestamp of the extraction call",
    )
    extraction_schema_version: int = Field(
        default=EXTRACTION_SCHEMA_VERSION,
        description="Schema version the extractor was told to produce",
    )
```

```python
"""
Extraction schema for LLM-driven community post analysis.

Version 2 (Epic 3 — INF-005):
- Adds PostExtraction: canonical structured output for a single LLM extraction call.
- Replaces the per-attribute rapidfuzz extraction plan with a single LLM call returning
  a PostExtraction instance.
- Preserves RouteAttributes for backward compatibility with v1 data.
- Adds CACHE_POLICY for (post_id, schema_version) caching.

Cost model (reference only, not enforced here):
- Claude Haiku 4.5: ~$0.001 per extraction (600 input + 300 output tokens typical)
- 100k posts -> ~$100 one-time for full backfill (Epic 9/10 budget)
"""

from __future__ import annotations

from datetime import datetime, timezone
from typing import Literal, Optional

from pydantic import BaseModel, Field

EXTRACTION_SCHEMA_VERSION: int = 2

DEFAULT_EXTRACTION_MODEL: str = "claude-haiku-4-5-20251001"

CACHE_POLICY: dict = {
    "key_format": "{post_id}|{extraction_schema_version}",
    "ttl_seconds": 30 * 24 * 60 * 60,
    "invalidate_on_version_bump": True,
}


# ============================================================================
# V2: PostExtraction — canonical LLM output contract
# ============================================================================


class PostExtraction(BaseModel):
    """
    Structured output from a single LLM call over a community post.

    Produced once per raw post (Epic 9 ingestion) and stored in Convex
    route_posts_raw.payload. Downstream matching, enrichment, and reconciliation
    read from this shape.
    """

    # Identifier mentions
    road_name_mentions: list[str] = Field(default_factory=list)
    highway_refs: list[str] = Field(default_factory=list)
    state_refs: list[str] = Field(default_factory=list)
    landmark_refs: list[str] = Field(default_factory=list)

    # Sentiment & aspects
    sentiment: Literal["positive", "neutral", "negative"] = Field(...)
    aspect_scores: dict[str, float] = Field(default_factory=dict)
    attributes: dict[str, bool] = Field(default_factory=dict)
    warnings: list[str] = Field(default_factory=list)

    # Extraction metadata
    extraction_confidence: float = Field(..., ge=0.0, le=1.0)
    extraction_model: str = Field(...)
    extraction_cost: float = Field(..., ge=0.0)
    extracted_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
    extraction_schema_version: int = Field(default=EXTRACTION_SCHEMA_VERSION)

    model_config = {"extra": "forbid"}  # strict — reject unknown fields


# ============================================================================
# V1: RouteAttributes — PRESERVED for backward compatibility
# ============================================================================
# (existing RouteAttributes / Season / RoadSurface definitions stay intact)


def is_v1_extraction(payload: dict) -> bool:
    """Return True if the payload looks like v1 RouteAttributes (not v2 PostExtraction)."""
    return "curvature_score" in payload and "road_name_mentions" not in payload
```

Anti-pattern: Do NOT use `str` for sentiment — always `Literal["positive", "neutral", "negative"]` so Pydantic and OpenAI Structured Outputs both enforce the enum.

Anti-pattern: Do NOT use `Union[PostExtraction, RouteAttributes]` as a function parameter type — instead, use `is_v1_extraction()` to detect and route.

NOTES
--------------------------------------------------------------------------------

- **The old INF-005 plan** just added an `extraction_confidence` field to RouteAttributes. The new plan adds a whole new model (PostExtraction) because the downstream pipeline needs structured mentions, sentiment, aspects, and metadata in one round-trip — not scattered across multiple extraction stages.
- **Why Literal for sentiment instead of Enum:** OpenAI Structured Outputs and Anthropic tool use both handle Literal types cleanly; Enums work but add indirection. Literal is simpler and maps directly to JSON schema `enum: ["positive", "neutral", "negative"]`.
- **aspect_scores is a dict not a fixed-field object** because we want to add new aspects (e.g., "beginner_friendliness") without schema version bumps. The known aspects are documented in the field description; unknown aspects are allowed but ignored by scoring.
- **`model_config = {"extra": "forbid"}`** — strictly rejects unknown fields. This is a mild prompt-injection defense: if an LLM tries to return extra structure ("jailbreak": true), Pydantic will reject it loudly instead of silently accepting.
- **CACHE_POLICY is a documentation constant, not an enforcement mechanism** — the actual cache implementation lives in Epic 9. This task just documents the contract so everyone agrees on the key format.
- **Boy Scout rule:** If you find the old Season or RoadSurface enum has stale values, fix them as part of this task (don't change their names, just their members if needed).
