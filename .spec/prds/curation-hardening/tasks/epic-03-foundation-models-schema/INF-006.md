BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The convex_push.py serialization function (_route_to_dict) needs to serialize all 27 new fields from the extended Route and EnrichedRoute dataclasses with proper snake_case → camelCase conversion.

**Why it matters:** Without serialization updates, new model fields will be silently dropped during push to Convex. The pipeline will appear to work but data will be incomplete.

**Current state:** _route_to_dict() converts Route/EnrichedRoute to camelCase JSON, pushes via POST /api/ingest-routes in batches of 50. Uses hasattr() for enriched fields. Only serializes the original 12 Route fields + 6 EnrichedRoute scores.

**Desired state:** _route_to_dict() serializes all 18 new Route fields + 9 new EnrichedRoute fields with camelCase conversion. Existing Route objects without new fields still serialize correctly via hasattr() pattern.
