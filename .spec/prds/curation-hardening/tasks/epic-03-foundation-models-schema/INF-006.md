BACKGROUND
--------------------------------------------------------------------------------

**Problem:** With the schema now carrying a `by_embedding` vectorIndex on `curated_routes` (INF-003), the pipeline needs production-ready Convex query and mutation wrappers for (a) semantic candidate retrieval via vector search, (b) reading and writing to `route_matches`, (c) updating individual route embeddings, and (d) reading `route_posts_raw` artifacts. These wrappers become the API that INF-004 and all Epic 6/9/10 code calls.

**Why it matters:** INF-004 (the embedding pipeline) writes via these wrappers. Epic 6 (dedup) queries `findCandidateRoutesByEmbedding` for every new post. Epic 9 (community ingestion) writes artifacts and matches. Without these functions, the semantic matching architecture is a schema with no API — nobody can use it.

**Current state:** `convex/geospatialIndex.ts` exists and handles map viewport queries via `@convex-dev/geospatial` — that code stays untouched. No file exists for vector search wrappers. `route_posts_raw` and `route_matches` tables (from INF-003) have no query/mutation wrappers.

**Desired state:** A new file `convex/semanticSearch.ts` contains:
1. `findCandidateRoutesByEmbedding` (query) — `ctx.vectorSearch("curated_routes", "by_embedding", { vector, limit, filter })`
2. `findRoutesByIdentifier` (query) — text fallback for exact/prefix matches on `candidateIdentifiers` or `highwayNumber`
3. `updateRouteEmbedding` (mutation) — write `searchText` + `searchEmbedding` to a route
4. `addRouteMatch` (mutation) — insert a row into `route_matches` with validated args
5. `getRouteMatchesForPost` (query) — read all matches for a given postId (uses `by_postId` index)
6. `getRouteMatchesForRoute` (query) — read all matches for a given routeId (uses `by_routeId_and_confidence` index, sorted desc by confidence)
7. `getRawPostsForRoute` (query) — join: find `route_matches.postId`s for a route, then fetch the corresponding `route_posts_raw` artifacts

**Architectural note:** The previously-planned INF-007 (old numbering — geospatial production wrappers like `findNearbyRoutes`, `findRoutesInBoundingBox`, `getRouteMentionsForRoute`, `addRouteMention`) is REPLACED by this task. The old geospatial wrappers can stay in `geospatialIndex.ts` for mobile map viewport queries, but they're no longer the primary matching mechanism. This task defines the semantic primitives.

### Function contracts

**1. findCandidateRoutesByEmbedding (query)**
```
args:
  embedding: v.array(v.number())              // 1536 floats
  limit: v.optional(v.number())               // default 10
  stateFilter: v.optional(v.string())         // use the vectorIndex filter
returns:
  list of { routeId, cosineSimilarity, name, state, candidateIdentifiers }
```

**2. findRoutesByIdentifier (query)**
```
args:
  identifier: v.string()                      // e.g. "Tail of the Dragon"
  stateFilter: v.optional(v.string())
  limit: v.optional(v.number())               // default 20
returns:
  list of { routeId, name, state, matchType: "name" | "highway" | "identifier" }
note:
  - Performs an exact match on `name` first
  - Falls back to matching `highwayNumber` if identifier looks like "US-129" / "SR-28"
  - Falls back to array-contains on `candidateIdentifiers`
  - Fallback logic is a fast first-pass for exact hits; semantic search is the primary matcher
```

**3. updateRouteEmbedding (mutation)**
```
args:
  routeId: v.id("curated_routes")
  searchText: v.string()
  searchEmbedding: v.array(v.number())        // must be length 1536 (enforced in handler)
returns:
  { ok: true }
note:
  - Patches the route; does not touch any other field
  - Used by INF-004 batch backfill AND incremental embed-on-upsert in Epic 9/10
```

**4. addRouteMatch (mutation)**
```
args:
  matchId: v.string()
  postId: v.string()
  routeId: v.id("curated_routes")
  matchConfidence: v.number()                 // enforce 0 <= x <= 1 in handler
  cosineSimilarity: v.number()                // enforce 0 <= x <= 1 in handler
  matchReasoning: v.string()
  rerankModel: v.string()
  rerankCost: v.number()                      // enforce >= 0 in handler
  matchedAt: v.number()
  isArbitrated: v.boolean()
  arbitrationNotes: v.optional(v.string())
returns:
  v.id("route_matches")
```

**5. getRouteMatchesForPost (query)**
```
args:
  postId: v.string()
returns:
  list of RouteMatch rows sorted by matchConfidence desc
index:
  .withIndex("by_postId", q => q.eq("postId", args.postId))
```

**6. getRouteMatchesForRoute (query)**
```
args:
  routeId: v.id("curated_routes")
  minConfidence: v.optional(v.number())       // default 0.0
  limit: v.optional(v.number())               // default 50
returns:
  list of RouteMatch rows sorted by matchConfidence desc
index:
  .withIndex("by_routeId_and_confidence", q => q.eq("routeId", args.routeId))
  .order("desc")
  .take(limit)
```

**7. getRawPostsForRoute (query)**
```
args:
  routeId: v.id("curated_routes")
  limit: v.optional(v.number())               // default 50
returns:
  list of { match: RouteMatch, post: RoutePostRaw }
implementation:
  - First query route_matches .withIndex("by_routeId_and_confidence")
  - For each match, query route_posts_raw .withIndex("by_postId")
  - Return the joined shape for UI display / reconciliation
```

```typescript
import { query, mutation } from "./_generated/server";
import { v } from "convex/values";

const EMBEDDING_DIMENSIONS = 1536;

// ============================================================================
// 1. Vector search — primary retrieval for semantic matching
// ============================================================================

export const findCandidateRoutesByEmbedding = query({
  args: {
    embedding: v.array(v.number()),
    limit: v.optional(v.number()),
    stateFilter: v.optional(v.string()),
  },
  handler: async (ctx, args) => {
    if (args.embedding.length !== EMBEDDING_DIMENSIONS) {
      throw new Error(
        `embedding must have ${EMBEDDING_DIMENSIONS} dimensions, got ${args.embedding.length}`,
      );
    }
    const results = await ctx.vectorSearch("curated_routes", "by_embedding", {
      vector: args.embedding,
      limit: args.limit ?? 10,
      filter: args.stateFilter
        ? (q) => q.eq("state", args.stateFilter!)
        : undefined,
    });
    // results is { _id, _score } — fetch full doc and join the score
    const docs = await Promise.all(
      results.map(async (r) => {
        const doc = await ctx.db.get(r._id);
        return doc && {
          routeId: doc._id,
          cosineSimilarity: r._score,
          name: doc.name,
          state: doc.state,
          candidateIdentifiers: doc.candidateIdentifiers ?? [],
        };
      }),
    );
    return docs.filter((x): x is NonNullable<typeof x> => x !== null);
  },
});

// ============================================================================
// 2. Identifier fallback — exact/prefix text match
// ============================================================================

export const findRoutesByIdentifier = query({
  args: {
    identifier: v.string(),
    stateFilter: v.optional(v.string()),
    limit: v.optional(v.number()),
  },
  handler: async (ctx, args) => {
    // Strategy:
    //   1. If identifier looks like a highway (e.g. "US-129"), try highwayNumber
    //   2. Otherwise, search by name (exact or prefix)
    //   3. Fall back to array-contains on candidateIdentifiers
    // For the MVP, use a scan with take() — for 5k routes this is acceptable.
    // In production, add a fts or secondary index.
    const limit = args.limit ?? 20;
    const rows = await ctx.db.query("curated_routes").take(limit * 5);
    const lowered = args.identifier.toLowerCase();
    const matches = rows
      .filter((r) => {
        if (args.stateFilter && r.state !== args.stateFilter) return false;
        if (r.name?.toLowerCase() === lowered) return true;
        if (r.highwayNumber?.toLowerCase() === lowered) return true;
        if (r.candidateIdentifiers?.some((id) => id.toLowerCase() === lowered)) return true;
        return false;
      })
      .slice(0, limit)
      .map((r) => ({
        routeId: r._id,
        name: r.name,
        state: r.state,
        matchType:
          r.name?.toLowerCase() === lowered
            ? ("name" as const)
            : r.highwayNumber?.toLowerCase() === lowered
            ? ("highway" as const)
            : ("identifier" as const),
      }));
    return matches;
  },
});

// ============================================================================
// 3. Update embedding (used by INF-004 backfill and Epic 9/10 incremental embed)
// ============================================================================

export const updateRouteEmbedding = mutation({
  args: {
    routeId: v.id("curated_routes"),
    searchText: v.string(),
    searchEmbedding: v.array(v.number()),
  },
  handler: async (ctx, args) => {
    if (args.searchEmbedding.length !== EMBEDDING_DIMENSIONS) {
      throw new Error(
        `searchEmbedding must have ${EMBEDDING_DIMENSIONS} dimensions, got ${args.searchEmbedding.length}`,
      );
    }
    await ctx.db.patch(args.routeId, {
      searchText: args.searchText,
      searchEmbedding: args.searchEmbedding,
    });
    return { ok: true as const };
  },
});

// ============================================================================
// 4. Add match — audit log for every (post, route) decision
// ============================================================================

export const addRouteMatch = mutation({
  args: {
    matchId: v.string(),
    postId: v.string(),
    routeId: v.id("curated_routes"),
    matchConfidence: v.number(),
    cosineSimilarity: v.number(),
    matchReasoning: v.string(),
    rerankModel: v.string(),
    rerankCost: v.number(),
    matchedAt: v.number(),
    isArbitrated: v.boolean(),
    arbitrationNotes: v.optional(v.string()),
  },
  handler: async (ctx, args) => {
    if (args.matchConfidence < 0 || args.matchConfidence > 1) {
      throw new Error(`matchConfidence out of [0,1]: ${args.matchConfidence}`);
    }
    if (args.cosineSimilarity < 0 || args.cosineSimilarity > 1) {
      throw new Error(`cosineSimilarity out of [0,1]: ${args.cosineSimilarity}`);
    }
    if (args.rerankCost < 0) {
      throw new Error(`rerankCost negative: ${args.rerankCost}`);
    }
    return await ctx.db.insert("route_matches", args);
  },
});

// ============================================================================
// 5. Read matches for a post
// ============================================================================

export const getRouteMatchesForPost = query({
  args: { postId: v.string() },
  handler: async (ctx, args) => {
    const rows = await ctx.db
      .query("route_matches")
      .withIndex("by_postId", (q) => q.eq("postId", args.postId))
      .collect();
    return rows.sort((a, b) => b.matchConfidence - a.matchConfidence);
  },
});

// ============================================================================
// 6. Read matches for a route (top-confidence first)
// ============================================================================

export const getRouteMatchesForRoute = query({
  args: {
    routeId: v.id("curated_routes"),
    minConfidence: v.optional(v.number()),
    limit: v.optional(v.number()),
  },
  handler: async (ctx, args) => {
    const limit = args.limit ?? 50;
    const minConfidence = args.minConfidence ?? 0.0;
    const rows = await ctx.db
      .query("route_matches")
      .withIndex("by_routeId_and_confidence", (q) => q.eq("routeId", args.routeId))
      .order("desc")
      .take(limit * 2); // over-fetch then filter by minConfidence
    return rows.filter((r) => r.matchConfidence >= minConfidence).slice(0, limit);
  },
});

// ============================================================================
// 7. Join: raw posts for a route
// ============================================================================

export const getRawPostsForRoute = query({
  args: {
    routeId: v.id("curated_routes"),
    limit: v.optional(v.number()),
  },
  handler: async (ctx, args) => {
    const matches = await ctx.db
      .query("route_matches")
      .withIndex("by_routeId_and_confidence", (q) => q.eq("routeId", args.routeId))
      .order("desc")
      .take(args.limit ?? 50);

    const joined = await Promise.all(
      matches.map(async (m) => {
        const post = await ctx.db
          .query("route_posts_raw")
          .withIndex("by_postId", (q) => q.eq("postId", m.postId))
          .first();
        return post ? { match: m, post } : null;
      }),
    );
    return joined.filter((x): x is NonNullable<typeof x> => x !== null);
  },
});
```

Anti-pattern: Do NOT use `ctx.db.query("route_matches").filter(q => q.eq(q.field("postId"), postId))` — always use `.withIndex("by_postId")` for indexed lookups. Filters force full-table scans.

Anti-pattern: Do NOT try to sort vectorSearch results in the handler — `ctx.vectorSearch` already returns them sorted by descending similarity.
