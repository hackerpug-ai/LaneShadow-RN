BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The Route and EnrichedRoute dataclasses need new fields for semantic matching (embedding, candidate_identifiers, search_text), LLM artifact tracking (llm_reconciliation_log, match_confidence), and enrichment outputs (description, rating, surface, aadt, elevation, etc.). Three new dataclasses are also needed at module level: `LLMExtractionArtifact`, `RouteMatch`, and `PostExtraction` — the shared contracts used by embedding (INF-004), extraction schema (INF-005), vector search wrappers (INF-006), and serialization (INF-007).

**Why it matters:** Every downstream task depends on these types. INF-003 (Convex schema) mirrors their shape. INF-004 (embedding pipeline) reads `search_text` and writes `embedding`. INF-005 (extraction contract) uses `PostExtraction`. INF-006 (vector search) writes `RouteMatch`. INF-007 (serialization) marshals them all to Convex.

**Current state:** Route has ~12 required/positional fields (route_id, name, state, source, centroid_lat/lng, length_miles, bounds_ne/sw, composite_score, curvature_score). EnrichedRoute extends Route with 6 scores + classification + content fields. No `LLMExtractionArtifact`, `RouteMatch`, or `PostExtraction` dataclasses.

**Desired state:** Route has new Optional fields (semantic matching + enrichment outputs). EnrichedRoute has 9 additional Optional scoring fields. Three new dataclasses exist at module level. All new fields default to `None` or `field(default_factory=list)` / `field(default_factory=dict)` — backward compatible.

**Architectural note:** The old INF-002 plan added `merged_at`, `merge_count`, and `field_provenance` to support deterministic merging. Those three fields are REMOVED in this rewrite — merge reasoning now lives in `llm_reconciliation_log` (a list of LLM decisions) instead.

New fields follow the same pattern (all Optional, grouped by purpose):
```python
    # ========================================================================
    # Semantic matching fields (Epic 3 — INF-002)
    # ========================================================================
    candidate_identifiers: list[str] = field(default_factory=list)
    # e.g. ["Tail of the Dragon", "The Dragon", "Deals Gap", "US-129"]

    search_text: Optional[str] = None
    # Concatenated string used to generate the embedding (built by INF-004)

    embedding: Optional[list[float]] = None
    # 1536-dim vector from text-embedding-3-small. None until INF-004 backfill

    match_confidence: Optional[float] = None
    # 0.0-1.0, confidence from most recent LLM match decision

    llm_reconciliation_log: list[dict] = field(default_factory=list)
    # List of reconciliation decision records:
    # [{"run_id": str, "reconciled_at": str, "conflicts_resolved": int, "notes": str}, ...]

    # ========================================================================
    # Enrichment output fields (populated by Epic 9/10 LLM pipeline)
    # ========================================================================
    description: Optional[str] = None
    rating: Optional[float] = None                 # 0.0-5.0
    designation: Optional[str] = None              # "National Scenic Byway", "State Route", etc.
    source_url: Optional[str] = None
    source_refs: list[str] = field(default_factory=list)
    highway_number: Optional[str] = None           # "US-129", "SR-28"
    elevation_gain_m: Optional[float] = None
    surface: Optional[str] = None                  # paved|gravel|dirt|mixed

    aadt_max: Optional[float] = None
```

New module-level dataclasses:
```python
@dataclass
class LLMExtractionArtifact:
    """Record of a single LLM extraction run for a community post."""
    artifact_id: str                    # uuid4
    post_id: str                        # upstream post identifier
    post_url: str
    source: str                         # "reddit", "advrider", "rider_magazine", etc.
    raw_text: str                       # the post text sent to the LLM
    extraction_schema_version: int      # EXTRACTION_SCHEMA_VERSION from schema.py
    extraction_model: str               # "claude-haiku-4-5-20251001", etc.
    extraction_cost: float              # USD
    extracted_at: str                   # ISO timestamp
    payload: dict = field(default_factory=dict)  # serialized PostExtraction.model_dump()
    extraction_confidence: Optional[float] = None


@dataclass
class RouteMatch:
    """Audit record for a (post → route) match decision made via vector search + LLM rerank."""
    match_id: str                       # uuid4
    post_id: str
    route_id: str
    match_confidence: float             # 0.0-1.0 from LLM rerank
    match_reasoning: str                # LLM's stated reason for the match
    cosine_similarity: float            # 0.0-1.0 from vector search
    rerank_model: str                   # "claude-haiku-4-5-20251001"
    rerank_cost: float                  # USD
    matched_at: str                     # ISO timestamp
    is_arbitrated: bool = False         # True if this match required LLM arbitration (mid-confidence)
    arbitration_notes: Optional[str] = None


# Re-export PostExtraction from extraction/schema.py for convenience
from scripts.curation.pipeline.extraction.schema import PostExtraction  # noqa: E402
```
