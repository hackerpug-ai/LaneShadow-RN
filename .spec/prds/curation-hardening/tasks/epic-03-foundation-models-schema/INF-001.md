BACKGROUND
--------------------------------------------------------------------------------

**Problem:** The curation pipeline needs semantic matching infrastructure (vector embeddings + LLM extraction) to replace the previously-planned deterministic string-matching approach. This requires new Python dependencies: OpenAI SDK (for embeddings), Anthropic SDK (for reasoning calls), plus the geometry/source libraries still needed by downstream epics.

**Why it matters:** Without these packages, Epic 3 foundation tasks (INF-002 through INF-007) cannot import the required types, INF-004 cannot generate embeddings, and Epic 4/9/10 cannot ingest data from government, editorial, or community sources. The LLM SDKs are the core shift — they enable semantic extraction, reranking, enrichment, and reconciliation downstream.

**Current state:** `scripts/curation/requirements.txt` has Epic 2 baseline dependencies (requests, python-dotenv, httpx, beautifulsoup4, playwright, anthropic, instructor, pydantic, scipy, shapely, pyproj). Missing: openai, fiona, praw, srtm.py, gpxpy, haversine. Anthropic is already present — it may need a version bump.

**Desired state:** `requirements.txt` includes all new packages with pinned versions. `pip install -r scripts/curation/requirements.txt` runs clean on a fresh virtualenv. All new packages importable in Python.

**Architectural note:** `rapidfuzz` is INTENTIONALLY EXCLUDED — it was in the previous Epic 3 plan but has been replaced by LLM-based matching (see EPIC.md "Architectural Decision" section).
