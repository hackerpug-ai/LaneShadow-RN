## Shared molecule bundle

`organisms/_molecules.css` is the bundled concatenation of every molecule's `<style>` block. Organism HTMLs link it as `../_molecules.css` so they can compose molecule classes (e.g. `.mol-bottom-sheet`, `.mol-instrument-readout`, `.mol-route-attachment-card`) without re-defining any molecule styling.

The bundle is regenerated by re-running the same extraction script used during Phase 4 setup (`python3` snippet that strips `<style>` blocks out of each `molecules/*/*.html` and concatenates them under per-molecule section headers). If a molecule's style block changes, the bundle must be re-regenerated before the change is reflected in organism previews.

## Organism-local geometry literals

The audit allowed a few structural / preview-only literals per organism that are not tokenizable; each is documented in the component's README under "Organism-local constants":

| Organism | Literal | Reason |
|---|---|---|
| topbar-navbar | `backdrop-filter: blur(8px)` | Visual-effect blur radius (lighter than molecule glass) |
| topbar-navbar | `z-index: 5` | Structural z-order contract from UC-ORG-02 |
| map-layer | `z-index: 0..5` | Seven-layer z-order contract (the organism's core purpose) |
| map-layer | `isolation: isolate` | Creates stacking context for child z-indices |
| map-layer | `aspect-ratio: 9 / 17` | Preview-only phone-ish proportions |
| map-layer | `82%` drawer width | Content-adaptive percentage (canonical 312pt at 390pt viewport) |
| navigator-callouts | `backdrop-filter: blur(16px) saturate(1.2)` | Visual-effect blur radius |
| navigator-callouts | `border-top: dashed` on suggestions | Stylistic softener between error body + recovery chips |
| route-sheet | `flex: 1` / `flex: 2` | Save vs Ride ratio (UX decision, not tokenizable) |
| route-card | `.ls-card` padding override to `space-0` | Atom modifier pattern so embedded map extends edge-to-edge |
| route-card | `aspect-ratio: 9 / 4` on map slot | Enforces ~160pt map height at 360pt card width |
| sessions-drawer | `-webkit-line-clamp: 1` on preview | Non-standard CSS required for single-line ellipsis truncation |

| section-header | `align-items: baseline` | Ensures title + trailing chevron share typographic baseline |

## Cascade Notes

During Phase 4, organism subagent runs DID NOT emit any `VARIANT_REQUEST` for atoms or molecules — every organism composed cleanly from existing atom/molecule variants. Zero `TOKEN_GAP` requests were needed: the Phase 1 token system + Phase 2/3 class inventories covered the full organism surface area.

If a future organism requires a new atom variant (e.g., a dashed outline button, a new pill size), the request should be emitted via `VARIANT_REQUEST: atom={name} variant={class} recipe={css-rules}` at the start of the subagent reply. The orchestrator will extend the lower layer, regen its quartet, update the shared bundle, and resume.
