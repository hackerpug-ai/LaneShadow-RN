## Shared organism bundle

`views/_organisms.css` is the bundled concatenation of every organism's `<style>` block (generated from `organisms/*/*.html`). Views link it as `../_organisms.css` so they can compose organism classes (e.g. `.org-map-layer`, `.org-topbar`, `.org-route-sheet`, `.org-sessions-drawer`, `.org-navigator-callouts__msg`) without redefining any organism styling.

The bundle is regenerated by running the extraction script used during Phase 5 setup:

```python
# From .spec/design/system, in canonical organism order:
for name in ['topbar-navbar','map-layer','navigator-callouts','route-sheet',
             'sessions-drawer','route-card','section-header']:
    # Read organisms/{name}/{name}.html <style> block, append to views/_organisms.css
```

If an organism's style block changes, rerun the bundle extraction before viewing any view HTML.

## Responsive

Every view HTML declares:

- **Desktop (default, width > 375px):** full-width section wrappers with outer padding `var(--space-9)` (40pt). Phone frames centered at ~390pt × 844pt with `var(--radius-xl)` corners + `var(--stroke-sm) solid var(--border-default)` border.
- **Mobile (`@media (max-width: 375px)`):** section outer padding collapses to `var(--space-4)` (12pt). Phone frames expand to the section inner width. Some views also include an intermediate `@media (max-width: 900px)` breakpoint that stacks the caption band vertically.

Each view is rendered twice during Phase 5:

- Desktop PNG at 1320×6400 window (`{view}.png`)
- Mobile PNG at 375×6400 window (`{view}-mobile.png`)

Both are trimmed to content via a PIL bottom-detection pass that looks for the canvas color (`--paper-100` light / `--ink-800` dark) and snaps the height to the last non-canvas row plus a 40px pad.
