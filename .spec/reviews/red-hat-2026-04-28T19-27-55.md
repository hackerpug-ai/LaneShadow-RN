# Red-Hat Review Report

**Report Date**: 2026-04-28T19:27:55Z
**Target**: Sprint 02 — V2 Variants, Motion & Sandbox Coverage (FID-S02-T01..T10)
**Reviewed By**: swift-reviewer, kotlin-reviewer

## Executive Summary

Sprint 02 iOS implementation is substantially real (components exist, stories registered, snapshots captured) but test coverage is theatre — half the AC tests are empty stubs or TODO comments, and a critical token gap forces hardcoded durations. Android implementation is in significantly worse shape: 28 of 35 ACs FAIL or PARTIAL, with multiple missing molecule files, unregistered stories, wrong motion timings (400-600ms instead of 1400ms), and core callback wiring absent. Both reviewers agree: the sprint needs significant remediation before it can serve as a regression-protected baseline for Sprint 03.

## HIGH Confidence Findings (Both Reviewers Agree)

- [ ] **Finding 1**: Motion token gap — hardcoded durations on both platforms | Severity: CRITICAL
      Agents: swift-reviewer, kotlin-reviewer
      Evidence: iOS PlanningScreen.swift:11-14 hardcodes `1.4` with "TOKEN GAP" comment; Android PlanningScreen.kt:114-122 uses 600ms "deliberate" token, not 1400ms. Spec says "NEVER hardcode durations — if a key is missing, flag it and stop." Neither platform reads from a `theme.motion.sketchPolylineLoop` recipe token because it doesn't exist in `tokens/semantic/motion.json`.

- [ ] **Finding 2**: Test coverage is theatre — empty test bodies, TODO stubs, missing test files | Severity: CRITICAL
      Agents: swift-reviewer, kotlin-reviewer
      Evidence: iOS RouteResultsDetailsVariantTests.swift (T05, 8 ACs) does not exist. iOS SessionsErrorVariantTests.swift has 6 empty test bodies with "This test will fail until X is implemented" comments despite components existing. iOS MotionTests.swift has TODO comments ("Verify animation duration is 1400ms") and only asserts `XCTAssertNotNil(view)`. Android MotionTests.kt validates spec construction, not component usage — tests pass despite implementation using wrong values.

- [ ] **Finding 3**: Stories registered without implementation backing (Android) | Severity: CRITICAL
      Agents: kotlin-reviewer
      Evidence: 21 stories registered across IdleScreenStory.kt and PlanningScreenStory.kt on Android, but 14+ have no variant rendering logic in the actual screen composables. Stories render default UI instead of variant UI. iOS has the same pattern to a lesser degree — IdlePlanningVariantTests only check provider state, not UI rendering.

- [ ] **Finding 4**: Missing molecule/atom files on Android (LSCancelConfirmSheet, LSSavedPill, LSWifiOffWatermark, LSConfirmDialog) | Severity: CRITICAL
      Agents: kotlin-reviewer
      Evidence: Files referenced in task scope as NEW but do not exist in Android codebase. iOS counterparts exist (confirmed by swift-reviewer). Blocking 7+ ACs across T04, T06, T08.

- [ ] **Finding 5**: onRouteCardTap callback missing from LSNavigatorMessage (Android) | Severity: HIGH
      Agents: kotlin-reviewer
      Evidence: LSNavigatorMessage.kt:96-103 has `onPin` and `onDismiss` callbacks but no `onRouteCardTap: (String) -> Unit`. Blocks alt-selection re-promote flow (T06 AC-1).
