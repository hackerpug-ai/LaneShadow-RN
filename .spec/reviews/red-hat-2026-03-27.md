# Red-Hat Review Report

**Report Date**: 2026-03-27T05:43:00Z
**Target**: Epics 1-3 (Complete Weather Planning, Browse & View Saved Routes, Search Filter & Organize)
**Reviewed By**: product-manager, react-native-ui-reviewer, convex-reviewer

## Executive Summary

Three independent adversarial reviewers identified **28 distinct findings** across product, frontend, and backend concerns. The most critical issues cluster around: (1) a publicly-exposed hard-delete mutation that bypasses the undo flow, (2) the `collect()` full-table-scan pattern that will degrade as route counts grow, (3) a soft-delete race condition that can cause data loss, (4) SavedRouteCard's `onPress` being wired to only the chevron icon instead of the full card, and (5) a missing PRD acceptance criterion (location filtering). Several findings were independently discovered by multiple reviewers, increasing confidence.

---

## HIGH Confidence Findings (2+ Agents Agree)

### H1. SavedRouteCard `onPress` only fires on chevron icon, not the full card
**Severity: Critical** | **Agents: product-manager, react-native-ui-reviewer**

`components/ui/saved-route-card.tsx:115-121` -- `onPress` is attached to the `MaterialCommunityIcons` chevron, not the card's outer `View`. The card body (name, thumbnail, stats) is not tappable. Users must find and tap the small chevron to navigate. This is a major UX and accessibility defect.

**Remediation**: Wrap card in `Pressable` with `onPress` forwarded to the entire surface.

---

### H2. `collect()` fetches ALL user routes into memory -- no server-side filtering or pagination
**Severity: Critical** | **Agents: product-manager, convex-reviewer**

`convex/db/savedRoutes.ts:189` -- `listByOwner` calls `.collect()` loading every document for the owner. Soft-delete exclusion, search, date range, and limit are all applied in-memory afterward. No cursor pagination, no `hasMore` flag, no total count. The `MAX_LIST_LIMIT` of 50 only slices *after* the full scan.

A compound index `['ownerType', 'ownerId', 'createdAt']` would allow Convex to do index range filtering. Future epics (7-8) add more routes and query patterns, compounding this.

**Remediation**: Add compound index, push `deletedAt` and date range filtering to Convex `.filter()`, implement cursor-based pagination.

---

### H3. Soft-delete race condition -- double invocation schedules two permanent deletions
**Severity: High** | **Agents: product-manager, convex-reviewer**

`softDeleteRouteHandler` does not guard against `doc.deletedAt` already being set. If called twice (network retry, optimistic re-invocation), two `_scheduled_functions` entries are created. The first fires and deletes the doc; the second fires on a null doc (silent no-op). Additionally, `permanentlyDeleteRouteHandler` does not check whether `deletedAt` is still set before deleting -- if a user undoes at 4.9s and the scheduler fires at 5.0s, the route is permanently deleted despite the undo.

**Remediation**: Add guard in `softDeleteRouteHandler`: `if (doc.deletedAt !== undefined) return`. Add guard in `permanentlyDeleteRouteHandler`: `if (!doc.deletedAt) return null`.

---

### H4. `deleteRoute` (hard-delete) is a public mutation that bypasses soft-delete/undo
**Severity: High** | **Agents: product-manager (C4), convex-reviewer (#1)**

`savedRoutes.ts:398` exposes `deleteRoute` as a public mutation doing immediate permanent deletion. `softDeleteRoute` (line 408) provides the undo flow. Both are on the same API surface. The hook `useDeleteRoute` in `hooks/use-saved-routes.ts` still wires the hard-delete path. A client calling the wrong mutation gets no undo window.

**Remediation**: Remove or make `deleteRoute`/`deleteById` internal-only. Audit all consumers to ensure only `softDeleteRoute` is used.

---

### H5. UC-SR-02 AC "filter by start or end location" is completely unimplemented
**Severity: High** | **Agents: product-manager (#1)**

PRD `05-uc-sr.md` line 48 requires: "Rider can filter routes by start or end location." No task was created. No backend support. No UI. Epic 3 INDEX claims "100% of PRD acceptance criteria covered" -- this is false.

**Remediation**: Either implement location filtering or formally descope with PRD amendment.

---

### H6. Route card `path` prop hardcoded to empty string -- start/end locations missing
**Severity: High** | **Agents: product-manager (#2)**

UC-SR-01 AC: "Rider can see route name, date saved, start/end locations, and distance on each card." `saved-routes.tsx:166` passes `path=""`. The `getSavedRoutesList` query does not return `planInput` data. The card renders empty text where locations should appear.

**Remediation**: Add `planInput.start.label` / `planInput.end.label` to the list view API response and wire into the card.

## MEDIUM Confidence Findings (Single Agent, Strong Evidence)

### M1. `permanentlyDeleteRoute` performs NO ownership check
**Agent: convex-reviewer** | **Severity: Medium**

The `internalMutation` only checks `doc !== null` before deleting. No `requireIdentity`, no `isOwnedByViewer`. It's scheduler-only today, but carries no principal context. If accidentally promoted to public or called via a bug in another internal function, it deletes any route.

---

### M2. `patchName` and `deleteById` throw `new Error()` instead of `ConvexError`
**Agent: convex-reviewer** | **Severity: Medium**

Generic `Error` objects surface as opaque "Internal server error" to the client. All other auth-failure paths use `ConvexError`. The client cannot distinguish a "not found" from a crash. Same issue in `requireIdentity` (`guards.ts`).

---

### M3. OverlayToggleItem uses `onTouchEnd` instead of `Pressable`/`onPress`
**Agent: react-native-ui-reviewer** | **Severity: Medium**

`components/map/overlay-toggle.tsx:222` -- Breaks Android ripple feedback, accessibility activation (TalkBack/VoiceOver), keyboard navigation, and long-press cancellation. Same anti-pattern in `toggle-group.tsx:223`.

---

### M4. FlatList missing `keyboardShouldPersistTaps="handled"`
**Agent: react-native-ui-reviewer** | **Severity: Medium**

`saved-routes.tsx:190` -- When the keyboard is visible from the search bar, tapping a route card is swallowed by keyboard dismissal. Requires two taps to navigate on iOS.

---

### M5. SavedRouteCard has 11+ hardcoded style values, zero semantic tokens
**Agent: react-native-ui-reviewer** | **Severity: Medium**

`saved-route-card.tsx:129-161` -- `borderRadius: 16`, `padding: 16`, `marginBottom: 12`, `gap: 12`, `fontSize: 16`, `borderColor: 'rgba(255,255,255,0.05)'`, etc. This is the most-rendered component, yet has zero semantic token usage. The fix task US-023 corrected a single `gap: 8` in `route-search-bar.tsx` but the same rule was not applied to this critical component.

---

### M6. RouteThumbnail is a decorative placeholder, not an actual map preview
**Agent: product-manager** | **Severity: Medium**

UC-SR-01 AC says "mini map preview." The `RouteThumbnail` renders a gradient background with a rotated rectangle -- no map tiles, no route geometry, no geographic content.

---

### M7. `ctx as any` casts suppress type safety on 3 mutations + internal reference
**Agent: convex-reviewer** | **Severity: Medium**

`softDeleteRoute`, `undoDeleteRoute`, `permanentlyDeleteRoute` all use `ctx as any`. The `(internal as any).db.savedRoutes` cast bypasses generated type checking. If Convex APIs change, mismatches are masked until runtime.

---

### M8. No name validation on `saveRoute` / `insert` -- inconsistent with `patchName`
**Agent: convex-reviewer** | **Severity: Medium**

`saveRoute` accepts any name (500+ chars, all whitespace). `patchName` validates trim + 100 char limit. A route can be created with an invalid name but cannot be renamed to the same name.

---

### M9. Detail screen action buttons overlap OverlayToggle on small-inset devices
**Agent: react-native-ui-reviewer** | **Severity: Medium**

`[id].tsx:177-223` -- Action buttons at `top: insets.top` (zIndex 30), OverlayToggle at `top: semantic.space.xl` (24px, zIndex 25). On iPhone SE or non-notch Android, these overlap.

---

### M10. `RenameRouteDialog` resets draft input on real-time Convex sync
**Agent: react-native-ui-reviewer** | **Severity: Medium**

`rename-route-dialog.tsx:23-25` -- `useEffect` resets `name` to `currentName` on every change. If Convex pushes a name update while the dialog is open (e.g., viewed elsewhere), the user's draft is silently discarded.

---

### M11. SwipeableRouteCard `renderRightActions` re-created on every render
**Agent: react-native-ui-reviewer** | **Severity: Medium**

`saved-routes.components.tsx:220` -- Inline non-memoized function forces Swipeable to remeasure action width on every parent re-render (e.g., every search keystroke).

---

### M12. Double `requireIdentity` in `getSavedRoutesList` and `getSavedRouteDetail`
**Agent: convex-reviewer** | **Severity: Low**

Outer call discards the identity; inner call re-fetches it. Dead code that obscures where auth actually happens.

## Recommendations by Category

### 1. Gaps (PRD Coverage)
- **Implement or formally descope** location filtering (UC-SR-02 AC)
- **Wire start/end locations** into list API and card (`path` prop)
- **Replace RouteThumbnail placeholder** with actual static map image (or document as intentional MVP gap)

### 2. Risks (Data Integrity & Security)
- **Remove or internalize `deleteRoute`** -- only `softDeleteRoute` should be public
- **Add `deletedAt` guard** to both `softDeleteRouteHandler` (prevent double-schedule) and `permanentlyDeleteRouteHandler` (prevent deletion after undo)
- **Add ownership context** to `permanentlyDeleteRoute` or at minimum a `deletedAt` check
- **Fix error types**: `patchName`, `deleteById`, `requireIdentity` should use `ConvexError`
- **Add name validation** to `saveRoute`/`insert` matching `patchName` rules

### 3. Performance
- **Add compound index** `['ownerType', 'ownerId', 'createdAt']` to push date filtering to Convex
- **Push soft-delete filtering** to `.filter()` instead of in-memory
- **Implement cursor-based pagination** with `hasMore` flag
- **Memoize `renderRightActions`** in SwipeableRouteCard

### 4. UX & Accessibility
- **Wrap SavedRouteCard in Pressable** -- full card must be tappable
- **Replace `onTouchEnd` with `Pressable`/`onPress`** in OverlayToggle and ToggleGroup
- **Add `keyboardShouldPersistTaps="handled"`** to FlatList
- **Migrate SavedRouteCard styles** to semantic tokens
- **Guard RenameRouteDialog** against real-time sync resetting draft input
