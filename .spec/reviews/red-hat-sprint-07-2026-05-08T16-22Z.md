## HIGH Confidence Findings (Both Agents Agree)

### H-1: T05 AC-7 FAIL — All map controls callbacks are empty closures (iOS)
- **Severity**: CRITICAL
- **Agents**: swift-reviewer (file:line verified), kotlin-reviewer (cross-platform concern)
- **Evidence**: `ios/LaneShadow/Views/Templates/IdleScreen.swift:116-120` shows `onZoomIn: {}, onZoomOut: {}, onRecenter: {}, onLayers: {}, onToggleView: {}`
- **Contradiction**: REQUIREMENT-CONTRACT says `satisfied: true` citing evidence from `IdleScreenContainer.swift:99-101` — a different file than the one T05 modified. The template has empty closures; the container may have real wiring. The test exercises the container, not the template.
- **Impact**: Users can tap zoom/recenter/layers/toggle buttons and nothing happens. This violates CLAUDE.md's anti-stub mandate.
- **Fix**: Wire callbacks through IdleScreen to the LSMapHost camera proxy, or update IdleScreenContainer to override the template's empty closures and document this pattern.

### H-2: T04 AC-3 FAIL — Saved route uses wrong token (Android)
- **Severity**: CRITICAL
- **Agents**: kotlin-reviewer (file:line verified)
- **Evidence**: `LSMapControls.kt:216,221` uses `theme.colors.accent.default` instead of spec-required `theme.colors.signal.default`
- **Contradiction**: AC-3 explicitly requires `signal.default` copper fill. Design recipe specifies `signal.default` for background + border.
- **Impact**: Wrong copper color for saved route state on Android.
- **Fix**: Change `accent.default` to `signal.default` in LSMapControls.kt:216,221. Update AC-3 test to verify correct token.

### H-3: T05 AC-3 PARTIAL — Map controls vertical-centering not verified by test (iOS)
- **Severity**: HIGH
- **Agents**: swift-reviewer (test code review)
- **Evidence**: `IdleScreenRetrofitTests.swift` test `test_idle_rendersMapControlsVerticallyCentered` checks that controls exist (`!= nil`) but does not verify the "vertical middle of right edge" positioning requirement (±20pt of map canvas center).
- **Impact**: Controls could be top-aligned or bottom-aligned and the test would still pass.
- **Fix**: Add position verification using ViewInspector frame inspection or XCUITest coordinate assertions.

### H-4: T03 AC-5 PARTIAL — Zoom callback test doesn't verify actual behavior (iOS)
- **Severity**: HIGH
- **Agents**: swift-reviewer (test code review)
- **Evidence**: `LSMapControlsTests.swift:150-206` tests a `zoomCallbacksBound` boolean flag, never invokes callbacks or measures camera deltas. The AC describes "+1/-1 zoom on Mapbox camera" but the test only checks that callbacks were passed to the view.
- **Impact**: Test theatre — the test passes even if callbacks don't produce the required deltas.
- **Fix**: Implement real delta verification with a fake camera proxy, or update AC-5 description to match what's actually tested.

### H-5: Mode-toggle spec says "log+no-op" but implementation has no logging
- **Severity**: HIGH
- **Agents**: swift-reviewer
- **Evidence**: SPRINT.md and T05 spec say "mode-toggle stubbed to log+no-op until Sprint 08." `IdleScreen.swift:120` shows `onToggleView: {}` — no logging.
- **Impact**: Spec violation. User taps mode-toggle and gets zero feedback — not even a console log.
- **Fix**: Add logging: `onToggleView: { os_log(.info, "Mode toggle stub — Sprint 08") }` or update spec.

### H-6: Sprint 07 gate is BLOCKED (decisions.md D-001)
- **Severity**: HIGH
- **Agents**: kotlin-reviewer (documented in gate evidence)
- **Evidence**: `gate-evidence/decisions.md:11-42` — iOS sandbox full-screen template rendering deferred.
- **Impact**: Sprint 07 cannot be formally closed per its own gate requirements.
- **Fix**: Resolve the sandbox rendering issue or formally descoped the gate requirement.
