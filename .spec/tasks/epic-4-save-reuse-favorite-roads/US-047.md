# US-047: Pass favorites to planning graph as preferred segments

> Task ID: US-047
> Type: FEATURE
> Priority: P2
> Estimate: 120 minutes
> Assignee: backend-engineer

## CRITICAL CONSTRAINTS

### MUST
- Read convex/actions/agent/planRide.ts for existing planning patterns
- Read convex/actions/agent/graphs/planningGraph.ts
- Fetch user favorites when includeFavorites=true
- Pass favorite geometries to routing provider

### NEVER
- Break existing route planning without favorites
- Allow including favorites without authentication
- Modify routing provider signature significantly

### STRICTLY
- Only fetch favorites when includeFavorites=true
- Pass geometries as preferred segments to routing
- Handle case when user has no favorites

## SPECIFICATION

**Objective:** Integrate favorite roads into the route planning pipeline so routes can prefer user's favorite segments

**Success looks like:** When includeFavorites=true, generated routes show preference for favorite road segments

## DESIGN

### References

### Interaction Notes
- Fetch favorites via internalQuery (authenticated context)
- Pass favorite geometries as preferred segments to routing
- Routing provider returns metadata about which favorites were used
- Handle gracefully when no favorites or all too far

### Code Pattern

Source: convex/actions/agent/planRide.ts

```typescript
import { internalQuery } from '../_generated/server'
import { favoriteRoads } from '../db/favoriteRoads'

// Add to planning input
export const planInputValidator = v.object({
  // ... existing fields
  includeFavorites: v.optional(v.boolean()),
})

// In planRide action handler:
export const planRide = action({
  args: { input: planInputValidator },
  handler: async (ctx, args) => {
    // ... existing code

    let favoriteGeometries: string[] = []
    if (args.input.includeFavorites) {
      const favorites = await ctx.runQuery(internal.favoriteRoads.list)
      favoriteGeometries = favorites.map(f => f.geometry)
    }

    // Pass to routing provider
    const routes = await routingProvider({
      // ... existing params
      preferredSegments: favoriteGeometries,
    })

    // ... rest of planning logic
  },
})
```

Source: convex/actions/agent/providers/routingProvider.ts

```typescript
export const routingProvider = async ({
  start,
  end,
  preferences,
  preferredSegments = [], // New parameter

  // Pass preferred segments to routing API

  return routes.map(route => ({
    ...route,
    preferredSegmentsUsed: [], // Populated by routing API
  }))
}
```

### Anti-pattern (DO NOT)
- Don't always fetch favorites (only when includeFavorites=true)
- Don't fail if no favorites exist
- Don't modify route structure significantly
