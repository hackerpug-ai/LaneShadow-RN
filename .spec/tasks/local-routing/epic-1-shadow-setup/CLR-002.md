### Runtime Token Access

```typescript
// lib/mapbox/tokens.ts
import Constants from 'expo-constants';

export function getMapboxAccessToken(): string {
  const token = Constants.expoConfig?.extra?.mapboxAccessToken;
  
  if (!token) {
    throw new Error('MAPBOX_ACCESS_TOKEN not configured');
  }
  
  return token;
}

export function getMapboxDownloadToken(): string {
  const token = Constants.expoConfig?.extra?.mapboxDownloadToken;
  
  if (!token) {
    throw new Error('MAPBOX_DOWNLOAD_TOKEN not configured');
  }
  
  return token;
}
```

### Token Security Checklist

```markdown
# TOKEN_SECURITY.md

## Token Management Rules

1. **Never commit tokens to git**
   - Add `.env*` to `.gitignore`
   - Use pre-commit hook to check for tokens
   - Rotate any compromised tokens immediately

2. **Use separate tokens per environment**
   - Development: `pk.dev.xxx`
   - Staging: `pk.staging.xxx`
   - Production: `pk.prod.xxx`

3. **Token rotation**
   - Rotate tokens every 90 days
   - Document rotation in TOKEN_ROTATION.md
   - Update only environment variables

4. **Access control**
   - Limit token scopes to required APIs
   - Use download tokens for offline regions only
   - Monitor token usage in Mapbox dashboard

5. **Emergency procedures**
   - If token leaked: rotate immediately
   - If quota exceeded: upgrade plan or optimize usage
   - If token invalid: check env vars and rebuild
```
