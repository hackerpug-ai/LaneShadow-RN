```markdown
# docs/offline-regions-guide.md

# Offline Maps User Guide

## What Are Offline Maps?

Offline maps allow you to download map regions for use without an internet connection. This is especially useful for:

- Adventure riders in remote mountain areas
- Urban commuters in parking garages and tunnels
- Touring riders planning multi-state trips

## How to Download Offline Regions

### Step 1: Connect to WiFi

Offline map downloads require WiFi:
1. Connect to a WiFi network
2. Cellular downloads are blocked to prevent data overage charges

### Step 2: Access Offline Maps

1. Open Settings
2. Tap "Offline Maps"
3. Tap "Download New Region"

### Step 3: Select Your Region

1. Pan and zoom the map to your desired area
2. Drag the corners to adjust the bounding box
3. Name your region (e.g., "Rocky Mountains")
4. Tap "Download"

### Step 4: Monitor Download Progress

- Progress bar shows percentage complete
- Estimated time remaining displays
- You can leave the screen - download continues in background

## Managing Downloaded Regions

### View Your Regions

1. Open Settings > Offline Maps
2. See all downloaded regions with:
   - Name
   - Size
   - Download date
   - Last used date

### Delete a Region

1. Tap the region you want to delete
2. Tap "Delete Region"
3. Confirm deletion

## Storage Management

### Typical Region Sizes

- City: 50-100MB
- Metro area: 200-500MB
- State: 1-2GB
- Multi-state: 4GB+

### Check Available Storage

Before downloading, the app will:
1. Check available storage
2. Warn if insufficient space
3. Suggest deleting old regions

## Troubleshooting

### Download Fails

**Problem:** Download stops before completing

**Solutions:**
1. Check WiFi connection
2. Ensure sufficient storage
3. Try again (app resumes from last progress)

### Region Not Working

**Problem:** Route calculation fails in downloaded region

**Solutions:**
1. Verify region covers your route area
2. Check region status (should be "Complete")
3. Try re-downloading the region

## FAQ

**Q: Can I download regions on cellular?**
A: No, WiFi is required to prevent unexpected data charges.

**Q: How many regions can I download?**
A: As many as your device storage allows.

**Q: Do regions expire?**
A: No, but we recommend updating them monthly for latest map data.

**Q: Can I use offline maps for navigation?**
A: Yes! Route calculation, turn-by-turn directions, and map display all work offline.
```

```markdown
# docs/migration-notes.md

# Migration Notes: Google Maps to Mapbox

## Overview

LaneShadow migrated from Google Maps API to Mapbox SDK with @rnmapbox/maps for offline routing capability.

## Migration Date

2026-04-09

## Architecture Changes

### Before (Google Maps)

```
React Native
  └─ Google Maps Directions API (server-side)
      └─ Cost: $1,500/month
      └─ No offline capability
```

### After (Mapbox + Hybrid)

```
React Native
  └─ Mapbox SDK (client-side)
      ├─ Offline routing (free after download)
      ├─ Qwen3.5 local model (leg labels)
      └─ Haiku cloud (quality enrichment)
  └─ Cost: $30/month
```

## Key Technical Changes

### 1. Coordinate System

**Before:** [latitude, longitude]
**After:** [longitude, latitude]

**Impact:** All coordinate arrays must be converted before rendering.

**Solution:** `lib/polyline.ts` handles conversion automatically.

### 2. Route Geometry

**Before:** Google encoded polyline
**After:** Mapbox encoded polyline (polyline6 format)

**Impact:** Encoded polylines are not compatible.

**Solution:** Store geometry as string, convert on render.

### 3. Offline Capability

**Before:** No offline routing
**After:** Full offline routing with downloaded regions

**Impact:** New UX patterns for offline management.

**Solution:** Offline region download/management UI.

### 4. Enrichment

**Before:** Haiku-only enrichment
**After:** Hybrid (Qwen3.5 local + Haiku cloud)

**Impact:** Progressive loading, faster initial response.

**Solution:** Background enrichment jobs with progressive UI.

## Migration Steps

### Phase 1: Foundation (Week 1-2)
- Install @rnmapbox/maps
- Configure Expo plugin
- Create MapboxMapView wrapper
- Implement coordinate conversion

### Phase 2: Offline Maps (Week 3-5)
- Implement offline region download
- Add region management UI
- Implement offline route calculation
- Integrate local model for leg labels

### Phase 3: Sync (Week 6-7)
- Install @trestleinc/replicate
- Implement CRDT sync
- Add offline editing
- Test conflict resolution

### Phase 4: Weather (Week 8)
- Migrate weather overlay rendering
- Implement ShapeSource rendering
- Optimize batch rendering

### Phase 5: Launch (Week 9-10)
- QA testing
- Performance optimization
- Production build
- App store submission

## Breaking Changes

### API Changes

- `getDirections()` now requires offline region check
- Route geometry format changed to Mapbox polyline6
- Enrichment now progressive (not all-at-once)

### UI Changes

- New "Download Offline Maps" settings screen
- "Your Shadow" onboarding flow
- Progressive loading indicators
- Offline mode banner

### Data Changes

- `routes` table now Replicate-backed
- New `offline_regions` table
- `route_enrichments` has `phase` field

## Rollback Plan

If critical issues arise:

1. **Feature flag:** Disable Mapbox, revert to Google Maps
2. **Database:** Migrate routes back to Google format
3. **API:** Switch Directions API back to Google
4. **Timeline:** 1-2 days to rollback

1. **Cost savings:** 98% reduction in routing costs
```

```markdown
# docs/troubleshooting.md

# Troubleshooting Guide

## Common Issues

### Issue 1: "Download Your Shadow" Stuck at 0%

**Symptoms:**
- Model download progress shows 0%
- "Awakening Your Shadow" message displays
- No error message

**Causes:**
1. WiFi not connected
2. Insufficient storage
3. Network firewall blocking download

**Solutions:**
1. Verify WiFi connection in Settings
2. Check available storage (Settings > General > iPhone Storage)
3. Try different WiFi network (home vs work)
4. Restart app and retry download

**If persists:**
- Contact support with device model and iOS/Android version

---

### Issue 2: Route Calculation Fails with "Download Offline Maps First"

**Symptoms:**
- Error message when calculating route
- Route doesn't display on map

**Causes:**
1. No offline region downloaded for route area
2. Route extends outside downloaded region bounds

**Solutions:**
1. Download offline region for route area (Settings > Offline Maps)
2. Zoom map to ensure region covers entire route
3. Try calculating shorter route within downloaded region

**Prevention:**
- Download larger regions before planning trips
- Check region coverage before creating routes

---

### Issue 3: Weather Overlays Not Showing

**Symptoms:**
- Route displays but no weather colors
- Polylines show as single color

**Causes:**
1. Weather enrichment still in progress
2. Weather API rate limit reached
3. Network connectivity issue

**Solutions:**
1. Wait 20-30 seconds for weather enrichment
2. Check "Enhancing..." indicator
3. Pull to refresh weather data
4. Ensure device is online

**Expected Behavior:**
- Weather badges appear within 20 seconds
- Progress indicator shows during fetch

---

### Issue 4: App Crashes on Route Calculation

**Symptoms:**
- App crashes when calculating route
- Crash report shows Mapbox error

**Causes:**
1. Corrupted offline region
2. Insufficient memory
3. Mapbox SDK bug

**Solutions:**
1. Delete and re-download offline region
2. Close other apps to free memory
3. Restart device
4. Update app to latest version

**Report If:**
- Crash happens repeatedly
- Crash report shows "Mapbox" in stack trace

---

### Issue 5: Offline Edits Not Syncing

**Symptoms:**
- Route edits made offline
- Changes don't appear on other devices

**Causes:**
1. Device still offline
2. Sync conflict requiring resolution
3. Authentication token expired

**Solutions:**
1. Connect to WiFi or cellular
2. Pull to refresh on routes list
3. Check "Sync pending" indicator
4. Sign out and sign back in

**Expected Behavior:**
- Sync happens automatically when online
- "Sync pending" badge disappears when complete

---

### Issue 6: Local Model Inference Slow (> 2 seconds)

**Symptoms:**
- Leg labels take long time to generate
- "Generating labels..." shows for extended period

**Causes:**
1. First-time model loading
2. Device performance limitations
3. Model cache cleared

**Solutions:**
1. Wait for initial model load (one-time)
2. Close other apps to free resources
3. Re-download model if cache corrupted

**Expected Performance:**
- First load: ~2-3 seconds
- Subsequent: < 0.5 seconds

---

## Performance Issues

### Slow Map Rendering

**Symptoms:** Map stutters when panning/zooming

**Solutions:**
1. Reduce number of visible routes
2. Zoom in to reduce detail level
3. Close other apps
4. Restart app

### High Battery Usage

**Symptoms:** Battery drains faster than usual

**Causes:**
1. GPS tracking active
2. Background location updates
3. High screen brightness

**Solutions:**
1. Disable location when not needed
2. Reduce screen brightness
3. Use WiFi instead of cellular (for downloads)

## Error Messages

### "Model Download Failed"

**Meaning:** Local AI model couldn't download

**Action:** Connect to WiFi and retry

### "Insufficient Storage"

**Meaning:** Not enough device storage for region

**Action:** Delete unused offline regions or other apps

### "Route Not Found in Offline Data"

**Meaning:** Route area not covered by downloaded regions

**Action:** Download offline region for route area

### "Sync Conflict Detected"

**Meaning:** Same route edited on multiple devices

**Action:** Changes merged automatically (CRDT)
```
