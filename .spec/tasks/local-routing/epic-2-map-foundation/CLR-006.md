## DESIGN

### app.json Configuration

```json
{
  "expo": {
    "plugins": [
      [
        "@rnmapbox/maps",
        {
          "RNMapboxMapsImpl": "mapbox",
          "RNMapboxMapsAccessToken": "${RNMapboxMapsToken}",
          "RNMapboxMapsDownloadToken": "${RNMapboxMapsDownloadToken}"
        }
      ]
    ],
    "extra": {
      "RNMapboxMapsToken": null,
      "RNMapboxMapsDownloadToken": null
    }
  }
}
```

### Environment Variables

Create `.env` file (not committed):
```
RNMapboxMapsToken=your_access_token_here
RNMapboxMapsDownloadToken=your_download_token_here
```

### Token Sources

**Access Token:**
- Sign up at https://www.mapbox.com/
- Create access token in account dashboard
- Required for map rendering

**Download Token:**
- Found in Mapbox account dashboard
- Required for offline region downloads

### Prebuild Process

```bash
# Clean prebuild
npx expo prebuild --clean

# iOS only
npx expo prebuild -p ios

# Android only
npx expo prebuild -p android
```

## DESIGN NOTES

### Why Expo Plugin?

The @rnmapbox/maps Expo plugin:
- Automatically configures native projects
- Handles iOS and Android differences
- Manages native dependencies
- Simplifies upgrades

### Token Security

**Never commit tokens:**
- Use environment variables
- Add .env to .gitignore
- Document token setup separately
- Rotate tokens if compromised

### Prebuild vs Development Build

**Prebuild (this task):**
- Generates native files once
- Good for initial setup
- Must re-run after config changes

**Development Build (future):**
- Continuous native synchronization
- Better for active development
- Requires EAS setup

### Troubleshooting

**Common Issues:**
1. Token errors → Verify token validity in Mapbox dashboard
2. Build failures → Clear cache and run prebuild --clean
3. Plugin not found → Verify npm install completed
4. Platform-specific errors → Check iOS/Android logs separately
