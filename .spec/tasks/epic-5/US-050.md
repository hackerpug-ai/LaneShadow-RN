## DESIGN

### Output Shape

```typescript
export type ScenicWaypoint = {
  lat: number
  lng: number
  name: string
  type: 'viewpoint' | 'peak' | 'pass' | 'scenic_road'
  score: number  // 1-3, used for variant clustering
}

export type RouteVariant = {
  id: string               // e.g., 'scenic-north', 'scenic-south', 'direct-scenic'
  waypoints: ScenicWaypoint[]
}
```

### Overpass Query

```overpassql
[out:json][timeout:8];
(
  node["tourism"="viewpoint"](S,W,N,E);
  node["mountain_pass"="yes"](S,W,N,E);
  node["natural"="peak"]["name"](S,W,N,E);
  way["scenic"="yes"](S,W,N,E);
);
out body;
```

Where S/W/N/E are the padded bounding box from start+end coords.

### Scoring & Clustering

- `mountain_pass` â†’ score 3
- `natural=peak` â†’ score 2
- `tourism=viewpoint` â†’ score 1
- Filter: only nodes with a non-empty `name` tag
- Sort by score descending, take top 8
- Cluster into variants: if â‰¥4 nodes, create north-biased / south-biased / balanced subsets (by lat centroid); otherwise put all in one variant
- Each variant gets max 4 waypoints sorted by distance along the route corridor

### Fallback

```typescript
const FALLBACK: RouteVariant[] = [{ id: 'direct-scenic', waypoints: [] }]
```

Return `FALLBACK` on any error (network, timeout, parse failure, <2 nodes).
