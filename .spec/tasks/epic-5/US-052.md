- Input to LLM: waypoint names + route stats + user preferences (structured)
- Output from LLM: `{label, rationale, highlights[]}` per route
- No tool calls, no multi-turn conversation

## SPECIFICATION

**Objective:** After routes are compiled deterministically, make a single OpenAI call that generates human-readable labels, rationale, and highlights for all routes at once. The LLM receives structured data (real waypoint names from Overpass) and produces NL copy.

**Success looks like:** Routes returned from `planRideOrchestrator` have meaningful labels like "High Sierra Crest Loop via Tioga Pass" and rationales that describe the actual landmarks in the route.

## DESIGN

### Output Schema (enforced via `response_format: json_schema`)

```typescript
export type RouteEnrichment = {
  label: string        // e.g., "High Sierra Crest via Tioga Pass"
  rationale: string    // 1-2 sentences: why this route is scenic
  highlights: string[] // 2-4 short phrases: "Mountain pass", "Alpine meadows"
}
```

### Prompt Structure

```
System: You are a motorcycle route naming specialist. You receive structured route data 
and write compelling, accurate names and descriptions.

User:
Name and describe these {N} motorcycle routes. For each route, provide:
- label: a punchy route name (≤8 words) referencing the most iconic waypoint
- rationale: 1-2 sentences about why this route is scenic (mention waypoints)
- highlights: 2-4 short phrases (max 4 words each)

Route 1:
- Waypoints: {names}
- Distance: {X} miles, Duration: {Y} hours
- Preferences: scenic bias={high}, avoid highways={true}

Route 2:
...

Respond with a JSON object: { "routes": [ {label, rationale, highlights}, ... ] }
```

### Fallback

```typescript
const fallbackEnrichment = (idx: number): RouteEnrichment => ({
  label: `Route ${idx + 1}`,
  rationale: 'A scenic route through the area.',
  highlights: ['Scenic roads', 'Local character'],
})
```

### Code Pattern

```typescript
'use node'

import OpenAI from 'openai'
import { OPENAI_API_KEY } from '../../../lib/env'
import { withTimeout } from '../lib/reliability'

const ENRICH_TIMEOUT_MS = 10_000

export const enrichRoute = async (params: {
  routes: Array<{
    waypoints: Array<{ name: string; type: string }>
    stats: { distanceMeters: number; durationSeconds: number }
    preferences?: { scenicBias?: string; avoidHighways?: boolean }
  }>
}): Promise<RouteEnrichment[]> => {
  try {
    const client = new OpenAI({ apiKey: OPENAI_API_KEY })
    const model = process.env.ENRICH_MODEL ?? 'gpt-4o-mini'
    
    const result = await withTimeout(
      () => client.chat.completions.create({
        model,
        response_format: { type: 'json_object' },
        messages: [
          { role: 'system', content: SYSTEM_PROMPT },
          { role: 'user', content: buildUserPrompt(params.routes) },
        ],
        max_tokens: 500,
      }),
      { ms: ENRICH_TIMEOUT_MS, label: 'enrichRoute' }
    )

    const parsed = JSON.parse(result.choices[0].message.content ?? '{}')
    return (parsed.routes ?? []).map((r: any, idx: number) => ({
      label: r.label ?? `Route ${idx + 1}`,
      rationale: r.rationale ?? '',
      highlights: Array.isArray(r.highlights) ? r.highlights : [],
    }))
  } catch {
    console.warn('[enrichRoute] failed, using fallback labels')
    return params.routes.map((_, idx) => fallbackEnrichment(idx))
  }
}
```
