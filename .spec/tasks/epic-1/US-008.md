## CRITICAL CONSTRAINTS

### MUST
- Calculate expected rain timing based on departure time + segment arrival times
- Display human-readable time range (e.g., '2-4pm')
- Only show when rain is actually expected (not for clear routes)
- Use 12-hour format with am/pm

### NEVER
- Show timing if all segments are 'no rain'
- Show overlapping or confusing time ranges

### STRICTLY
- Time calculation must account for user timezone
- Show 'throughout ride' if entire route has rain

## SPECIFICATION

**Objective:** Display a human-readable rain timing summary on the route overview showing when rain is expected during the ride.

**Success looks like:** Users see 'Rain expected 2-4pm' on routes with precipitation, helping them decide when to depart.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Route departing at 1pm has rain in segments 2-4 (arriving ~2pm-4pm) | Route summary renders | Shows 'Rain expected 2-4pm' | `pnpm vitest run components/planning/rain-timing-summary.test.tsx --grep 'rain timing display'` |
| 2 | Route has no rain segments | Route summary renders | Rain timing section is not displayed | `pnpm vitest run components/planning/rain-timing-summary.test.tsx --grep 'no rain hidden'` |

| 4 | Rain data is unavailable | Route summary renders | Shows 'Rain data unavailable' in muted style | `pnpm vitest run components/planning/rain-timing-summary.test.tsx --grep 'unavailable timing'` |
