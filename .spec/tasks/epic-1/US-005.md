# US-005: Implement temperature-based polyline coloring (cold/comfortable/hot)

> Task ID: US-005
> Type: FEATURE

## CRITICAL CONSTRAINTS

### MUST
- Color route polyline segments based on temperature levels
- Use thermal color scheme: blue=cold, green=comfortable, orange/red=hot
- Apply coloring only when temperature overlay is selected
- Handle segment boundaries matching RouteSnapshot.overlays.temperature.byLeg

### NEVER
- Modify the actual route geometry - only change stroke colors
- Apply temperature colors when wind or rain overlay is active

### STRICTLY
- Colors must be accessible (WCAG contrast)
- Match temperature thresholds from TemperatureBadge component

## SPECIFICATION

**Objective:** Render route polyline segments with colors indicating temperature comfort (cold/mild/warm/hot) when temperature overlay is active.

**Success looks like:** When temperature overlay is toggled on, route polyline shows thermal colors allowing users to identify hot/cold zones.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Temperature overlay is active and route has cold segments (<40F) | Map renders the route | Cold segments display in blue (#60a5fa) color | `pnpm vitest run components/map/route-polyline.test.tsx --grep 'temp cold color'` |
| 2 | Temperature overlay is active and route has hot segments (>90F) | Map renders the route | Hot segments display in red (#ef4444) color | `pnpm vitest run components/map/route-polyline.test.tsx --grep 'temp hot color'` |
| 3 | Route passes through mild temperature zone (65-75F) | Temperature overlay is active | Mild segments display in green (#22c55e) color | `pnpm vitest run components/map/route-polyline.test.tsx --grep 'temp mild color'` |
| 4 | Temperature data missing for a leg | Temperature overlay renders that leg | Leg renders in neutral gray, not a misleading thermal color | `pnpm vitest run components/map/route-polyline.test.tsx --grep 'missing temp segment'` |
