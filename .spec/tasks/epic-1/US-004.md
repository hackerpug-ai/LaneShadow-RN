## CRITICAL CONSTRAINTS

### MUST
- Color route polyline segments based on rain intensity
- Use consistent color scheme: blue tints for rain (light=sky, moderate=blue, heavy=indigo)
- Apply coloring only when rain overlay is selected
- Handle segment boundaries matching RouteSnapshot.overlays.rain.byLeg

### NEVER
- Modify the actual route geometry - only change stroke colors
- Apply rain colors when wind or temp overlay is active

### STRICTLY
- Colors must be accessible (WCAG contrast on dark map)
- Segment transitions should be visually smooth

## SPECIFICATION

**Objective:** Render route polyline segments with colors indicating rain intensity (none/light/moderate/heavy) when rain overlay is active.

**Success looks like:** When rain overlay is toggled on, route polyline shows distinct colors for each rain intensity level, allowing users to visually identify precipitation risk zones.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | Rain overlay is active and route has light rain segments | Map renders the route | Light rain segments display in sky blue (#60a5fa) color | `pnpm vitest run components/map/route-polyline.test.tsx --grep 'rain light color'` |
