## CRITICAL CONSTRAINTS

### MUST
- Use existing ToggleGroup component from components/ui/toggle-group.tsx
- Support single-selection mode (only one overlay active at a time)
- Persist selected overlay in local state (not global/backend)
- Default to wind overlay when route is first selected

### NEVER
- Create new toggle component - use existing ToggleGroup
- Allow multiple overlays active simultaneously (single-select only)

### STRICTLY
- Position toggle above map controls, not obstructing route view
- Disable toggles when overlay data is unavailable

## SPECIFICATION

**Objective:** Add a toggle control to the map view allowing users to switch between wind, rain, and temperature overlay visualizations.

**Success looks like:** Users can tap toggle buttons to switch between wind/rain/temp overlays on the map, with the selected overlay highlighted and polyline colors updating accordingly.

## ACCEPTANCE CRITERIA

| # | Given | When | Then | Verify |
|---|-------|------|------|--------|
| 1 | User is viewing a selected route on the map | User taps the 'Rain' toggle option | Rain toggle becomes selected and polyline colors switch to rain-based coloring | `pnpm vitest run components/map/overlay-toggle.test.tsx --grep 'switch to rain'` |
| 2 | Route has wind data but no rain data (unavailable) | User views the overlay toggle | Rain toggle is disabled with visual indication | `pnpm vitest run components/map/overlay-toggle.test.tsx --grep 'disabled toggle'` |
