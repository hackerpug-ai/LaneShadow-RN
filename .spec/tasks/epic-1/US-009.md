## CRITICAL CONSTRAINTS

### MUST
- Calculate min and max temperatures across route segments
- Display in Fahrenheit for US locale
- Show both values: e.g., 'High 85F / Low 62F'
- Handle missing temperature data gracefully

### NEVER
- Mix Celsius and Fahrenheit
- Show temperature range wider than route duration warrants

### STRICTLY
- Round temperatures to nearest degree
- Use thermal color coding matching badges

## SPECIFICATION

**Objective:** Display high and low temperature values on the route summary to help riders prepare appropriate gear.

**Success looks like:** Users see temperature range (e.g., 'High 85F / Low 62F') on route overview, informing gear decisions.

### Code Pattern

```typescript
const calculateTempRange = (segments: TempSegment[]): {
  high: number,
  low: number
} | null => {
  const temps = segments.flatMap(s => s.values);
  if (!temps.length) return null;
  return {
    high: Math.max(...temps),
    low: Math.min(...temps)
  };
};
```

### Anti-pattern (DO NOT)
Do NOT calculate average - show actual high/low
