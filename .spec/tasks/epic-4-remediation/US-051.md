**Priority**: P0

## CRITICAL CONSTRAINTS

### MUST
- MUST connect SaveFavoriteSheet to favoriteRoads.insert mutation
- MUST capture segment geometry and bounds from long-press
- MUST validate user input (name required, max length)
- MUST show success feedback after saving
- MUST dismiss sheet after successful save

### NEVER
- NEVER allow empty favorite names
- NEVER bypass ownership validation (user must own created favorites)
- NEVER leave sheet open after successful save

### STRICTLY
- STRICTLY follow existing mutation patterns (savedRoutes, etc.)

- STRICTLY handle mutation errors with user-friendly messages
- STRICTLY reset form state after dismiss

## SPECIFICATION

**Objective**: Wire the SaveFavoriteSheet component to the backend favoriteRoads.insert mutation so users can successfully save route segments as favorites.

**Success looks like**: When a user long-presses a route segment, enters a name, and taps Save, the favorite is created in the database, the sheet dismisses, and success feedback is shown.

## NOTES

**Component Location**:
- SaveFavoriteSheet: `src/features/favorites/SaveFavoriteSheet.tsx` or similar
- Should already exist from US-043 (unwired component)

**Implementation Steps**:
1. Import useMutation hook for favoriteRoads.insert
2. Add onSave handler to SaveFavoriteSheet
3. Validate favorite name (required, max length, trim)
4. Call favoriteRoads.insert with: name, geometry, bounds, ownerId
5. Handle success: dismiss sheet, show toast, reset form
6. Handle error: display error message, keep sheet open
7. Test with invalid geometry bounds

**Mutation Parameters**:
```typescript
{
  name: string; // User input, trimmed
  geometry: Float64Array; // Segment coordinates from long-press
  bounds: { north, south, east, west }; // Segment bounds
  ownerId: string; // Current user ID (from auth)
}
```

**Form Validation**:
- Name required: Cannot be empty or whitespace only
- Max length: 50 characters (enforce in input)
- Trim: Remove leading/trailing whitespace
- Characters: Allow letters, numbers, spaces, basic punctuation

**Success Feedback**:
- Sheet dismisses with slide-down animation
- Toast message: "Favorite road saved!"
- Optional: Haptic success feedback
- Update favorites list if visible

**Error Handling**:
- Network error: "Couldn't save. Check your connection."
- Validation error: "Invalid favorite name."
- Auth error: "You must be signed in to save favorites."
- Geometry error: "Couldn't save this segment. Try a different one."

**State Management**:
- Reset form on successful save
- Keep form on error (allow retry)
- Clear geometry on dismiss
- Track loading state (disable Save button during mutation)

**Testing**:
- Unit test for onSave handler
- Integration test with favoriteRoads.insert
- Visual test for success/error states
- Manual test for form validation

**Security**:
- Validate ownerId matches current user (backend)
- Sanitize favorite name (prevent XSS)
- Validate geometry bounds (prevent malformed data)
- Rate limit saves (prevent spam)

**Edge Cases**:
- Very long names (>50 chars): Truncate or reject
- Special characters: Allow but sanitize
- Duplicate names: Allow (different segments)
- Same segment, different name: Allow
- Same segment, same name: Warn or allow

**Performance**:
- Mutation should be fast (<1s)
- Show loading spinner during save
- Optimistic UI update (optional)
