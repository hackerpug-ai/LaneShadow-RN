# TASK: US-053 - Accessibility Verification

**Task Type**: TESTING
**Priority**: P0
**Estimate**: 30 minutes

## CRITICAL CONSTRAINTS

### MUST
- MUST verify all touch targets meet 44×44px minimum (iOS/Android)
- MUST test with VoiceOver (iOS) and TalkBack (Android)
- MUST ensure all interactive elements are reachable via assistive tech
- MUST verify semantic labels and roles are correct
- MUST check color contrast meets WCAG 2.1 AA standards

### NEVER
- NEVER rely solely on automated testing (manual testing required)
- NEVER skip screen reader testing
- NEVER assume keyboard navigation works without testing (web)
- NEVER ignore accessibility warnings from Xcode/Android Studio

### STRICTLY
- STRICTLY follow platform accessibility guidelines (iOS Human Interface, Android Material)
- STRICTLY test on real devices, not simulators (for screen readers)
- STRICTLY document any a11y issues found with severity levels
- STRICTLY verify fixes before closing the task

## SPECIFICATION

**Objective**: Verify that the favorite roads feature meets accessibility standards, including touch targets, screen reader support, semantic labels, and color contrast.

**Success looks like**: All interactive elements are accessible via assistive technologies, touch targets meet minimum size requirements, and color contrast ratios pass WCAG 2.1 AA standards.

## NOTES

**Testing Equipment**:
- Physical iOS device (VoiceOver)
- Physical Android device (TalkBack)
- Contrast checker tool (web or plugin)
- Screen ruler/measurement tool

**Test Checklist**:

### Toggle (US-046)
- [ ] Reachable with VoiceOver/TalkBack
- [ ] Announces "Include favorite roads, toggle button, on/off"
- [ ] Touch target ≥44×44px (iOS) or 48×48dp (Android)
- [ ] State change announced immediately

### Long-Press (US-050)
- [ ] Alternative trigger available for screen reader users
- [ ] Haptic feedback can be disabled
- [ ] Visual highlight is visible (not color-only)
- [ ] Touch target on segment is sufficient

### SaveFavoriteSheet (US-051)
- [ ] All inputs have accessibility labels
- [ ] Save button announces "Save, button"
- [ ] Cancel button announces "Cancel, button"
- [ ] Error messages are announced
- [ ] Form validation errors are accessible

### Favorite Count Badge (US-048)
- [ ] Badge announces "3 favorite roads included"
- [ ] Zero count announced as "0 favorite roads included"
- [ ] Not reachable when toggle is OFF (correct behavior)
- [ ] Color contrast sufficient

### Exclusion Message (US-049)
- [ ] Full message announced when appears
- [ ] Dismissible via keyboard/touch
- [ ] Role="alert" or "status" for screen readers
- [ ] Auto-dismiss is announced

### Color Contrast
- [ ] All text meets 4.5:1 (normal) or 3:1 (large)
- [ ] Icons and graphics meet 3:1
- [ ] Focus indicators are visible
- [ ] Disabled states have sufficient contrast

### Touch Targets
- [ ] Toggle: ≥44×44px
- [ ] Save button: ≥44×44px
- [ ] Cancel button: ≥44×44px
- [ ] Dismiss buttons: ≥44×44px
- [ ] Route segments: Reasonable tap area

**Screen Reader Testing**:
1. Enable VoiceOver (iOS) or TalkBack (Android)
2. Navigate to PlanRideSheet
3. Swipe to toggle, verify label and state
4. Navigate to route, activate long-press alternative
5. Verify SaveFavoriteSheet is fully accessible
6. Check badges and messages are announced

**Contrast Testing**:
- Use online checker: WebAIM Contrast Checker
- Test all text colors on backgrounds
- Test badge colors
- Test message backgrounds
- Test disabled states

**Alternative Trigger for Long-Press**:
Since long-press is difficult for screen reader users, provide:
- Double-tap alternative
- Context menu option
- "Save as favorite" button in route details

**Documentation**:
Create US-053-a11y-report.md with:
- Test results (pass/fail for each item)
- Screenshots of contrast issues
- Severity levels: Critical, High, Medium, Low
- Recommended fixes for each issue

**Severity Levels**:
- **Critical**: Blocks screen reader users (fix before ship)
- **High**: Significant difficulty (fix before ship)
- **Medium**: Usable with difficulty (fix soon)
- **Low**: Minor annoyance (fix when possible)

**Common Issues to Check**:
- Missing accessibilityLabel on custom components
- Icon-only buttons without labels
- Color-only indicators (no text/icon alternative)
- Focus order issues
- Unlabeled form inputs
- Hidden elements announced to screen readers

**Testing Tips**:
- Test with eyes closed (simulate blind user)
- Test with screen zoom (simulate low vision)
- Test with switch control (motor impairment)
- Test without sound (hearing impairment)

**Tools**:
- iOS: Accessibility Inspector (Xcode)
- Android: Accessibility Scanner
- Web: axe DevTools, Lighthouse
- Contrast: WebAIM Contrast Checker

**Pass Criteria**:
- All critical issues fixed
- No high-severity issues
- Medium/low issues documented
- Screen reader flow works end-to-end
- All touch targets meet minimum size
- All contrast ratios meet WCAG AA
