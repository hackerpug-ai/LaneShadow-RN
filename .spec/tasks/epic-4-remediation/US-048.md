# TASK: US-048 - Build Favorite Inclusion Indicator

**Task Type**: FEATURE
**Priority**: P0
**Estimate**: 90 minutes

## CRITICAL CONSTRAINTS

### MUST
- MUST display count of favorites included in each route option
- MUST show "0 favorites" badge when count is zero
- MUST place badge alongside existing weather badges
- MUST use consistent styling with other route badges
- MUST be accessible with screen reader announcements

### NEVER
- NEVER show badge when includeFavorites toggle is OFF
- NEVER use confusing terminology (e.g., "saved roads" instead of "favorites")
- NEVER obstruct route distance/time information

### STRICTLY
- STRICTLY follow existing badge component patterns
- STRICTLY match weather badge styling and placement
- STRICTLY ensure badge updates when route options regenerate
- STRICTLY handle empty state gracefully (no favorites included)

## SPECIFICATION

**Objective**: Add a visual badge to each route option indicating how many of the user's favorite roads are included in that route.

**Success looks like**: Each route option card displays a badge showing the count of included favorites (e.g., "3 favorites") positioned alongside weather badges, with proper accessibility labels.

**Implementation Steps**:
1. Create FavoriteCountBadge component (if no generic badge exists)
2. Add badge to RouteOptionCard component
3. Pass includedFavorites count from route data
4. Conditionally render based on includeFavorites toggle state
5. Add accessibility label for screen readers
6. Style to match weather badges

**Badge Design**:
- Text: "{N} favorites" (e.g., "3 favorites", "0 favorites")
- Icon: Consider heart/star icon to indicate favorites
- Color: Match app's favorite/heart color (likely pink or red)
- Size: Match weather badge dimensions
- Position: Right side or below weather badges

**Accessibility**:
- AccessibilityLabel: "Route includes {N} favorite roads"
- Use native accessibility props (accessibilityLabel, role="text")
- Ensure contrast ratio meets WCAG AA standards
- Test with VoiceOver (iOS) and TalkBack (Android)

**Testing**:
- Unit test for badge rendering with different counts
- Integration test with RouteOptionCard
- Snapshot test for badge appearance
- Accessibility test with screen reader

**Edge Cases**:
- Count is 0: Still show badge ("0 favorites")
- Count is null/undefined: Don't show badge
- includeFavorites is false: Don't show badge
- Large counts (10+): Truncate or show "10+ favorites"

**Performance**:
- Badge render should be fast (no expensive calculations)
- Use React.memo if badge re-renders unnecessarily
