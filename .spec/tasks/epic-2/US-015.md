## SPECIFICATION

**Objective:** Create a full route detail screen that shows a saved route on a map with stats, weather badges, and route highlights when a user taps a card in the saved routes list.

**Success looks like:** User taps a saved route card, navigates to a detail screen showing the route on a full map, stats panel with distance/duration/legs, weather badges (wind/rain/temp), and route annotations as highlights.

### Interaction Notes
- Screen file: app/(app)/saved-route/[id].tsx (Expo Router dynamic segment)
- Screen receives savedRouteId via: `const { id } = useLocalSearchParams<{ id: string }>()`
- Layout: SafeAreaView > View flex:1 > MapViewWrapper (flex:0.5 top) + ScrollView (flex:0.5 bottom)
- Header: MapHeaderOverlay leftAction={icon:'arrow-left', onPress: router.back()}, title=route.name numberOfLines=1
- Map polylines via buildRoutePolylines with variant='selected', showLegs=true
- Map area accessibilityLabel: 'Route map for ' + route.name
- Stats section: StatRow for distance (formatDistance), duration (formatDuration), legs count
- Date saved caption: 'Saved {date}' using snapshotMeta.savedAt with semantic.color.onSurface.subtle
- Weather conditions row: WindBadge + RainBadge + TemperatureBadge side by side (flexDirection:'row', gap: semantic.space.sm)
- Highlights section: routeSnapshot.annotations as bulleted list (omit section if annotations empty)
- All section headers: Text variant labelMedium, semantic.color.onSurface.subtle, textTransform:'uppercase', letterSpacing:0.5
- ScrollView contentContainerStyle: paddingHorizontal semantic.space.lg, paddingTop semantic.space.md, paddingBottom safeArea.bottom + semantic.space.xl
- Back navigation preserves list scroll position automatically via expo-router native stack push semantics
