## SPECIFICATION

**Objective:** Add the OverlayToggle component to the route detail screen so users can switch between wind, rain, and temperature overlays on the saved route map.

**Success looks like:** Route detail screen shows OverlayToggle floating over the map, user can tap to switch overlays, and polyline colors update to reflect the selected overlay data.

### Interaction Notes
- Position OverlayToggle absolute top-right over the map area
- top = safeArea.top + semantic.space.xl, right = semantic.space.lg (mirrors home screen pattern)
- Derive OverlayAvailability: { wind: !!overlays.wind, rain: !!overlays.rain, temperature: !!overlays.temperature }
- Use useState<OverlayType | ''>('') for selected overlay state
- Default overlay on mount: '' (no overlay selected) or 'wind' if available
- Pass selected overlay to buildRoutePolylines to change line coloring

### Code Pattern
Source: app/(app)/(tabs)/index.tsx:348-365

```typescript
const [selectedOverlay, setSelectedOverlay] = useState<OverlayType | ''>('')

const availability: OverlayAvailability = {
  wind: !!data.routeSnapshot.overlays.wind,
  rain: !!data.routeSnapshot.overlays.rain,
  temperature: !!data.routeSnapshot.overlays.temperature,
}

// In render, positioned absolute over map:
<OverlayToggle
  selected={selectedOverlay}
  onSelect={setSelectedOverlay}
  availability={availability}
/>
```

### Anti-pattern (DO NOT)
Do NOT create a new toggle component. Do NOT store overlay selection in Convex or any persistent state. Do NOT use OverlayPill (it has a known non-interactive bug).
