CODE PATTERN (Reference)
--------------------------------------------------------------------------------

Example table definition pattern:
```typescript
// models/favorite-roads.ts (NEW FILE)
import { v } from 'convex/values'

export const favoriteRoadValidator = v.object({
  ownerType: v.union(v.literal('user'), v.literal('group'), v.literal('org')),
  ownerId: v.string(),
  createdByUserId: v.string(),
  visibility: v.union(v.literal('private'), v.literal('shared'), v.literal('public')),
  name: v.string(),
  geometry: v.object({
    format: v.literal('polyline'),
    encoding: v.string(),
    precision: v.number(),
    value: v.string(),
  }),
  bounds: v.object({
    north: v.number(),
    south: v.number(),
    east: v.number(),
    west: v.number(),
  }),
  createdAt: v.number(),
  updatedAt: v.number(),
})
```

Schema import pattern:
```typescript
// convex/schema.ts (MODIFY)
import { favoriteRoadValidator } from '../models/favorite-roads'

export default defineSchema({
  // ... existing tables
  favorite_roads: defineTable(favoriteRoadValidator)
    .index('by_ownerType_and_ownerId', ['ownerType', 'ownerId'])
    .index('by_createdByUserId', ['createdByUserId']),
})
```

**Why this pattern:**
- Mirrors saved_routes structure for consistency
- Supports user/group/org ownership (future-proof)
- Includes geometry for route segment storage
- Includes bounds for mini map preview
- Proper indexing for efficient queries

**Anti-pattern (DO NOT do this):**
```typescript
// DON'T: Define validator inline in schema.ts
favorite_roads: defineTable(v.object({
  userId: v.string(),
  name: v.string(),
  // ...
}))
```
