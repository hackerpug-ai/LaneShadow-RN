--------------------------------------------------------------------------------
BACKGROUND
--------------------------------------------------------------------------------

**Problem:** No existing badge pattern for showing favorite road inclusion in routes.

**Why it matters:** Users need visual feedback confirming that their favorites influenced route generation.

**Current state:** Route badges exist (rain, wind, temp) but no "favorites included" badge.

**Desired state:** Complete badge specification with icon, text, colors, and states for inclusion feedback.

--------------------------------------------------------------------------------
DESIGN REQUIREMENTS
--------------------------------------------------------------------------------

### Badge Placement

**Location**: On RouteOptionCard, in badges array
**Position**: After existing badges (rain, wind, temperature)
**Order`: [...existingBadges, favoritesBadge]

**Badge Container**:
- Layout: row (horizontal)
- Gap: 8px between badges
- Wrap: false (single line)
- Alignment: center

### Badge Component Specs

**Size**:
- Height: 24px
- Width: auto (expands to fit content)
- Padding horizontal: 8px
- Border radius: semantic.radius.sm (4px)
- Gap between icon and text: 4px

**Icon**:
- Name: "heart" (MaterialCommunityIcons)
- Size: 14px
- Color: dynamic (based on state)

**Text**:
- Variant: labelMedium (12px, 500 weight)
- Content: "{count} favorite{plural} included"
- Examples:
  - 0: "0 favorites included"
  - 1: "1 favorite included"
  - 2+: "N favorites included"

### States

**State 1: Included (count > 0)**
- Background: rgba(semantic.color.primary.default, 0.1)
- Border: 1px solid rgba(semantic.color.primary.default, 0.2)
- Icon color: semantic.color.primary.default
- Text color: semantic.color.primary.default
- Variant: "filled" (solid background)

**State 2: None Included (count = 0)**
- Background: transparent
- Border: 1px solid semantic.color.onSurface.subtle
- Icon color: semantic.color.onSurface.subtle
- Text color: semantic.color.onSurface.subtle
- Variant: "ghost" (no background)

**State 3: Favorites Disabled (includeFavorites = false)**
- Badge not shown
- Route card layout unchanged

### Colors
```
Included (count > 0):
  - Background: rgba(primary, 0.1)
  - Border: rgba(primary, 0.2)
  - Icon: semantic.color.primary.default
  - Text: semantic.color.primary.default

None Included (count = 0):
  - Background: transparent
  - Border: semantic.color.onSurface.subtle
  - Icon: semantic.color.onSurface.subtle
  - Text: semantic.color.onSurface.subtle
```

### Typography
```
Badge Text:
  - Variant: labelMedium
  - Size: 12px
  - Weight: 500
  - Letter spacing: 0.2 (slightly expanded)
  - Text transform: none (no uppercase)
```

### Accessibility

**Badge Label**:
- Format: "{count} favorite road{plural} included in this route"
- Examples:
  - 0: "0 favorite roads included in this route"
  - 1: "1 favorite road included in this route"
  - 3: "3 favorite roads included in this route"
- Role: text (informational, not interactive)
- Announcement: Part of route card label

**Screen Reader**:
- Route card announcement includes: "{route name}, {distance}, {duration}, {count} favorites included"

### Pluralization Rules
```
count === 1: "1 favorite included"
count !== 1: "{count} favorites included"
```

### Visual Hierarchy

**Priority** (in badges array):
1. Rain badge (if rain present)
2. Wind badge (if wind present)
3. Temperature badge (always present)
4. **Favorites badge (if includeFavorites = true)** ← NEW

**When to Hide**:
- If includeFavorites = false (toggle disabled)
- If route data doesn't include favorite count

### Edge Cases

**Case 1: count > 10**
- Badge shows actual count: "12 favorites included"
- Badge expands to fit text
- No truncation

**Case 2: Very long count**
- Badge still shows full number
- Badge may wrap to multiple lines if needed (avoid if possible)

**Case 3: Missing data**
- Badge not shown if count is undefined/null
- Graceful fallback to no badge
