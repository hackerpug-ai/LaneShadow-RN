BACKGROUND
--------------------------------------------------------------------------------

**Problem:** No existing long-press interaction pattern on route polylines.

**Why it matters:** Long-press is the primary interaction for saving favorite road segments. Users need clear visual feedback during the 500ms press duration.

**Current state:** Route polylines are tappable but have no long-press handling or visual feedback.

**Desired state:** Complete interaction specification with visual states, haptic feedback, and accessibility alternatives.

--------------------------------------------------------------------------------
DESIGN REQUIREMENTS
--------------------------------------------------------------------------------

### Visual States

**State 1: Idle**
- Route polyline displays normally
- No glow or highlight
- Opacity: 1.0

**State 2: Pressing (0-500ms)**
- Segment opacity: 0.6
- Subtle glow effect
- Glow color: semantic.color.primary.default at 30% opacity
- Blur radius: 4px

**State 3: Selected (500ms+)**
- Segment opacity: 0.8
- Strong glow effect
- Glow color: semantic.color.primary.default at 60% opacity
- Blur radius: 8px
- Haptic feedback: HapticFeedback.impactAsync(HapticFeedback.ImpactFeedbackStyle.Medium)

### Touch Target
- Minimum: 44×44px (WCAG 2.1 AA standard)
- Actual: Full polyline segment width (typically 6px stroke)
- Solution: Expand touch area to 44px around segment center

### Accessibility Alternative
- Add "Save segment as favorite" button for screen reader users
- Button appears in segment detail view
- Trigger: Long-press gesture OR tap button
- First-use hint: "Long-press a road segment to save it as a favorite"

### Colors
```
Pressing:
  - Border: semantic.color.primary.default with 0.3 opacity
  - Glow: rgba(primary, 0.3)
  - Shadow: blur radius 4

Selected:
  - Border: semantic.color.primary.default with 0.6 opacity
  - Glow: rgba(primary, 0.6)
  - Shadow: blur radius 8
```

### Animation Timing
```
0ms: User touches segment
0-300ms: Fade in pressing state (opacity 1.0 → 0.6)
300-500ms: Maintain pressing state
500ms: Trigger selected state
500-600ms: Fade in selected glow (opacity 0.6 → 0.8)
600ms+: Maintain selected until action sheet dismissed
```
