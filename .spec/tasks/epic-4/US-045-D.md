BACKGROUND
--------------------------------------------------------------------------------

**Problem:** Settings screen exists but is placeholder. No section layout for favorite roads list.

**Why it matters:** Settings section is the main interface for viewing and managing favorites. Users need clear navigation and list organization.

**Current state:** Settings screen has basic structure but no favorite roads section.

**Desired state:** Complete settings section specification with navigation, list layout, empty state, and delete confirmation.

--------------------------------------------------------------------------------
DESIGN REQUIREMENTS
--------------------------------------------------------------------------------

### Navigation Pattern

**Settings Menu Item**:
- Label: "Favorite Roads"
- Icon: "heart" (MaterialCommunityIcons)
- Icon color: semantic.color.primary.default
- Position: After "Saved Routes" (group with route-related settings)
- Navigation: Taps open FavoriteRoadsSection screen

**Screen Header**:
- Title: "Favorite Roads"
- Back button: Navigates to Settings
- Style: Standard navigation header

### List Layout

**Container**:
- Component: View (flex column)
- Background: semantic.color.background.default
- Padding: 16px horizontal, 16px top
- Safe area: bottom (for iOS home indicator)

**List Component**:
- Component: FlatList
- Data: favorite roads array
- Key extractor: favoriteRoadId
- Render item: FavoriteRoadCard
- Item separator: 8px vertical gap
- Max items: 50 (limit)
- Order: createdAt desc (newest first)
- Inverted: false

**Separator**:
- Component: View (height: 8px)
- Background: transparent
- No divider line (cards have borders)

**Refresh Behavior**:
- Pull to refresh: not implemented (favorites update via Convex reactivity)
- Auto-refresh: Yes (via useQuery reactivity)

### Empty State

**Layout**:
```
        [Heart Icon + Road Icon]
       No favorite roads yet
  Save a road segment to see it here
         [Plan a ride] [CTA]
```

**Component Specs**:
- Icon: MaterialCommunityIcons "heart" + "road-variant" (stacked or adjacent)
- Icon size: 64px
- Icon color: semantic.color.onSurface.muted
- Title: "No favorite roads yet"
- Title variant: titleMedium
- Title color: semantic.color.onSurface.default
- Subtitle: "Save a road segment to see it here"
- Subtitle variant: bodyMedium
- Subtitle color: semantic.color.onSurface.subtle
- CTA button: "Plan a ride"
- Button variant: default
- Button action: Navigate to planning screen
- Layout: center aligned, vertical column
- Padding: 64px vertical (center in viewport)
- Gap: semantic.space.lg (16px) between elements

### Delete Confirmation Dialog

**Dialog Type**: BottomSheetModal or Alert (Platform appropriate)

**Layout (iOS - Bottom Sheet)**:
```
┌────────────────────────────────────────┐
│         Remove Favorite Road           │
│                                        │
│  Are you sure you want to remove      │
│  "Alpine Loop" from your favorites?    │
│                                        │
│  This action cannot be undone.         │
│                                        │
│        [Cancel]    [Remove]            │
└────────────────────────────────────────┘
```

**Layout (Android - Alert)**:
```
Title: "Remove favorite road?"
Message: "Are you sure you want to remove "Alpine Loop" from your favorites? This action cannot be undone."
Buttons:
  - Cancel (negative)
  - Remove (destructive, red text)
```

**Component Specs**:
- Title: "Remove Favorite Road" (iOS) / "Remove favorite road?" (Android)
- Message: "Are you sure you want to remove "{name}" from your favorites? This action cannot be undone."
- Buttons: "Cancel" (outline), "Remove" (danger/destructive)
- Snap point (iOS): 40% height
- Stack behavior: push (can stack on other sheets)

**Colors**:
- Remove button: semantic.color.danger.default
- Cancel button: semantic.color.onSurface.default
- Message text: semantic.color.onSurface.default
