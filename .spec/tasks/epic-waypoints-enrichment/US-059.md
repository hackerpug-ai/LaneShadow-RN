# US-059: Implement enrichment invalidation trigger

> Task ID: US-059
> Type: FEATURE
> Priority: P0
> Estimate: 60 minutes
> Assignee: convex-implementer

## CRITICAL CONSTRAINTS

### MUST
- Hook into routing agent's route creation
- Cancel all pending/running enrichments for session
- Mark cancelled enrichments as 'cancelled'
- Cancel scheduled jobs

### NEVER
- Cancel completed enrichments
- Block route creation on invalidation
- Leave stale jobs running

### STRICTLY
- All stale enrichments cancelled before new route
- Scheduled jobs cancelled via ctx.scheduler.cancel
- Status updated to 'cancelled'

## SPECIFICATION

**Objective:** Automatically cancel pending enrichment jobs when user creates a new route in the same session.

**Success looks like:** New route creation triggers immediate cancellation of all stale enrichment jobs.

## DESIGN

### Pattern
```typescript
// In convex/db/routeEnrichments.ts
export const invalidateStaleEnrichments = mutation({
  args: {
    planningSessionId: v.id('planning_sessions'),
    newRoutePlanId: v.id('route_plans'),
  },
  handler: async (ctx, args) => {
    const staleEnrichments = await ctx.db
      .query('route_enrichments')
      .withIndex('by_planningSessionId_and_status', (q) =>
        q
          .eq('planningSessionId', args.planningSessionId)
          .eq('status', 'running')
      )
      .collect()

    for (const enrichment of staleEnrichments) {
      if (enrichment.scheduledJobId) {
        await ctx.scheduler.cancel(enrichment.scheduledJobId)
      }
      await ctx.db.patch(enrichment._id, {
        status: 'cancelled',
        updatedAt: Date.now(),
      })
    }
  },
})
```
