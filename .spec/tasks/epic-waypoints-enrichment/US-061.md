# US-061: Wire enrichment into routing agent

> Task ID: US-061
> Type: FEATURE
> Priority: P0
> Estimate: 60 minutes
> Assignee: pi-agent-implementer

## CRITICAL CONSTRAINTS

### MUST
- Add enrichment orchestration tools to routing agent
- Call invalidation when new route created
- Use forced tool-call pattern for structured output
- Handle enrichment failures gracefully

### NEVER
- Block route creation on enrichment
- Call enrichRoute directly (use orchestrator)
- Add new tools without testing

### STRICTLY
- Enrichment is orchestrator's responsibility
- Agent only orchestrates invalidation
- Failures don't prevent route creation

## SPECIFICATION

**Objective:** Integrate enrichment orchestration into routing agent tools so agent can manage enrichment lifecycle.

**Success looks like:** Routing agent can check cache, schedule enrichment, and cancel stale jobs.

## DESIGN

### Pattern
```typescript
// In convex/actions/agent/lib/piTools.ts
export const enrichmentOrchestrationTools = [
  {
    name: 'check_enrichment_cache',
    description: 'Check if enrichment exists for this route',
    parameters: Type.Object({
      contentFingerprint: Type.String(),
      phase: Type.String(),
    }),
  },
  {
    name: 'invalidate_stale_enrichment',
    description: 'Cancel pending enrichment jobs for this session',
    parameters: Type.Object({
      planningSessionId: Type.Id('planning_sessions'),
      newRoutePlanId: Type.Id('route_plans'),
    }),
  },
]

// In routingAgent.ts - after creating routePlan
if (ctx.args.planningSessionId) {
  await ctx.runMutation(internal.db.routeEnrichments.invalidateStaleEnrichments, {
    planningSessionId: ctx.args.planningSessionId,
    newRoutePlanId: newRoutePlanId,
  })
}
```
