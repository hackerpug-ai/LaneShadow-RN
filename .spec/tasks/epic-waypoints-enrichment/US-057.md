## CRITICAL CONSTRAINTS

### MUST
- Use MD5 hash for fingerprint generation
- Include start, end, departure time in fingerprint
- Bucket departure time to 5-minute intervals
- Return hex string hash

### NEVER
- Include non-deterministic values in fingerprint
- Use different hash algorithm without justification
- Cache enrichments without fingerprint

### STRICTLY
- Same inputs always produce same fingerprint
- Fingerprint changes when route parameters change
- Hash must be consistent across requests

## SPECIFICATION

**Objective:** Generate deterministic content fingerprints for enrichment cache keys and invalidation.

**Success looks like:** Function returns consistent hash for identical route parameters, different hash when parameters change.

## DESIGN

### Pattern
```typescript
export function generateContentFingerprint(planInput: PlanInput): string {
  const key = {
    start: { lat: planInput.start.lat, lng: planInput.start.lng },
    end: { lat: planInput.end.lat, lng: planInput.end.lng },
    departureTime: Math.floor(planInput.departureTime / 300000), // 5-min buckets
    preferences: planInput.preferences,
  }
  return crypto.createHash('md5').update(JSON.stringify(key)).digest('hex')
}
```

### Anti-pattern
Don't include timestamps or random values - breaks cache.
Don't use JSON.stringify without sorting keys - order matters.
