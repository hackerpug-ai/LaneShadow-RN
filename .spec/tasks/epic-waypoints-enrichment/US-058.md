## CRITICAL CONSTRAINTS

### MUST
- Hook enrichment into orchestrator return path (line 119-128)
- Return immediately with fallback labels (no blocking)
- Schedule background enrichment job after return
- Check cache before scheduling new job

### NEVER
- Block route generation on enrichment
- Return enriched labels immediately (must be async)
- Schedule enrichment without fingerprint check

- Orchestrator returns in <200ms with fallback labels
- Background job scheduled 100ms after return
- Cache hit skips job scheduling

## SPECIFICATION

**Objective:** Wire enrichment into orchestrator return path so routes return immediately while enrichment continues in background.

**Success looks like:** User sees routes instantly with fallback labels, then background enrichment updates UI when complete.

## DESIGN

### Pattern
```typescript
// In planRideOrchestrator, after line 128
const immediateResults = withConditions.map(({ routeSnapshot, sketch }, idx) => ({
  routeSnapshot,
  sketch: {
    ...sketch,
    label: sketch.label || `Route ${idx + 1}`,
    rationale: sketch.rationale ?? '',
  },
}))

// Non-blocking: schedule enrichment
const fingerprint = generateContentFingerprint(planInput)
const cached = await findCachedEnrichment(ctx, { clerkUserId, planInput, phase: 'fast' })

if (!cached) {
  const enrichmentId = await ctx.runMutation(internal.db.routeEnrichments.createEnrichment, {
    routePlanId: currentPlanId,
    clerkUserId,
    contentFingerprint: fingerprint,
    phase: 'fast',
  })

  await ctx.scheduler.runAfter(100, internal.actions.agent.enrichment.runEnrichmentJob, {
    enrichmentId,
    phase: 'fast'
  })
}

return immediateResults
```
