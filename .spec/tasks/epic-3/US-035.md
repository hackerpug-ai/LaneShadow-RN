## CRITICAL CONSTRAINTS

### MUST
- Add the same trim + length validation to `insert` handler that `patchName` already has
- Reject empty/whitespace-only names with `ConvexError('Route name cannot be empty')`
- Reject names over 100 characters with `ConvexError('Route name must be 100 characters or less')`
- Store the trimmed name (not the raw input)

### NEVER
- Change the `patchName` validation rules (this task makes `insert` match them)
- Change the public `saveRoute` mutation signature
- Add validation at the hook/UI layer only (must be backend-enforced)

### STRICTLY
- `insert` and `patchName` must enforce identical name rules
- Validation must happen before `ctx.db.insert`
- Error messages must match `patchName` exactly for consistency

## SPECIFICATION

**Objective:** Enforce consistent name validation on route creation matching the existing rename validation, preventing invalid names from being stored.

**Success looks like:** A route cannot be created with a 500-character name or all-whitespace name. The same rules apply at create and update time.
