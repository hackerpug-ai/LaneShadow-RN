## CRITICAL CONSTRAINTS

### MUST
- Add `startLabel` and `endLabel` to the `getSavedRoutesList` query response
- Populate `path` prop on `SavedRouteCard` with "{startLabel} → {endLabel}" format
- Source labels from `planInput.start.label` and `planInput.end.label`
- Handle missing/empty labels gracefully (fallback to empty string, no crash)

### NEVER
- Return the full `planInput` object in the list query (too much data for list view)
- Change the `SavedRouteCard` component API beyond using the existing `path` prop
- Break existing list performance by adding large payloads

### STRICTLY
- Only add the two label strings to the list view response (minimal data)
- The card must show the path text where it currently shows blank
- List query validator must be updated to include the new fields

**Objective:** Display start and end location labels on each saved route card to fulfill UC-SR-01 AC requirement "see start/end locations."

**Success looks like:** Each route card in the list shows "Downtown → Mountain Pass" (or similar) in the path area that currently displays empty.
