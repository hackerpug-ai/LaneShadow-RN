## CRITICAL CONSTRAINTS

### MUST
- Guard the `useEffect` in `RenameRouteDialog` so it only resets `name` state when the dialog opens (not while user is typing)
- Preserve the existing behavior of initializing the input with the current route name when dialog opens
- Ensure Convex real-time updates to `currentName` do not discard user's draft mid-typing

### NEVER
- Remove the `useEffect` entirely (it's needed for initialization on open)
- Add a debounce or timer-based workaround
- Change the dialog's public API

### STRICTLY
- The `name` state should only reset when the dialog transitions from hidden to visible
- Once the dialog is visible and the user is typing, external `currentName` changes must not reset the input
- Dialog must still show the latest name when opened fresh

## SPECIFICATION

**Objective:** Prevent real-time Convex sync from resetting the rename input field while the user is actively typing.

**Success looks like:** If a user is typing a new name and Convex pushes a `currentName` update (e.g., same route viewed elsewhere), the user's draft is preserved.

## DESIGN

### References
- components/ui/rename-route-dialog.tsx:23-25 (useEffect that resets on currentName change)

### Interaction Notes
- Track dialog `visible` state transition. Only reset `name` when `visible` changes from `false` to `true`.
- Use a `useRef` to track previous `visible` value, or use `visible` as a dependency with a guard.

### Code Pattern
```typescript
// BEFORE (resets on every currentName change):
useEffect(() => {
  setName(currentName)
}, [currentName])

// AFTER (only resets when dialog opens):
const prevVisibleRef = useRef(visible)
useEffect(() => {
  if (visible && !prevVisibleRef.current) {
    // Dialog just opened - initialize with current name
    setName(currentName)
  }
  prevVisibleRef.current = visible
}, [visible, currentName])
```

### Anti-pattern (DO NOT)
Do NOT use a "dirty" flag that requires manual reset. Do NOT debounce the effect. Do NOT remove the initialization behavior.
