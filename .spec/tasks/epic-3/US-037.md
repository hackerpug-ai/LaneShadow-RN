### Interaction Notes
- Map hardcoded values to nearest semantic tokens:
  - `borderRadius: 16` → `semantic.radius.lg` (or `md` depending on token values)
  - `padding: 16` → `semantic.space.lg`
  - `marginBottom: 12` → `semantic.space.md`
  - `gap: 12` → `semantic.space.md`
  - `gap: 4` → `semantic.space.xs`
  - `rgba(255,255,255,0.05)` → `semantic.color.surfaceVariant.default` or `semantic.color.outline.subtle`
  - Font sizes → Use React Native Paper `<Text variant="...">` props
- Read the theme to verify token values before mapping

### Code Pattern
Source: components/ui/route-search-bar.tsx (recently fixed for same issue)

```typescript
const { semantic } = useSemanticTheme()

// In styles:
style={[
  {
    borderRadius: semantic.radius.lg,
    padding: semantic.space.lg,
    marginBottom: semantic.space.md,
    gap: semantic.space.md,
    borderColor: semantic.color.outline.subtle,
  }
]}
```

### Anti-pattern (DO NOT)
Do NOT hardcode numeric values then comment "matches semantic.space.lg". Do NOT create new semantic tokens for this one component. Do NOT use inline styles where StyleSheet is already used (convert StyleSheet values to dynamic styles via the theme hook).
