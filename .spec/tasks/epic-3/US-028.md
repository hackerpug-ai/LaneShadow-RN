## CRITICAL CONSTRAINTS

### MUST
- Add rename and delete action buttons to the route detail screen header
- Wire `useRenameRoute` and new `useSoftDeleteRoute` / `useUndoDeleteRoute` hooks
- Show undo toast (via `react-native-notifier`) for 5 seconds after delete confirmation
- Navigate back to list after successful delete (when undo window expires or is dismissed)
- Update `hooks/use-saved-routes.ts` with `useSoftDeleteRoute` and `useUndoDeleteRoute` hooks

### NEVER
- Call permanent delete directly from the UI (always use soft delete flow)
- Dismiss the undo toast automatically before 5 seconds
- Hardcode any colors, spacing, or font sizes
- Show both rename and delete dialogs simultaneously

### STRICTLY
- Undo toast duration: exactly 5000ms with an "Undo" action button
- Tapping "Undo" on toast calls `undoDeleteRoute` and shows success notification
- After delete, navigate back to list only after toast is dismissed (user didn't undo)
- Rename success updates the header title immediately (Convex reactivity)
- All new hooks follow the existing `useMutationRunner` pattern

## SPECIFICATION

**Objective:** Wire the rename dialog, delete dialog, and undo toast into the route detail screen, connecting them to the backend mutations.

**Success looks like:** User can tap rename in header -> dialog opens -> save -> title updates. User can tap delete in header -> confirmation dialog -> confirm -> undo toast appears for 5 seconds -> route deleted and navigates back. Tapping undo within 5 seconds restores the route.

### Code Pattern
Source: hooks/use-saved-routes.ts (new hooks)

```typescript
export const useSoftDeleteRoute = () => {
  const mutation = useMutation(api.db.savedRoutes.softDeleteRoute)
  return useMutationRunner<{ savedRouteId: Id<'saved_routes'> }, null>(
    mutation,
    'Route deleted.'
  )
}

export const useUndoDeleteRoute = () => {
  const mutation = useMutation(api.db.savedRoutes.undoDeleteRoute)
  return useMutationRunner<{ savedRouteId: Id<'saved_routes'> }, null>(
    mutation,
    'Route restored.'
  )
}
```

### Anti-pattern (DO NOT)
Do NOT use setTimeout for the undo window (the toast duration handles timing). Do NOT call permanent delete from the UI. Do NOT navigate back immediately after delete (wait for toast to dismiss).
