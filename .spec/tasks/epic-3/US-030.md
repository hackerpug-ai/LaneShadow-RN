## CRITICAL CONSTRAINTS

### MUST
- Wrap the entire `SavedRouteCard` in a `Pressable` so the full card surface responds to taps
- Preserve the existing chevron icon as a visual affordance (but it no longer needs its own `onPress`)
- Add `accessibilityRole="button"` and `accessibilityLabel` to the pressable wrapper
- Maintain existing press visual feedback (opacity or highlight)

### NEVER
- Remove the chevron icon (keep as visual indicator)
- Add a nested `Pressable` inside another `Pressable` (single press target)
- Break the `SwipeableRouteCard` gesture handling (swipe must still work)

### STRICTLY
- The entire card surface must be tappable, not just the icon
- `onPress` prop fires when any part of the card is tapped
- Card must pass accessibility audit (TalkBack/VoiceOver can activate)

**Objective:** Make the entire SavedRouteCard surface tappable for navigation, not just the small chevron icon.

**Success looks like:** Tapping anywhere on a route card navigates to the detail screen. VoiceOver/TalkBack can activate the card as a single button.

### Interaction Notes
- Replace the outer `View` with `Pressable` and move `onPress` from the `MaterialCommunityIcons` to the wrapper
- The chevron icon becomes purely decorative (remove its `onPress`)
- Press feedback: use `android_ripple` + opacity change for cross-platform feel
- Must not conflict with swipe gesture in parent `SwipeableRouteCard`

### Code Pattern
Source: components/ui/button.tsx (Pressable usage)

```typescript
<Pressable
  onPress={onPress}
  accessibilityRole="button"
  accessibilityLabel={`View route: ${name}`}
  style={({ pressed }) => [
    styles.card,
    pressed && { opacity: 0.7 },
  ]}
>
  {/* card content */}
  <MaterialCommunityIcons name="chevron-right" ... />
</Pressable>
```

### Anti-pattern (DO NOT)
Do NOT nest a `Pressable` inside another `Pressable`. Do NOT use `TouchableOpacity` (use `Pressable` per RN best practices). Do NOT use `onTouchEnd` on a `View`.

## CODING STANDARDS

- **brain/docs/REACT-RULES.md**:
  - Use `Pressable` not `TouchableOpacity`
  - `accessibilityRole="button"` on interactive elements
  - Semantic theme tokens for any new styles
