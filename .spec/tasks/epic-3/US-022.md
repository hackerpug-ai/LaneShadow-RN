## CRITICAL CONSTRAINTS

### MUST
- Add name validation to `renameRoute` and `patchName`: trim whitespace, enforce 1-100 character length
- Return user-facing error messages via ConvexError for validation failures
- Trim the name before saving (strip leading/trailing whitespace)

### NEVER
- Change the existing `renameRoute` function signature (args stay the same)
- Add new dependencies for validation (use built-in string methods)
- Allow empty string or whitespace-only names after trimming

### STRICTLY
- Name length validation: 1-100 characters after trimming
- Error messages must be user-facing strings (not internal codes)
- Validation happens in `patchName` (the internal mutation) so all callers benefit

## SPECIFICATION

**Objective:** Add input validation to the rename mutation so users cannot save empty, whitespace-only, or excessively long route names.

**Success looks like:** `renameRoute({ name: "  " })` throws a user-facing error. `renameRoute({ name: "  Morning Ride  " })` succeeds and saves "Morning Ride" (trimmed).

```typescript
handler: async (ctx, args) => {
  const { clerkUserId } = await requireIdentity(ctx)
  const doc = await ctx.db.get(args.savedRouteId)
  if (!doc || !isOwnedByViewer(doc, clerkUserId)) {
    throw new Error('NOT_FOUND')
  }

  const trimmed = args.name.trim()
  if (trimmed.length === 0) {
    throw new ConvexError('Route name cannot be empty')
  }
  if (trimmed.length > 100) {
    throw new ConvexError('Route name must be 100 characters or less')
  }

  await ctx.db.patch(doc._id, { name: trimmed, updatedAt: Date.now() })
  return null
}
```
