## CRITICAL CONSTRAINTS

### MUST
- Add `deletedAt` optional field to saved_routes schema (epoch ms, null = not deleted)
- Create `softDeleteRoute` mutation that sets `deletedAt` and schedules permanent delete after 5 seconds
- Create `undoDeleteRoute` mutation that clears `deletedAt` and cancels the scheduled deletion
- Exclude soft-deleted routes from `listByOwner` and `getSavedRoutesList` results
- Create `permanentlyDeleteRoute` internal mutation for the scheduler callback

### NEVER
- Remove the existing `deleteById` internal mutation (keep for direct deletion if needed)
- Allow undo after permanent deletion has executed
- Change the existing `deleteRoute` public mutation signature (add new `softDeleteRoute` instead)

### STRICTLY
- Soft delete window is exactly 5000ms (5 seconds)
- `undoDeleteRoute` must throw if route is already permanently deleted
- Scheduled deletion ID must be stored on the document for cancellation
- `softDeleteRoute` returns `{ scheduledDeletionId: string }` for the undo flow

## SPECIFICATION

**Objective:** Implement a soft-delete pattern with 5-second undo window using Convex scheduled functions, enabling the delete-with-undo UX flow.

**Success looks like:** Calling `softDeleteRoute` marks the route as deleted (hidden from lists), schedules permanent deletion in 5 seconds, and returns a handle. Calling `undoDeleteRoute` within 5 seconds restores the route. After 5 seconds, the route is permanently deleted.

### Code Pattern
Source: Convex scheduled functions pattern

```typescript
// softDeleteRoute mutation:
export const softDeleteRoute = mutation({
  args: { savedRouteId: v.id('saved_routes') },
  handler: async (ctx, args) => {
    // 1. Verify ownership
    // 2. Set deletedAt = Date.now()
    // 3. Schedule permanent deletion
    const scheduledId = await ctx.scheduler.runAfter(
      5000,
      internalSavedRoutes.permanentlyDelete,
      { savedRouteId: args.savedRouteId }
    )
    // 4. Store scheduledId on document
    await ctx.db.patch(args.savedRouteId, {
      deletedAt: Date.now(),
      scheduledDeletionId: scheduledId,
    })
  },
})

// undoDeleteRoute mutation:
export const undoDeleteRoute = mutation({
  args: { savedRouteId: v.id('saved_routes') },
  handler: async (ctx, args) => {
    // 1. Verify ownership and that route exists
    // 2. Cancel scheduled deletion
    // 3. Clear deletedAt and scheduledDeletionId
  },
})
```
