## CRITICAL CONSTRAINTS

### MUST
- Add optional `afterDate` and `beforeDate` (epoch ms) args to `getSavedRoutesList` and `listByOwner`
- Filter routes where `createdAt` falls within the date range (inclusive)
- Compose with existing `searchQuery` filter from US-019 (both can be active simultaneously)
- Return the same response shape as existing query

### NEVER
- Add new database indexes for date filtering (post-fetch filter is sufficient for MVP)
- Change the `createdAt` field semantics or type
- Break backward compatibility when both date args are omitted

### STRICTLY
- `afterDate` is inclusive lower bound: `createdAt >= afterDate`
- `beforeDate` is inclusive upper bound: `createdAt <= beforeDate`
- Either arg can be provided independently (afterDate-only, beforeDate-only, or both)

## SPECIFICATION

**Objective:** Extend `getSavedRoutesList` to accept optional date range bounds and filter routes by creation date.

**Success looks like:** Calling `getSavedRoutesList({ afterDate: lastWeekMs })` returns only routes created within the last week. Combines with searchQuery when both provided.

### Interaction Notes
- Date filtering composes with name search: apply both filters in sequence
- Filter chain: fetch all -> filter by name -> filter by date -> apply limit
- `afterDate`/`beforeDate` are epoch milliseconds (same unit as `createdAt`)

### Code Pattern
Source: convex/db/savedRoutes.ts:81-97 (extend filter chain)

```typescript
// Add to args:
afterDate: v.optional(v.number()),
beforeDate: v.optional(v.number()),

// In handler, chain filters:
let filtered = rows
if (searchQuery) {
  filtered = filtered.filter(r => r.savedRoute.name.toLowerCase().includes(searchQuery.toLowerCase()))
}
if (afterDate) {
  filtered = filtered.filter(r => r.savedRoute.createdAt >= afterDate)
}
if (beforeDate) {
  filtered = filtered.filter(r => r.savedRoute.createdAt <= beforeDate)
}
```

### Anti-pattern (DO NOT)
Do NOT create separate query functions for date filtering. Do NOT add database indexes. Do NOT use Date objects instead of epoch ms.
