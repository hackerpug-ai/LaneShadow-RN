### Interaction Notes
- `"handled"` means: the keyboard stays visible, but taps on pressable children are handled. The keyboard dismisses when the user taps something that isn't a responder.
- This is a one-line fix with outsized UX impact on iOS

### Code Pattern
Source: React Native FlatList docs

```typescript
<FlatList
  keyboardShouldPersistTaps="handled"
  // ... existing props
/>
```

### Anti-pattern (DO NOT)
Do NOT use `keyboardDismissMode="on-drag"` as a substitute -- that's a different behavior. Do NOT use `"always"` -- it prevents keyboard dismissal entirely.
