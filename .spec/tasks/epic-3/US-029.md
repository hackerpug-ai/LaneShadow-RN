**Objective:** Add swipe-to-delete gesture to route cards in the saved routes list, providing a quick alternative to the detail screen delete flow.

**Success looks like:** User swipes left on a route card, a red delete area slides in from the right. Completing the swipe opens the delete confirmation dialog. After confirming, the soft-delete + undo toast flow activates (same as US-028).

### Code Pattern
Source: react-native-gesture-handler Swipeable pattern

```typescript
import { Swipeable } from 'react-native-gesture-handler'

const SwipeableRouteCard = ({ children, onDelete }: { children: React.ReactNode, onDelete: () => void }) => {
  const { semantic } = useSemanticTheme()
  const swipeableRef = useRef<Swipeable>(null)

  const renderRightActions = () => (
    <Pressable
      onPress={() => {
        swipeableRef.current?.close()
        onDelete()
      }}
      style={{
        backgroundColor: semantic.color.error.default,
        justifyContent: 'center',
        alignItems: 'center',
        width: 80,
        borderRadius: semantic.radius.lg,
      }}
      testID="swipe-delete-action"
    >
      <MaterialCommunityIcons name="trash-can-outline" size={24} color="#fff" />
    </Pressable>
  )

  return (
    <Swipeable ref={swipeableRef} renderRightActions={renderRightActions}>
      {children}
    </Swipeable>
  )
}
```

### Anti-pattern (DO NOT)
Do NOT bypass the confirmation dialog on swipe. Do NOT modify SavedRouteCard internals. Do NOT use PanGestureHandler directly (use Swipeable). Do NOT hardcode the delete area color.
