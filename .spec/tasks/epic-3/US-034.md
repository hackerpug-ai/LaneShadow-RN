## SPECIFICATION

**Objective:** Standardize error handling so all not-found/auth-failure paths use ConvexError, enabling client-side error display.

**Success looks like:** When `patchName` or `deleteById` fail on not-found/ownership check, the client receives a structured `ConvexError` with a human-readable message instead of an opaque "Internal server error."
